> **CSAPP** Chapter 12 요약

## 핵심 주장

**동시성은 현대 시스템의 핵심이다.** 스레드, 동기화, 레이스 컨디션을 이해해야 한다.

## 동시성이란?

### 왜 동시성?

이유:
- **성능**: 멀티코어 활용
- **응답성**: 블로킹 회피
- **자연스러운 모델링**: 독립적 활동

### 동시성 vs 병렬성

- **동시성**: 논리적으로 동시 (인터리빙)
- **병렬성**: 물리적으로 동시 (멀티코어)

## 동시성 접근법

### 1. 프로세스 기반

```c
if (fork() == 0) {
    // 자식: 클라이언트 처리
    close(listenfd);
    handle_client(connfd);
    close(connfd);
    exit(0);
}
// 부모: 계속 수락
close(connfd);
```

특성:
- 주소 공간 분리
- IPC 필요
- 오버헤드 큼

### 2. I/O 멀티플렉싱

```c
fd_set read_set;
FD_ZERO(&read_set);
FD_SET(listenfd, &read_set);

while (1) {
    fd_set ready = read_set;
    select(maxfd+1, &ready, NULL, NULL, NULL);

    if (FD_ISSET(listenfd, &ready)) {
        // 새 연결
    }
    for (int i = 0; i <= maxfd; i++) {
        if (FD_ISSET(i, &ready)) {
            // 데이터 처리
        }
    }
}
```

특성:
- 단일 스레드
- 이벤트 기반
- 상태 관리 복잡

### 3. 스레드 기반

```c
void *thread_func(void *arg) {
    int connfd = *((int *)arg);
    free(arg);
    handle_client(connfd);
    close(connfd);
    return NULL;
}

while (1) {
    int *connfd = malloc(sizeof(int));
    *connfd = accept(listenfd, ...);
    pthread_create(&tid, NULL, thread_func, connfd);
}
```

특성:
- 주소 공간 공유
- 가벼움
- 동기화 필요

## Posix 스레드

### 기본 API

```c
// 생성
pthread_create(&tid, NULL, func, arg);

// 종료
pthread_exit(retval);

// 대기
pthread_join(tid, &retval);

// 분리
pthread_detach(tid);
```

## 동기화

### 레이스 컨디션

```c
// 위험: 공유 변수
volatile int cnt = 0;

void *thread_func(void *arg) {
    for (int i = 0; i < N; i++)
        cnt++;  // 레이스!
    return NULL;
}
```

### 뮤텍스

```c
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;

void *thread_func(void *arg) {
    pthread_mutex_lock(&lock);
    cnt++;
    pthread_mutex_unlock(&lock);
    return NULL;
}
```

### 세마포어

```c
sem_t sem;
sem_init(&sem, 0, 1);

void *thread_func(void *arg) {
    sem_wait(&sem);  // P
    // 임계 영역
    sem_post(&sem);  // V
    return NULL;
}
```

## 동기화 패턴

### 생산자-소비자

```c
// 공유 버퍼
int buf[N];
int in = 0, out = 0;
sem_t empty, full, mutex;

void produce(int item) {
    sem_wait(&empty);
    sem_wait(&mutex);
    buf[in] = item;
    in = (in + 1) % N;
    sem_post(&mutex);
    sem_post(&full);
}

int consume() {
    sem_wait(&full);
    sem_wait(&mutex);
    int item = buf[out];
    out = (out + 1) % N;
    sem_post(&mutex);
    sem_post(&empty);
    return item;
}
```

### 리더-라이터

```c
int readcnt = 0;
sem_t mutex, w;

void reader() {
    sem_wait(&mutex);
    readcnt++;
    if (readcnt == 1)
        sem_wait(&w);
    sem_post(&mutex);

    // 읽기

    sem_wait(&mutex);
    readcnt--;
    if (readcnt == 0)
        sem_post(&w);
    sem_post(&mutex);
}

void writer() {
    sem_wait(&w);
    // 쓰기
    sem_post(&w);
}
```

## 스레드 안전

### 스레드 안전 함수

조건:
- 전역 변수 보호
- 정적 변수 없음
- 재진입 가능

### 재진입 함수

더 강한 조건:
- 공유 데이터 없음
- 락 없이 안전

### 예시

```c
// 스레드 불안전
char *ctime(const time_t *timep);  // 정적 버퍼

// 스레드 안전
char *ctime_r(const time_t *timep, char *buf);  // 버퍼 전달
```

## 데드락

### 조건

4가지 필요조건:
1. 상호 배제
2. 점유 대기
3. 비선점
4. 순환 대기

### 방지

순환 대기 방지:
- 순서대로 락 획득
- 타임아웃

```c
// 항상 같은 순서
pthread_mutex_lock(&lock1);
pthread_mutex_lock(&lock2);
```

## 병렬 프로그램 성능

### 속도 향상

$$
S_p = \frac{T_1}{T_p}
$$

### 효율

$$
E_p = \frac{S_p}{p}
$$

### Amdahl의 법칙

$$
S_p = \frac{1}{s + \frac{1-s}{p}}
$$

s = 순차 부분
