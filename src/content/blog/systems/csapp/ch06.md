## 핵심 주장

**메모리 계층을 이해하면 성능이 10배 차이난다.** 캐시 친화적 코드가 핵심이다.

## 저장 장치 기술

### SRAM vs DRAM

| 특성 | SRAM | DRAM |
|-----|------|------|
| 속도 | 빠름 | 느림 |
| 가격 | 비쌈 | 저렴 |
| 밀도 | 낮음 | 높음 |
| 용도 | 캐시 | 메인 메모리 |

### 디스크 (HDD)

구성:
- 플래터 (자기 표면)
- 헤드 (읽기/쓰기)
- 섹터 (최소 단위)

접근 시간:
- 탐색 시간 (seek): ~8ms
- 회전 지연 (rotational): ~4ms
- 전송 시간: 상대적으로 작음

### SSD

특성:
- 플래시 메모리 기반
- 무작위 접근 빠름
- 쓰기 마모 문제

## 지역성 (Locality)

### 시간 지역성 (Temporal)

최근 접근 → 곧 다시 접근:
- 루프 변수
- 자주 쓰는 함수

### 공간 지역성 (Spatial)

주변 주소 접근:
- 배열 순회
- 순차적 명령어

### 지역성 예시

```c
// 좋은 지역성
for (i = 0; i < n; i++)
    sum += a[i];

// 나쁜 지역성
for (i = 0; i < n; i++)
    sum += a[random() % n];
```

주소 분할:

![캐시 주소 분할](/images/blog/csapp/diagrams/ch06-address-partition.svg)

### 캐시 종류

| 종류 | E (연관도) | 특성 |
|-----|-----------|------|
| 직접 매핑 | 1 | 단순, 충돌 많음 |
| 세트 연관 | 2-8 | 균형 |
| 완전 연관 | C/B | 충돌 없음, 비쌈 |

### 캐시 동작

**읽기**:
1. 주소에서 세트 인덱스 추출
2. 태그 비교
3. 히트 → 데이터 반환
4. 미스 → 하위 계층에서 로드

**쓰기**:
- **Write-through**: 즉시 메모리 반영
- **Write-back**: 나중에 반영 (dirty bit)

### 교체 정책

미스 시 교체:
- **LRU**: 가장 오래된 것
- **Random**: 무작위
- **FIFO**: 먼저 들어온 것

## 캐시 성능

### 미스 종류

1. **Cold miss**: 처음 접근
2. **Capacity miss**: 용량 부족
3. **Conflict miss**: 같은 세트 경합

### 성능 지표

- **Hit rate**: 히트 비율
- **Miss rate**: 미스 비율
- **Miss penalty**: 미스 시 추가 시간

## 캐시 친화적 코드

### 행 우선 vs 열 우선

```c
// 좋음: 행 우선 (C 배열 저장 순서)
for (i = 0; i < N; i++)
    for (j = 0; j < N; j++)
        sum += a[i][j];

// 나쁨: 열 우선
for (j = 0; j < N; j++)
    for (i = 0; i < N; i++)
        sum += a[i][j];
```

### 블로킹

큰 배열을 블록 단위로:

```c
// 블로킹 행렬 곱셈
for (kk = 0; kk < N; kk += B)
  for (jj = 0; jj < N; jj += B)
    for (i = 0; i < N; i++)
      for (j = jj; j < jj+B; j++)
        for (k = kk; k < kk+B; k++)
          C[i][j] += A[i][k] * B[k][j];
```

## 시사점

1. **지역성 활용**: 시간적, 공간적
2. **캐시 구조 이해**: 세트, 라인, 블록
3. **스트라이드 최소화**: 순차 접근이 좋음
4. **작업 세트 크기**: 캐시에 맞추기
