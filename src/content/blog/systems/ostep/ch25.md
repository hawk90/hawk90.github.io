> **OSTEP** Chapter 25 요약

## 문제

파일 추가 = 3개 쓰기:
1. inode
2. 비트맵
3. 데이터

중간에 크래시 → 불일치

## 해결책

### FSCK

부팅 시 검사:
- 일관성 확인
- 문제 수정
- **느림**

### 저널링

트랜잭션 로그:
1. 변경 내용을 로그에
2. 커밋
3. 실제 적용
4. 로그 삭제

크래시 시:
- 로그 확인
- redo 또는 무시

### 저널링 모드

- **Data**: 모든 것 저널
- **Ordered**: 메타만 저널 (데이터 먼저)
- **Writeback**: 메타만 저널 (순서 없음)
