> **OSTEP** Chapter 26 요약

## LFS 아이디어

모든 쓰기를 순차적으로:
- 디스크를 로그처럼
- 쓰기 성능 최대화

## 구조

```
| 세그먼트1 | 세그먼트2 | ... |
```

세그먼트 내:
```
| 데이터 | inode | imap |
```

## inode map

inode 위치 추적:
- inode 번호 → 디스크 위치
- 체크포인트 영역에 저장

## 가비지 컬렉션

구버전 블록 정리:
- 세그먼트 정리
- 유효 블록 복사

## 장단점

장점:
- 쓰기 최적화
- 장애 복구 쉬움

단점:
- 읽기 성능
- GC 오버헤드
