> **OSTEP** Chapter 19 요약

## I/O 아키텍처

```
CPU ←→ 메모리 버스 ←→ 메모리
         ↓
      I/O 버스
         ↓
    장치들 (디스크, 네트워크, ...)
```

## 장치 인터페이스

- 상태 레지스터
- 명령 레지스터
- 데이터 레지스터

## 폴링 vs 인터럽트

**폴링**: 계속 확인
- CPU 낭비

**인터럽트**: 완료 시 알림
- 효율적
- 오버헤드 있음

## DMA

Direct Memory Access:
- CPU 개입 없이 전송
- 대용량 데이터

## 디바이스 드라이버

OS와 하드웨어 사이:
- 추상화 제공
- 하드웨어 종속
