> **CSAPP** Chapter 8 요약

## 핵심 주장

**예외적 제어 흐름이 시스템을 이해하는 핵심이다.** 인터럽트, 시그널, 프로세스가 어떻게 작동하는지 알아야 한다.

## 예외 (Exceptions)

### 예외란?

제어 흐름의 갑작스러운 변화:
- 프로세서 상태 변화에 대응
- 예외 핸들러로 전환
- 운영체제가 처리

### 예외 종류

![예외 종류](/images/blog/csapp/diagrams/ch08-exception-types.svg)

### 시스템 콜

트랩의 예:
- `read`, `write`, `open`
- 사용자 → 커널 전환
- 번호로 식별

## 프로세스

### 프로세스란?

실행 중인 프로그램:
- 독립적인 논리 제어 흐름
- 사적인 주소 공간

### 프로세스 생성: fork()

```c
pid_t pid = fork();

if (pid == 0) {
    // 자식 프로세스
} else {
    // 부모 프로세스, pid = 자식 PID
}
```

특성:
- 한 번 호출, 두 번 반환
- 부모 복제
- 독립적 실행

### 프로세스 종료: exit()

```c
exit(0);  // 정상 종료
exit(1);  // 오류 종료
```

### 자식 대기: wait()

```c
int status;
pid_t child = wait(&status);

if (WIFEXITED(status)) {
    int exit_code = WEXITSTATUS(status);
}
```

### 프로그램 실행: execve()

```c
execve("/bin/ls", argv, envp);
// 반환하면 오류
```

현재 프로세스를 새 프로그램으로 대체

## 컨텍스트 스위칭

### 컨텍스트

프로세스 상태:
- 레지스터 값
- PC
- 스택
- 커널 데이터 구조

### 스케줄러

커널의 결정:
- 언제 스위칭?
- 누구에게?
- 타임슬라이스

### 스위칭 과정

1. 현재 컨텍스트 저장
2. 다음 프로세스 선택
3. 새 컨텍스트 복원
4. 새 프로세스 실행

## 시그널

### 시그널이란?

프로세스에 보내는 비동기 알림:
- 커널 → 프로세스
- 프로세스 → 프로세스

### 주요 시그널

| 번호 | 이름 | 기본 동작 |
|-----|------|----------|
| 2 | SIGINT | 종료 (Ctrl+C) |
| 9 | SIGKILL | 종료 (강제) |
| 11 | SIGSEGV | 종료+코어덤프 |
| 14 | SIGALRM | 종료 |
| 17 | SIGCHLD | 무시 |

### 시그널 발송

```c
kill(pid, SIGKILL);  // 특정 프로세스에
raise(SIGINT);       // 자기 자신에
```

### 시그널 수신

**펜딩**: 보내졌지만 처리 안 됨
**블록**: 일시적으로 무시

### 시그널 핸들러

```c
void handler(int sig) {
    // 시그널 처리
}

signal(SIGINT, handler);
// 또는
sigaction(SIGINT, &action, NULL);
```

### 안전한 시그널 핸들러

규칙:
- 전역 변수 최소화
- `volatile sig_atomic_t` 사용
- 재진입 안전 함수만 호출
- `errno` 보존

## 비지역 점프

### setjmp/longjmp

```c
jmp_buf buf;

if (setjmp(buf) == 0) {
    // 직접 호출
} else {
    // longjmp에서 돌아옴
}

// 어디선가
longjmp(buf, 1);  // setjmp로 점프
```

사용 사례:
- 에러 처리
- 코루틴

## 프로세스 그룹과 세션

### 프로세스 그룹

```c
setpgid(pid, pgid);
pid_t pgrp = getpgrp();
```

### 포그라운드/백그라운드

```bash
./prog &    # 백그라운드
fg %1       # 포그라운드로
```

## 시사점

1. **예외 종류**: 인터럽트, 트랩, 폴트, 어보트
2. **프로세스**: fork, exec, wait
3. **시그널**: 비동기 이벤트 처리
4. **컨텍스트 스위칭**: 멀티태스킹 기반
