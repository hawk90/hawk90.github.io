> **CSAPP** Chapter 1 요약

## 핵심 주장

**프로그래머는 시스템을 이해해야 한다.** hello.c가 실행되기까지 무슨 일이 일어나는지 알아야 더 나은 코드를 쓸 수 있다.

## Hello World의 여정

### 소스 코드

```c
#include <stdio.h>

int main() {
    printf("hello, world\n");
    return 0;
}
```

이 단순한 프로그램이 실행되기까지 복잡한 시스템이 작동한다.

### 정보 = 비트 + 컨텍스트

모든 것은 비트:
- 소스 파일: ASCII 텍스트
- 실행 파일: 이진 데이터
- **해석이 의미를 결정**

## 컴파일 시스템

### 4단계 과정

```
hello.c → 전처리기 → hello.i → 컴파일러 → hello.s → 어셈블러 → hello.o → 링커 → hello
```

### 1. 전처리 (Preprocessing)

- `#include` 처리
- 매크로 확장
- 조건부 컴파일
- 출력: 확장된 C 소스 (.i)

### 2. 컴파일 (Compilation)

- C → 어셈블리
- 최적화 수행
- 출력: 어셈블리 코드 (.s)

### 3. 어셈블리 (Assembly)

- 어셈블리 → 기계어
- 재배치 가능 오브젝트
- 출력: 오브젝트 파일 (.o)

### 4. 링킹 (Linking)

- 오브젝트 파일들 결합
- 라이브러리 연결
- 출력: 실행 파일

## 하드웨어 구성

### 버스 (Buses)

- 워드 단위 데이터 전송
- 시스템의 혈관

### I/O 장치

- 디스크, 키보드, 디스플레이
- 컨트롤러로 버스와 연결

### 메인 메모리

- DRAM
- 프로그램과 데이터 저장
- 바이트 배열

### 프로세서 (CPU)

핵심 구성:
- **PC (Program Counter)**: 다음 명령어 주소
- **레지스터 파일**: 워드 크기 저장소
- **ALU**: 연산 수행

명령어 종류:
- Load: 메모리 → 레지스터
- Store: 레지스터 → 메모리
- Operate: ALU 연산
- Jump: PC 변경

## 프로그램 실행

### hello 실행 과정

1. 쉘에서 `./hello` 입력
2. 쉘이 로더 호출
3. 디스크 → 메모리 복사
4. CPU가 main() 실행
5. 출력 → 디스플레이

### 캐시의 중요성

메모리 계층:
- 레지스터: 가장 빠름
- L1/L2/L3 캐시
- 메인 메모리
- 디스크: 가장 느림

**지역성 활용이 성능의 핵심**

## 운영체제의 역할

### 추상화 제공

OS가 숨기는 것:
- **프로세스**: CPU + 메모리 추상화
- **가상 메모리**: 메모리 추상화
- **파일**: I/O 추상화

### 프로세스

프로세스란:
- 실행 중인 프로그램
- 독립적 실행 환상
- 컨텍스트 스위칭

### 가상 메모리

가상 주소 공간:
- 각 프로세스만의 메모리
- 코드, 데이터, 힙, 스택
- 물리 메모리와 분리

### 파일

파일 = 바이트 시퀀스:
- 모든 I/O를 추상화
- 디스크, 키보드, 네트워크

## 네트워크

### 네트워크도 I/O

네트워크 통신:
- 다른 시스템의 파일처럼
- 소켓으로 추상화
- 원격 실행 가능

## 시사점

1. **컴파일 이해**: 더 나은 디버깅
2. **메모리 계층**: 캐시 활용
3. **OS 추상화**: 프로세스, 가상 메모리, 파일
4. **하드웨어 이해**: 성능 최적화
