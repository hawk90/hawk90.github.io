> **CSAPP** Chapter 3 요약

## 핵심 주장

**어셈블리를 이해하면 더 나은 프로그래머가 된다.** 컴파일러가 만드는 코드를 읽고 이해할 수 있어야 한다.

## x86-64 역사

### 진화

- 8086 (16비트) → IA32 (32비트) → x86-64 (64비트)
- 하위 호환성 유지
- CISC 아키텍처

## 프로그램 인코딩

### 컴파일과 어셈블

```bash
gcc -Og -S hello.c   # 어셈블리 생성
gcc -Og -c hello.c   # 오브젝트 생성
objdump -d hello.o   # 디스어셈블
```

`-Og`: 디버깅용 최적화 (읽기 쉬움)

## 데이터 형식

### x86-64 데이터 크기

| C 타입 | Intel 명칭 | 크기 | 접미사 |
|--------|-----------|------|--------|
| char | Byte | 1 | b |
| short | Word | 2 | w |
| int | Double word | 4 | l |
| long | Quad word | 8 | q |
| pointer | Quad word | 8 | q |

## 레지스터

### 16개 범용 레지스터

```
64비트: %rax, %rbx, %rcx, %rdx, %rsi, %rdi, %rbp, %rsp, %r8-%r15
32비트: %eax, %ebx, %ecx, %edx, %esi, %edi, %ebp, %esp, %r8d-%r15d
```

### 특수 용도

- `%rsp`: 스택 포인터
- `%rbp`: 베이스 포인터 (선택적)
- `%rip`: 명령어 포인터

### 호출 규약

인자 전달 순서:
1. `%rdi`, 2. `%rsi`, 3. `%rdx`, 4. `%rcx`, 5. `%r8`, 6. `%r9`
7번째부터 스택

반환값: `%rax`

## 피연산자 형식

### 세 가지 종류

1. **즉시값 (Immediate)**: `$123`
2. **레지스터**: `%rax`
3. **메모리 참조**: `(%rax)`, `8(%rbp)`

### 메모리 주소 계산

```
Imm(rb, ri, s) = Imm + rb + ri * s

예: 8(%rbp, %rcx, 4) = %rbp + %rcx*4 + 8
```

## 데이터 이동

### MOV 명령어

```asm
movq $0, %rax    # 즉시값 → 레지스터
movq %rax, %rbx  # 레지스터 → 레지스터
movq (%rax), %rbx # 메모리 → 레지스터
movq %rax, (%rbx) # 레지스터 → 메모리
```

**메모리 ↔ 메모리 직접 불가**

### 확장

- `movzbq`: 0 확장 (unsigned)
- `movsbq`: 부호 확장 (signed)

## 산술 연산

### 단항 연산

```asm
incq %rax    # ++
decq %rax    # --
negq %rax    # 부호 반전
notq %rax    # 비트 반전
```

### 이항 연산

```asm
addq %rbx, %rax   # rax += rbx
subq %rbx, %rax   # rax -= rbx
imulq %rbx, %rax  # rax *= rbx
```

### 시프트

```asm
salq $4, %rax   # rax <<= 4
sarq $4, %rax   # rax >>= 4 (산술)
shrq $4, %rax   # rax >>= 4 (논리)
```

## 조건 코드

### 플래그 레지스터

- **CF (Carry)**: unsigned 오버플로우
- **ZF (Zero)**: 결과 0
- **SF (Sign)**: 결과 음수
- **OF (Overflow)**: signed 오버플로우

### 조건부 명령어

```asm
cmpq %rbx, %rax   # rax - rbx (결과 버림, 플래그만)
testq %rax, %rax  # rax & rax (0 테스트)

setl %al          # if (SF != OF) al = 1
sete %al          # if (ZF) al = 1
```

## 제어 흐름

### 점프

```asm
jmp .L1      # 무조건 점프
je .L1       # if (ZF) 점프
jl .L1       # if (SF != OF) 점프
```

### 조건부 이동

```asm
cmovl %rbx, %rax  # if (condition) rax = rbx
```

분기 예측 실패보다 효율적

## 프로시저

### 스택 프레임

```
높은 주소
+------------------+
| 인자 n           |
| ...              |
| 인자 7           |
+------------------+
| 반환 주소        |
+------------------+
| 저장된 %rbp      |  ← %rbp
+------------------+
| 지역 변수        |
+------------------+
| 임시 저장        |  ← %rsp
+------------------+
낮은 주소
```

### 호출과 반환

```asm
call func    # push %rip; jmp func
ret          # pop %rip
```

## 배열과 구조체

### 배열

```c
int A[5];     # A[i]의 주소 = A + 4*i
```

```asm
movl (%rdi, %rsi, 4), %eax  # A[i]
```

### 구조체

- 순서대로 메모리 배치
- 정렬 (alignment) 주의
- 패딩 추가됨
