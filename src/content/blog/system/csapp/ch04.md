## 핵심 주장

**프로세서 설계를 이해하면 성능을 이해한다.** 파이프라인과 해저드가 왜 중요한지 알아야 한다.

## Y86-64 명령어 세트

### 왜 Y86-64?

- x86-64의 단순화 버전
- 교육용 목적
- 핵심 개념만 포함

### 명령어 구조

```
| icode | ifun | rA | rB | [valC] |
```

- icode: 명령어 종류
- ifun: 기능 코드
- rA, rB: 레지스터
- valC: 상수 값

### 주요 명령어

```asm
halt           # 정지
nop            # 아무것도 안 함
rrmovq rA, rB  # 레지스터 → 레지스터
irmovq V, rB   # 즉시값 → 레지스터
rmmovq rA, D(rB)  # 레지스터 → 메모리
mrmovq D(rB), rA  # 메모리 → 레지스터
OPq rA, rB     # ALU 연산
jXX Dest       # 조건부 점프
call Dest      # 함수 호출
ret            # 함수 반환
pushq rA       # 스택 push
popq rA        # 스택 pop
```

## 순차 실행 (SEQ)

### 5단계

1. **Fetch**: 명령어 읽기
2. **Decode**: 레지스터 읽기
3. **Execute**: ALU 연산
4. **Memory**: 메모리 읽기/쓰기
5. **Write back**: 레지스터 쓰기
6. **PC update**: 다음 명령어

### 하드웨어 구조

각 단계에 해당하는 하드웨어:
- **Fetch**: PC + 명령어 메모리
- **Decode**: 레지스터 파일 (읽기)
- **Execute**: ALU
- **Memory**: 데이터 메모리
- **Write back**: 레지스터 파일 (쓰기)

## 파이프라인 (PIPE)

### 왜 파이프라인?

순차 실행의 문제:
- 각 단계가 끝날 때까지 대기
- 하드웨어 유휴 상태
- **처리량 낭비**

파이프라인의 이점:
- 여러 명령어 동시 진행
- 처리량 증가
- 각 사이클에 완료 가능

### 파이프라인 구조

```
시간 →
명령어1: F | D | E | M | W |
명령어2:   | F | D | E | M | W |
명령어3:     | F | D | E | M | W |
```

이상적: 5배 처리량 증가

## 해저드 (Hazards)

### 데이터 해저드

이전 명령어 결과가 필요:

```asm
addq %rax, %rbx    # rbx 변경
subq %rbx, %rcx    # rbx 사용 (아직 안 썼음!)
```

해결책:
- **Stalling**: 기다리기
- **Forwarding**: 결과 직접 전달
- **Load-use 해저드**: 메모리 읽기 후 사용

### 제어 해저드

분기 결과를 모름:

```asm
jne .L1           # 점프할지 모름
...               # 잘못 실행 가능
```

해결책:
- **Branch prediction**: 예측
- **Speculation**: 추측 실행
- **Misprediction penalty**: 잘못되면 취소

### 해저드 정리

| 해저드 종류 | 원인 | 해결책 |
|------------|------|--------|
| 데이터 | RAW 의존성 | 포워딩, 스톨링 |
| 제어 | 분기 | 예측, 추측 실행 |
| 구조적 | 리소스 경합 | 복제, 스톨링 |

## 성능 분석

### CPI (Cycles Per Instruction)

$$
CPI = 1 + \text{penalties}
$$

이상적 CPI = 1
실제로는 해저드로 인해 증가

### 처리량

$$
\text{Throughput} = \frac{1}{CPI \times \text{Clock period}}
$$

## 예외 처리

### 예외 종류

- **Halt**: 정상 종료
- **Invalid address**: 잘못된 주소
- **Invalid instruction**: 잘못된 명령어

### 파이프라인에서의 예외

예외 발생 시:
- 파이프라인 플러시
- 예외 핸들러로 점프
- 상태 롤백

## 시사점

1. **파이프라인**: 처리량 증가의 핵심
2. **해저드**: 의존성 문제
3. **포워딩**: 데이터 해저드 해결
4. **예측**: 제어 해저드 완화
