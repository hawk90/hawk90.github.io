> **CSAPP** Chapter 10 요약

## 핵심 주장

**모든 I/O는 파일이다.** 유닉스의 단순하고 일관된 I/O 모델을 이해해야 한다.

## 유닉스 I/O

### 파일 = 바이트 시퀀스

모든 I/O 장치:
- 일반 파일
- 디렉토리
- 소켓
- 장치
- **동일한 인터페이스**

### 기본 연산

```c
int fd = open(path, flags, mode);
ssize_t n = read(fd, buf, count);
ssize_t n = write(fd, buf, count);
close(fd);
```

## 파일

### 파일 종류

- **일반 파일**: 데이터
- **디렉토리**: 파일 목록
- **소켓**: 네트워크 통신
- **파이프**: 프로세스 간 통신
- **장치**: 특수 파일

### 파일 디스크립터

정수 핸들:
- 0: stdin
- 1: stdout
- 2: stderr
- 3+: 사용자 할당

## 열기와 닫기

### open()

```c
int fd = open("file.txt", O_RDONLY);
int fd = open("file.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
```

플래그:
- `O_RDONLY`: 읽기 전용
- `O_WRONLY`: 쓰기 전용
- `O_RDWR`: 읽기/쓰기
- `O_CREAT`: 생성
- `O_TRUNC`: 잘라내기
- `O_APPEND`: 추가

### close()

```c
close(fd);
```

항상 닫기:
- 리소스 누수 방지
- 디스크립터 재사용

## 읽기와 쓰기

### read()

```c
ssize_t n = read(fd, buf, count);
```

반환값:
- n > 0: 읽은 바이트 수
- n = 0: EOF
- n = -1: 오류

### write()

```c
ssize_t n = write(fd, buf, count);
```

반환값:
- n > 0: 쓴 바이트 수
- n = -1: 오류

### 짧은 카운트 (Short Counts)

요청보다 적게 읽거나 쓰는 경우:
- EOF 도달
- 시그널 인터럽트
- 네트워크 소켓
- 파이프

**항상 반환값 확인!**

## 견고한 I/O

### RIO 패키지

버퍼링 I/O:
```c
ssize_t rio_readn(int fd, void *buf, size_t n);
ssize_t rio_writen(int fd, void *buf, size_t n);
```

정확히 n 바이트 읽거나 쓰기

### 버퍼링된 읽기

```c
void rio_readinitb(rio_t *rp, int fd);
ssize_t rio_readlineb(rio_t *rp, void *buf, size_t maxlen);
ssize_t rio_readnb(rio_t *rp, void *buf, size_t n);
```

효율적인 라인 읽기

## 파일 메타데이터

### stat()

```c
struct stat statbuf;
stat("file.txt", &statbuf);

statbuf.st_size;   // 파일 크기
statbuf.st_mode;   // 권한
statbuf.st_mtime;  // 수정 시간
```

### 권한

```
rwxrwxrwx
│││││││││
│││││││└─ 기타 실행
││││││└── 기타 쓰기
│││││└─── 기타 읽기
││││└──── 그룹 실행
│││└───── 그룹 쓰기
││└────── 그룹 읽기
│└─────── 소유자 실행
└──────── 소유자 쓰기
         소유자 읽기
```

## 디렉토리

### opendir/readdir

```c
DIR *dp = opendir(".");
struct dirent *de;
while ((de = readdir(dp)) != NULL) {
    printf("%s\n", de->d_name);
}
closedir(dp);
```

## 파일 공유

### 커널 자료구조

```
프로세스 A 테이블    열린 파일 테이블     v-node 테이블
+--------+          +--------+          +--------+
| fd 0 --|--------->| refcnt |--------->| 파일 정보|
| fd 1 --|--+       | pos    |     +--->| 크기    |
+--------+  |       +--------+     |    +--------+
            |                      |
            |       +--------+     |
            +------>| refcnt |-----+
                    | pos    |
                    +--------+
```

### fork()와 파일

자식이 상속:
- 파일 디스크립터 복제
- 열린 파일 테이블 공유
- **위치(pos) 공유**

### dup2()

```c
dup2(oldfd, newfd);
```

newfd를 oldfd와 같은 파일로

### 리디렉션

```c
dup2(fd, STDOUT_FILENO);  // stdout → fd
```

## 표준 I/O

### stdio vs Unix I/O

| 특성 | stdio | Unix I/O |
|-----|-------|----------|
| 버퍼링 | 있음 | 없음 |
| 이식성 | 높음 | 낮음 |
| 성능 | 일반적으로 좋음 | 제어 가능 |

### stdio 사용

```c
FILE *fp = fopen("file.txt", "r");
char *line = fgets(buf, size, fp);
fprintf(fp, "Hello %s\n", name);
fclose(fp);
```

### 주의사항

소켓에서 stdio:
- 버퍼링 문제
- `fflush()` 필요
- 네트워크에서는 조심

3. **RIO 사용**: 견고한 I/O
