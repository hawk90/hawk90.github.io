> **CSAPP** Chapter 5 요약

## 핵심 주장

**최적화는 시스템 이해에서 시작한다.** 컴파일러의 한계를 알고, 프로그래머가 도와야 한다.

## 컴파일러 최적화

### 최적화 수준

```bash
gcc -O0  # 최적화 없음
gcc -O1  # 기본 최적화
gcc -O2  # 표준 최적화
gcc -O3  # 적극적 최적화
gcc -Os  # 크기 최적화
```

### 컴파일러의 한계

컴파일러가 못하는 것:
- **메모리 앨리어싱**: 같은 메모리 가리킬 수 있음
- **함수 부작용**: 함수가 뭘 할지 모름
- **undefined behavior**: 정의되지 않은 동작

## 성능 측정

### CPE (Cycles Per Element)

루프 성능 지표:
- 원소당 사이클 수
- 낮을수록 좋음

```c
void combine(vec_ptr v, data_t *dest) {
    for (long i = 0; i < vec_length(v); i++) {
        *dest = *dest OP data[i];
    }
}
```

## 최적화 기법

### 1. 코드 이동 (Code Motion)

루프 밖으로 이동:

```c
// 나쁨
for (i = 0; i < strlen(s); i++) { ... }

// 좋음
int len = strlen(s);
for (i = 0; i < len; i++) { ... }
```

### 2. 불필요한 계산 제거

공통 부분식:

```c
// 나쁨
x = a*b + c;
y = a*b + d;

// 좋음
t = a*b;
x = t + c;
y = t + d;
```

### 3. 메모리 앨리어싱 제거

로컬 변수 사용:

```c
// 나쁨 (dest가 data와 겹칠 수 있음)
for (i = 0; i < n; i++)
    *dest = *dest + data[i];

// 좋음
data_t acc = *dest;
for (i = 0; i < n; i++)
    acc = acc + data[i];
*dest = acc;
```

### 4. 루프 언롤링

```c
// 기본
for (i = 0; i < n; i++)
    sum += a[i];

// 2x1 언롤링
for (i = 0; i < n-1; i += 2) {
    sum += a[i];
    sum += a[i+1];
}
```

### 5. 병렬 누적 (Multiple Accumulators)

```c
// 2x2 언롤링 + 2개 누적기
for (i = 0; i < n-1; i += 2) {
    sum0 += a[i];
    sum1 += a[i+1];
}
sum = sum0 + sum1;
```

### 6. 재결합 변환

```c
// 순차적 의존
sum = (sum + a[i]) + a[i+1];

// 병렬화 가능
sum = sum + (a[i] + a[i+1]);
```

## 프로세서 최적화

### 명령어 수준 병렬성 (ILP)

현대 프로세서:
- 수퍼스칼라: 여러 명령어 동시 발행
- 비순차 실행: 순서 바꿔 실행
- 분기 예측: 미리 실행

### 중요 지표

- **지연 시간 (Latency)**: 연산 완료 시간
- **발행 시간 (Issue time)**: 발행 간격
- **용량 (Capacity)**: 동시 수행 수

| 연산 | 지연 시간 | 발행 시간 |
|------|----------|-----------|
| int + | 1 | 1 |
| int * | 3 | 1 |
| float + | 3 | 1 |
| float * | 5 | 1 |
| float / | 10-20 | 10-20 |

### 병목 분석

두 가지 한계:
1. **지연 시간 한계**: 의존성 체인
2. **처리량 한계**: 기능 유닛 수

## 메모리 성능

### 로드와 스토어

메모리 연산 비용:
- L1 캐시 히트: ~4 사이클
- L2 캐시 히트: ~10 사이클
- L3 캐시 히트: ~40 사이클
- 메인 메모리: ~100+ 사이클

### 쓰기/읽기 의존성

```c
// 의존성 있음 (같은 주소)
a[0] = 1;
x = a[0];

// 의존성 없음 (다른 주소)
a[0] = 1;
x = a[1];
```

## 프로파일링

### gprof 사용

```bash
gcc -pg -o prog prog.c
./prog
gprof prog gmon.out
```

### Amdahl의 법칙

전체 성능 향상:
$$
S = \frac{1}{(1-\alpha) + \alpha/k}
$$

- α: 최적화 부분 비율
- k: 그 부분의 속도 향상

**핵심**: 병목 먼저 최적화
