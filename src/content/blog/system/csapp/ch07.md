> **CSAPP** Chapter 7 요약

## 핵심 주장

**링킹을 이해하면 빌드 시스템을 이해한다.** 심볼 해석, 재배치, 라이브러리가 어떻게 작동하는지 알아야 한다.

## 링커의 역할

### 왜 링킹?

분리 컴파일의 장점:
- 모듈화
- 재사용
- 빌드 시간 단축

### 링커가 하는 일

1. **심볼 해석**: 심볼 참조 → 정의 연결
2. **재배치**: 주소 결정, 참조 수정

## 오브젝트 파일

### 세 종류

1. **재배치 가능 (.o)**: 컴파일러 출력
2. **실행 가능**: 링커 출력
3. **공유 오브젝트 (.so)**: 동적 라이브러리

### ELF 형식

```
+------------------+
| ELF 헤더         |
+------------------+
| 프로그램 헤더    |
+------------------+
| .text (코드)     |
+------------------+
| .rodata (읽기 전용) |
+------------------+
| .data (초기화된) |
+------------------+
| .bss (미초기화)  |
+------------------+
| .symtab (심볼)   |
+------------------+
| .rel.text (재배치) |
+------------------+
| 섹션 헤더        |
+------------------+
```

## 심볼과 심볼 테이블

### 심볼 종류

1. **전역 심볼**: 외부 참조 가능
2. **외부 심볼**: 다른 모듈에서 정의
3. **지역 심볼**: 파일 내부용 (static)

### 심볼 해석

**강한 심볼** vs **약한 심볼**:
- 함수, 초기화된 변수: 강함
- 미초기화 변수: 약함

해석 규칙:
1. 강한 심볼 중복 → 오류
2. 강한 + 약한 → 강한 선택
3. 약한 + 약한 → 아무거나 (위험!)

## 정적 라이브러리

### 아카이브 (.a)

여러 .o 파일 묶음:
```bash
ar rcs libmath.a add.o sub.o mul.o div.o
```

### 링킹 순서

**순서가 중요**:
```bash
gcc -o prog main.o libfoo.a libbar.a
```

의존하는 라이브러리가 뒤에 와야

## 재배치

### 재배치 엔트리

```
typedef struct {
    long offset;   // 수정할 위치
    long type;     // 재배치 타입
    long symbol;   // 심볼 인덱스
    long addend;   // 상수
} Elf64_Rela;
```

### 재배치 타입

- **R_X86_64_PC32**: PC 상대 주소
- **R_X86_64_32**: 절대 주소

### 재배치 공식

PC 상대:
```
ref = ADDR(s) + addend - refaddr
```

절대:
```
ref = ADDR(s) + addend
```

## 실행 파일 로딩

### 메모리 레이아웃

```
0x400000: 코드 (.text)
         읽기 전용 데이터 (.rodata)
         데이터 (.data)
         BSS (.bss)
         힙 (위로 성장)
         ...
         스택 (아래로 성장)
0x7fff...: 커널 영역
```

### 로더의 역할

1. 실행 파일 읽기
2. 메모리에 세그먼트 복사
3. 엔트리 포인트로 점프

## 동적 링킹

### 공유 라이브러리

장점:
- 메모리 절약 (한 복사본)
- 업데이트 용이
- 런타임에 로드 가능

```bash
gcc -shared -fPIC -o libfoo.so foo.c
gcc -o prog main.c -L. -lfoo
```

### PIC (Position-Independent Code)

GOT (Global Offset Table):
- 간접 참조
- 로드 시 주소 설정

PLT (Procedure Linkage Table):
- 함수 호출 지연 바인딩
- 첫 호출 시만 해석

### 지연 바인딩

첫 호출:
1. PLT 엔트리로 점프
2. 동적 링커 호출
3. 실제 주소로 GOT 수정
4. 함수 실행

이후 호출:
1. PLT → GOT → 바로 함수

## 라이브러리 인터포지셔닝

### 개념

### 사용 예
