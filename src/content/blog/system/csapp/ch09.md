> **CSAPP** Chapter 9 요약

## 핵심 주장

**가상 메모리는 강력한 추상화다.** 메모리 관리, 보호, 공유를 가능하게 한다.

## 가상 메모리 개념

### 왜 가상 메모리?

가상 메모리의 역할:
- **효율적 메모리 사용**: 필요한 것만 로드
- **메모리 관리 단순화**: 연속된 주소 공간
- **프로세스 보호**: 격리된 주소 공간

### 주소 종류

- **가상 주소 (VA)**: 프로그램이 보는 주소
- **물리 주소 (PA)**: 실제 메모리 주소

## 주소 변환

### 페이지 테이블

```
가상 주소: [VPN | VPO]
           ↓ 페이지 테이블
물리 주소: [PPN | PPO]
```

- VPN: 가상 페이지 번호
- VPO = PPO: 페이지 오프셋 (같음)
- PPN: 물리 페이지 번호

### 페이지 테이블 엔트리 (PTE)

```
| 유효 비트 | 권한 | PPN |
```

유효 비트:
- 1: 메모리에 있음
- 0: 디스크에 있음 (페이지 폴트)

### 페이지 히트와 폴트

**페이지 히트**:
- PTE 유효
- PPN으로 변환
- 메모리 접근

**페이지 폴트**:
- PTE 무효
- 디스크에서 로드
- 페이지 테이블 업데이트
- 재시도

## TLB (Translation Lookaside Buffer)

### 왜 TLB?

페이지 테이블 문제:
- 메모리에 있음
- 매번 접근 = 2배 메모리 접근
- **느림**

### TLB 캐시

최근 변환 캐싱:
- 작은 크기 (64-1024 엔트리)
- 완전 연관
- 히트 시 1 사이클

### TLB 구조

```
가상 주소: [TLBT | TLBI | VPO]
                    ↓
TLB 검색: 태그 비교, 인덱스 선택
                    ↓
히트: PPN 반환
미스: 페이지 테이블 접근
```

## 멀티레벨 페이지 테이블

### 문제

단일 페이지 테이블:
- 4KB 페이지, 48비트 주소
- 2^36 PTE 필요
- **너무 큼**

### 해결

멀티레벨:
- 계층적 테이블
- 필요한 부분만 할당
- 메모리 절약

### 4레벨 페이지 테이블 (x86-64)

```
CR3 → PML4 → PDPT → PD → PT → 물리 페이지
```

각 레벨 9비트 인덱스

## Core i7 메모리 시스템

### 주소 변환 경로

```
가상 주소
    ↓
TLB (L1 TLB 히트?)
    ↓ 미스
페이지 테이블 워크
    ↓
물리 주소
    ↓
L1 캐시 → L2 → L3 → 메모리
```

### TLB 계층

- L1 ITLB: 128 엔트리
- L1 DTLB: 64 엔트리
- L2 STLB: 1536 엔트리

## 메모리 매핑

### mmap

```c
void *mmap(void *addr, size_t length, int prot,
           int flags, int fd, off_t offset);
```

사용:
- 파일 I/O
- 프로세스 간 공유
- 익명 메모리

### Copy-on-Write

공유된 페이지:
- 초기: 읽기 전용으로 공유
- 쓰기 시: 복사 후 수정
- fork() 최적화

## 동적 메모리 할당

### 힙

`brk`/`sbrk`로 확장:
- 프로세스 데이터 영역 위
- 위로 성장

### malloc/free

```c
void *malloc(size_t size);
void free(void *ptr);
```

### 할당기 구현

**가용 블록 리스트**:
- 암묵적 가용 리스트
- 명시적 가용 리스트
- 분리 가용 리스트

**배치 정책**:
- First fit
- Next fit
- Best fit

**단편화**:
- 내부 단편화: 블록 내 낭비
- 외부 단편화: 블록 간 낭비

## 가비지 컬렉션

### 개념

자동 메모리 관리:
- 도달 불가능 메모리 회수
- Mark-and-Sweep
- 참조 카운팅

### Mark-and-Sweep

1. **Mark**: 루트에서 도달 가능 마킹
2. **Sweep**: 마킹 안 된 것 회수

## 메모리 관련 버그

### 흔한 버그

1. **댕글링 포인터**: 해제된 메모리 참조
2. **이중 해제**: free 두 번
3. **메모리 누수**: 해제 안 함
4. **버퍼 오버플로우**: 경계 초과

## 시사점

1. **가상 메모리**: 추상화와 보호
2. **TLB**: 변환 캐싱
3. **페이지 폴트**: 요구 페이징
4. **동적 할당**: 단편화 주의
