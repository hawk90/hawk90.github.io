## 핵심 주장

**모든 데이터는 비트다.** 정수, 부동소수점, 문자열 모두 비트의 해석일 뿐.

## 정보 저장

### 바이트

- 8비트 = 1바이트
- 주소 지정의 기본 단위
- 16진수로 표기 (0x00 ~ 0xFF)

### 워드 크기

- 32비트 시스템: 4바이트
- 64비트 시스템: 8바이트
- 포인터 크기 결정

### 바이트 순서

두 가지 방식:
- **Big Endian**: MSB 먼저 (IBM, 네트워크)
- **Little Endian**: LSB 먼저 (Intel, ARM)

```
0x01234567 저장:
Big Endian:    01 23 45 67
Little Endian: 67 45 23 01
```

## 정수 표현

### 부호 없는 정수 (Unsigned)

- 모든 비트가 크기
- 범위: 0 ~ 2^w - 1
- w비트로 2^w개 값

### 부호 있는 정수 (Signed)

**2의 보수 (Two's Complement)**:
- MSB가 부호 비트
- 음수: 2^w - x
- 범위: -2^(w-1) ~ 2^(w-1) - 1

### 변환

Signed ↔ Unsigned:
- 비트 패턴은 동일
- 해석만 다름
- **주의**: 암묵적 변환 버그

## 정수 연산

### 덧셈

오버플로우:
- Unsigned: 모듈러 연산 (mod 2^w)
- Signed: 정의되지 않은 동작 (C 표준)

### 곱셈

- 결과가 2w 비트
- 하위 w 비트만 유지
- 오버플로우 가능

### 2의 거듭제곱

최적화:
- 곱셈 → 좌측 시프트
- 나눗셈 → 우측 시프트
- 컴파일러가 자동 적용

## 부동소수점

### IEEE 754 표준

구조:
```
[부호 1비트][지수 k비트][가수 n비트]

Single (32비트): 1 + 8 + 23
Double (64비트): 1 + 11 + 52
```

### 정규화 값 (Normalized)

- 지수 ≠ 0 또는 모두 1
- 가수: 1.xxx... (암묵적 1)
- 바이어스 지수

### 비정규화 값 (Denormalized)

- 지수 = 0
- 가수: 0.xxx... (암묵적 0)
- 0에 가까운 작은 수

### 특수 값

- 지수 = 모두 1
- 가수 = 0: ∞ (Infinity)
- 가수 ≠ 0: NaN (Not a Number)

### 정밀도 문제

```c
0.1 + 0.2 != 0.3  // 근사값이므로
```

부동소수점 비교:
- 직접 == 비교 피하기
- 오차 범위 내 비교

## 비트 연산

### 논리 연산

- AND (&): 둘 다 1이면 1
- OR (|): 하나라도 1이면 1
- XOR (^): 다르면 1
- NOT (~): 반전

### 시프트 연산

- 좌측 시프트 (<<): 2배
- 논리 우측 시프트 (>>): unsigned
- 산술 우측 시프트 (>>): signed (부호 확장)

## 일반적인 버그

### 1. 부호 비교

```c
-1 > 0U  // true! (-1이 큰 unsigned로 변환)
```

### 2. 오버플로우

```c
int x = INT_MAX + 1;  // 정의되지 않은 동작
```

### 3. 정밀도 손실

```c
float f = 16777217;  // 16777216으로 저장됨
```
