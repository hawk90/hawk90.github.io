> **The Design of Everyday Things** Chapter 5 요약

## 핵심 주장

**"인간 오류"는 대부분 시스템 오류다.** 사람이 실수하면 디자인을 탓하라.

## 오류의 두 종류

### 슬립 (Slips)

슬립이란:
- 의도는 맞지만 실행이 잘못됨
- 자동화된 행동의 실수
- **손가락이 미끄러짐**

예시:
- 설탕 대신 소금
- 잘못된 버튼 누름
- 익숙한 길로 가버림

### 미스테이크 (Mistakes)

미스테이크란:
- 의도 자체가 잘못됨
- 잘못된 목표 설정
- **계획의 오류**

예시:
- 문제를 잘못 이해
- 잘못된 해결책 선택
- 상황 오판

## 슬립의 종류

### 캡처 에러 (Capture Error)

익숙한 행동이 가로챔:
- 출근길에 원래 회사로
- 자주 하는 행동이 이김
- **습관의 힘**

### 설명 에러 (Description Error)

유사한 것과 혼동:
- 비슷한 두 버튼
- 같은 색의 다른 아이콘
- **충분히 다르지 않음**

### 모드 에러 (Mode Error)

현재 모드 착각:
- Caps Lock 켜진 줄 모름
- 다른 모드에서 같은 행동
- **상태가 안 보임**

### 메모리 랩스 (Memory-Lapse)

잊어버림:
- 단계를 건너뜀
- 이미 한 것을 반복
- **기억의 한계**

## 미스테이크의 종류

### 규칙 기반 미스테이크

잘못된 규칙 적용:
- 상황에 맞지 않는 규칙
- 잘못 배운 규칙
- **규칙은 맞지만 상황이 다름**

### 지식 기반 미스테이크

지식 부족:
- 모르는 문제
- 잘못된 추론
- **지식의 한계**

### 편향 기반 미스테이크

인지 편향:
- 확증 편향
- 가용성 휴리스틱
- **생각의 함정**

## 근본 원인

### 스위스 치즈 모델

James Reason의 모델:
- 여러 층의 방어
- 각 층에 구멍 (약점)
- 구멍이 일렬로 = 사고
- **단일 원인은 드묾**

### 시스템 사고

사고 분석:
- 개인 탓 X
- 시스템 분석
- 여러 요인의 조합

## 오류 방지 디자인

### 제약 추가

실수를 불가능하게:
- 물리적 제약
- 시퀀스 강제
- 인터락

### 실행 취소 제공

복구 가능하게:
- Ctrl+Z
- 삭제 취소
- **실수해도 괜찮게**

### 확인 요청

중요한 행동 전:
- "정말 삭제하시겠습니까?"
- 하지만 남용 금지
- **정말 중요한 것만**

### 센시블 디폴트

안전한 기본값:
- 전원 켜면 안전 모드
- 기본값이 안전
- **아무것도 안 해도 괜찮게**

### 차별화

비슷한 것을 다르게:
- 다른 색
- 다른 위치
- 다른 크기
- **혼동 방지**

## 오류 감지

### 검사 목록

체크리스트:
- 항공, 의료에서 사용
- 단순하지만 효과적
- **기억에 의존하지 않음**

### 강제 확인

강제로 확인하게:
- 읽어야 진행
- 입력해야 확인
- **무심코 지나가지 않게**

## 시사점

1. **슬립 vs 미스테이크**: 원인이 다름
2. **시스템 사고**: 개인 탓 X
3. **제약 추가**: 실수 불가능하게
4. **복구 제공**: 실수해도 괜찮게
5. **차별화**: 혼동 방지

## 인용

> "When people err, change the system so that type of error will be reduced or eliminated."

> "Don't blame people. Find the real causes."

> "We must design our systems so that they are resilient to error, both human and mechanical."
