## 동시성 기회 찾기

모든 문제가 병렬화에 적합하지 않다. **기회를 식별**하는 것이 첫 단계.

```
질문:
1. 독립적으로 실행 가능한 부분이 있는가?
2. 같은 연산을 여러 데이터에 적용하는가?
3. 파이프라인으로 분해 가능한가?
```

**분해 전략 3가지**:

| 전략 | 분해 대상 | 예 |
|------|----------|-----|
| **작업 분해** | 기능/연산 | 서로 다른 함수 병렬 실행 |
| **데이터 분해** | 데이터 | 배열을 나눠 같은 연산 |
| **파이프라인** | 단계 | 비디오 프레임 처리 |

---

## 작업 분해 (Task Decomposition)

**서로 다른 작업**을 병렬로 실행.

```
예: 이미지 처리 애플리케이션

순차:
[로드] → [리사이즈] → [필터] → [저장]

작업 분해 (여러 이미지):
Image1: [로드] → [리사이즈] → [필터] → [저장]
Image2:         [로드] → [리사이즈] → [필터] → [저장]
Image3:                 [로드] → [리사이즈] → ...

또는 (기능 분해):
Thread1: 히스토그램 계산
Thread2: 엣지 검출
Thread3: 블러 처리
→ 마지막에 결과 합성
```

**적합한 경우**:

```
✓ 독립적인 기능들이 존재
✓ 기능 간 의존성이 낮음
✓ 각 기능이 충분히 큼 (오버헤드 상쇄)
```

**코드 예 (의사코드)**:

```text
// 작업 분해: 독립적인 분석 작업
parallel_invoke(
    [] { compute_histogram(image); },
    [] { detect_edges(image); },
    [] { apply_blur(image); }
);
// 모든 작업 완료 후 결과 합성
merge_results();
```

## 데이터 분해 (Data Decomposition)

**같은 연산을 여러 데이터**에 병렬 적용.

```
예: 배열 요소 제곱

순차:
for (i = 0; i < N; i++)
    A[i] = A[i] * A[i];

데이터 분해 (4 스레드):
Thread 0: A[0..N/4-1]
Thread 1: A[N/4..N/2-1]
Thread 2: A[N/2..3N/4-1]
Thread 3: A[3N/4..N-1]

┌─────────┬─────────┬─────────┬─────────┐
│ Thread0 │ Thread1 │ Thread2 │ Thread3 │
│ [0..24] │[25..49] │[50..74] │[75..99] │
└─────────┴─────────┴─────────┴─────────┘
    각자 독립적으로 제곱 연산
```

**적합한 경우**:

```
✓ 큰 데이터셋
✓ 각 요소 처리가 독립적
✓ 같은 연산 반복 적용
```

**분할 전략**:

| 전략 | 설명 | 적합한 경우 |
|------|------|------------|
| **블록 분할** | 연속된 청크 | 캐시 친화, 균등 작업 |
| **사이클릭 분할** | 라운드 로빈 | 불균등 작업 부하 |
| **블록-사이클릭** | 블록 단위 라운드 로빈 | 중간 |

```
블록 분할 (Block):
T0: [0,1,2,3]  T1: [4,5,6,7]  T2: [8,9,10,11]

사이클릭 분할 (Cyclic):
T0: [0,3,6,9]  T1: [1,4,7,10]  T2: [2,5,8,11]

블록-사이클릭 (Block size=2):
T0: [0,1,6,7]  T1: [2,3,8,9]  T2: [4,5,10,11]
```

## 파이프라인 분해

**단계별로 나눠** 동시에 다른 데이터 처리.

```
예: 비디오 인코딩

순차:
Frame1: [디코드] → [변환] → [인코드] → [쓰기]
Frame2:           [디코드] → [변환] → [인코드] → ...
Frame3:                     [디코드] → ...

파이프라인:
         t=1     t=2     t=3     t=4     t=5
Stage1   F1      F2      F3      F4      F5
(디코드)  ↓       ↓       ↓       ↓       ↓
Stage2           F1      F2      F3      F4
(변환)           ↓       ↓       ↓       ↓
Stage3                   F1      F2      F3
(인코드)                 ↓       ↓       ↓
Stage4                           F1      F2
(쓰기)
```

**파이프라인 특징**:

```
처리량 (Throughput):
- 정상 상태에서 단계당 1 아이템/시간
- 지연은 전체 단계 수만큼

지연 (Latency):
- 하나의 아이템이 완료되는 시간
- 모든 단계의 합

속도 제한:
- 가장 느린 단계에 의해 결정 (병목)
```

**적합한 경우**:

```
✓ 순차적 단계로 나눌 수 있음
✓ 처리할 아이템이 많음
✓ 각 단계가 독립적으로 실행 가능
```

## 분해 전략 선택

**결정 흐름도**:

```
시작
  │
  ▼
┌───────────────────────────┐
│ 데이터가 독립적으로        │
│ 분할 가능한가?             │
└─────────────┬─────────────┘
              │
      Yes ────┼──── No
              │      │
              ▼      ▼
        데이터 분해  │
                    │
        ┌───────────┴───────────┐
        │ 독립적인 작업들이       │
        │ 존재하는가?            │
        └───────────┬───────────┘
                    │
            Yes ────┼──── No
                    │      │
                    ▼      ▼
              작업 분해   │
                        │
              ┌─────────┴─────────┐
              │ 단계별로 나눌 수    │
              │ 있는가?            │
              └─────────┬─────────┘
                        │
                Yes ────┼──── No
                        │      │
                        ▼      ▼
                  파이프라인   병렬화
                              어려움
```

**혼합 접근**:

실제로는 여러 전략을 **조합**하는 경우가 많다.

```
예: 대규모 행렬 곱셈

1. 데이터 분해: 결과 행렬을 블록으로 분할
2. 작업 분해: 각 블록 계산을 작업으로
3. 파이프라인: 데이터 로드 → 계산 → 저장

┌───────────────────────────────────┐
│   [Load]  [Compute]  [Store]      │  파이프라인
│      │        │         │         │
│      ▼        ▼         ▼         │
│   Block     Block     Block       │  데이터 분해
│   [0,0]     [0,1]     [1,0]  ...  │
└───────────────────────────────────┘
```

## 의존성 분석

병렬화의 **핵심 제약**: 의존성.

### 데이터 의존성

```
RAW (Read After Write) - 진정한 의존성:
x = 1;     // S1
y = x + 1; // S2 (S1 후에 실행되어야)

WAR (Write After Read) - 반의존성:
y = x + 1; // S1
x = 2;     // S2 (S1이 x를 읽은 후에)

WAW (Write After Write) - 출력 의존성:
x = 1;     // S1
x = 2;     // S2 (최종 값은 S2의 것)
```

### 제어 의존성

```
if (condition) {
    // 이 블록은 condition에 의존
    do_something();
}
```

### 의존성 그래프

```
예: 순차 코드
S1: a = b + c
S2: d = a * 2
S3: e = c + d
S4: f = a + e

의존성:
S1 → S2 (RAW: a)
S2 → S3 (RAW: d)
S1 → S4 (RAW: a)
S3 → S4 (RAW: e)

그래프:
    S1
   ╱  ╲
  ▼    ▼
 S2    S4
  ╲   ╱
   ▼ ▼
    S3
     ↓
    S4

병렬화 기회:
- S1 후에 S2와 부분적 S4 가능
- 하지만 S3는 S2 후에
```

### 루프 의존성

```
독립 루프 (병렬화 가능):
for (i = 0; i < N; i++)
    A[i] = B[i] + C[i];

의존 루프 (병렬화 어려움):
for (i = 1; i < N; i++)
    A[i] = A[i-1] + B[i];  // A[i]가 A[i-1]에 의존

리덕션 (특별 처리 가능):
for (i = 0; i < N; i++)
    sum += A[i];  // 결합 연산으로 병렬화 가능
```

## 정리

- **작업 분해**: 다른 기능을 병렬 실행
- **데이터 분해**: 같은 연산을 분할된 데이터에 적용
- **파이프라인**: 단계별로 나눠 처리량 증가
- **의존성 분석**: RAW, WAR, WAW, 제어 의존성
- **혼합 접근**: 실제로는 여러 전략 조합

## 핵심 비교

| 분해 전략 | 분해 대상 | 확장성 | 적합한 문제 |
|----------|----------|--------|------------|
| 작업 | 기능 | 작업 수에 제한 | 독립 기능 다수 |
| 데이터 | 데이터 | 데이터 크기에 비례 | 대규모 데이터셋 |
| 파이프라인 | 단계 | 단계 수에 제한 | 스트리밍 처리 |

| 의존성 | 원인 | 해결 |
|--------|------|------|
| RAW | 쓰기 후 읽기 | 순서 유지 필수 |
| WAR | 읽기 후 쓰기 | 복사 또는 순서 |
| WAW | 쓰기 후 쓰기 | 마지막 쓰기 보장 |
