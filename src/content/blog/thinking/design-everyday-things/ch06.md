> **The Design of Everyday Things** Chapter 6 요약

## 핵심 주장

**좋은 디자인은 올바른 문제를 푸는 것이다.** 해결책보다 문제 정의가 먼저다.

## 디자인 사고란

### 정의

디자인 사고:
- 인간 중심 접근법
- 반복적 프로세스
- 창의적 문제 해결
- **올바른 문제 찾기**

### 핵심 원칙

두 가지 질문:
1. 올바른 문제를 풀고 있는가?
2. 올바르게 풀고 있는가?

## 이중 다이아몬드

### 모델

영국 디자인 카운슬의 모델:

```
발견 → 정의 → 개발 → 전달
  ◇         ◇
```

### 첫 번째 다이아몬드: 문제 찾기

1. **발견 (Discover)**: 관찰, 탐구, 확장
2. **정의 (Define)**: 수렴, 진짜 문제 찾기

### 두 번째 다이아몬드: 해결책 찾기

3. **개발 (Develop)**: 아이디어 생성, 확장
4. **전달 (Deliver)**: 수렴, 최종 해결책

## 인간 중심 디자인

### HCD 프로세스

4단계 반복:

1. **관찰 (Observation)**
2. **아이디어 생성 (Ideation)**
3. **프로토타이핑 (Prototyping)**
4. **테스트 (Testing)**

→ 반복

### 관찰

사용자 연구:
- 실제 환경에서 관찰
- 인터뷰
- **사람들이 말하는 것 ≠ 하는 것**

방법:
- 에스노그래피
- 그림자 따라가기
- 맥락적 조사

### 아이디어 생성

브레인스토밍:
- 양 > 질 (처음에는)
- 판단 유보
- 터무니없는 것도 환영
- **나중에 수렴**

### 프로토타이핑

빠른 시각화:
- 저충실도 프로토타입
- 종이, 스케치
- **빠르고 싸게**

목적:
- 아이디어 테스트
- 의사소통
- 실패해도 괜찮음

### 테스트

사용자 피드백:
- 실제 사용자로 테스트
- 관찰 + 인터뷰
- **가정 검증**

## 활동 중심 디자인

### 개념

활동 중심:
- 특정 사용자 아닌 활동에 집중
- 활동 = 여러 과제의 집합
- **더 넓은 관점**

### 예시

"글쓰기" 활동:
- 리서치
- 초안 작성
- 편집
- 출판

→ 하나의 제품이 모든 과제 지원

## 제약과 트레이드오프

### 현실의 제약

디자인 제약:
- 시간
- 예산
- 기술
- 조직

### 트레이드오프

모든 것을 가질 수 없음:
- 기능 vs 단순성
- 유연성 vs 학습 용이성
- 비용 vs 품질

### 의사결정

선택의 기준:
- 목표 우선순위
- 사용자 요구
- 비즈니스 요구

## 복잡성 관리

### 복잡성 vs 혼란

구분:
- **복잡성**: 시스템의 상태 수
- **혼란**: 이해하기 어려움

### 복잡성은 피할 수 없다

세상은 복잡:
- 문제 자체가 복잡
- 단순화에는 한계
- **혼란은 줄일 수 있음**

### 복잡성 관리 방법

구조화:
- 그룹화
- 계층화
- 점진적 공개
- 일관된 패턴

## 표준화

### 표준의 가치

표준화의 장점:
- 학습 전이
- 예측 가능
- 호환성

### 표준화의 한계

단점:
- 혁신 저해
- 잘못된 표준 고착

### 언제 표준화

좋은 경우:
- 안전 관련
- 자주 사용
- 여러 제품 공유

## 시사점

1. **올바른 문제**: 해결책보다 문제
2. **반복적 프로세스**: 한 번에 완성 X
3. **관찰 중심**: 가정하지 말고 관찰
4. **빠른 프로토타입**: 싸게 실패
5. **복잡성 관리**: 혼란은 줄여라
