> **Hackers and Painters** Essay 10 요약

## 핵심 주장

**프로그래밍 언어는 컴퓨터에게 할 일을 말하는 방법이다.** 하지만 동시에 **프로그래머가 생각하는 방식**을 결정한다.

## 프로그래밍이란?

### 기본 개념

프로그램이란:
- 컴퓨터에게 주는 지시 목록
- 단계별 명령
- 조건에 따른 분기

### 왜 여러 언어가 있는가?

언어가 다양한 이유:
- 다른 목적에 다른 도구
- 역사적 발전
- 다른 철학과 접근법

## 언어의 종류

### 저수준 vs 고수준

저수준 언어:
- 기계에 가까움
- 어셈블리, C
- 세밀한 제어
- 더 어렵고 길다

고수준 언어:
- 인간에 가까움
- Python, Ruby, Lisp
- 추상화 많음
- 더 쉽고 짧다

### 컴파일 vs 인터프리트

컴파일 언어:
- 미리 기계어로 번역
- C, C++, Go
- 빠른 실행

인터프리트 언어:
- 실행 시 해석
- Python, JavaScript
- 개발 빠름

## 언어 선택의 중요성

### 생산성 차이

언어에 따른 차이:
- 같은 일을 더 적은 코드로
- 더 빠른 개발
- 더 적은 버그

### 생각하는 방식

언어가 사고에 미치는 영향:
- 표현할 수 있는 것만 생각
- 언어의 한계 = 사고의 한계
- **사피어-워프 가설**의 프로그래밍 버전

## 언어 패러다임

### 명령형 (Imperative)

- 단계별 지시
- "이렇게 해라"
- C, Java

### 함수형 (Functional)

- 함수의 조합
- "이것은 저것이다"
- Lisp, Haskell

### 객체지향 (Object-Oriented)

- 객체들의 상호작용
- 현실 세계 모델링
- Java, Python

## 언어의 힘

### 표현력

강력한 언어:
- 더 적은 코드로 더 많이
- 추상화 수준 높음
- 프로그래머 의도 명확

### 간결함

간결한 코드:
- 읽기 쉬움
- 버그 적음
- 유지보수 쉬움

## Graham의 견해

### 언어는 평등하지 않다

모든 언어가 같지 않음:
- 어떤 언어는 더 강력
- 어떤 언어는 더 표현적
- **선택이 중요**

### Lisp 예찬

Graham이 Lisp를 좋아하는 이유:
- 극도의 유연성
- 매크로 시스템
- 코드 = 데이터

## 비프로그래머에게

### 핵심 메시지

프로그래밍 언어에 대해 알아야 할 것:
1. 언어는 도구
2. 도구에 따라 결과가 달라짐
3. 프로그래머에게 언어 선택은 중요

### 비유

일상의 비유:
- 언어 = 도구
- 망치로 나사 박기 vs 드라이버
- 적합한 도구 선택의 중요성

## 시사점

1. **언어는 도구**: 목적에 맞는 선택
2. **추상화의 힘**: 고수준 = 더 적은 일
3. **사고 방식**: 언어가 생각을 형성
4. **선택의 중요성**: 모든 언어가 같지 않음

## 인용

> "A programming language is for thinking about programs, not for expressing programs you've already thought of."

> "Languages differ not just in what they can express, but in what they make easy to express."
