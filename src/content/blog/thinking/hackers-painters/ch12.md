> **Hackers and Painters** Essay 12 요약

## 핵심 주장

**프로그래밍 언어 선택은 경쟁 우위가 될 수 있다.** Graham은 Viaweb에서 Lisp를 사용해 경쟁사를 이겼다.

## Viaweb 이야기

### 배경

Viaweb (1995-1998):
- 최초의 웹 기반 이커머스 플랫폼
- Yahoo에 5천만 달러에 인수
- **비밀 무기: Lisp**

### 경쟁 환경

당시 경쟁사들:
- 더 많은 자금
- 더 많은 인력
- 하지만 C++, Perl 사용

## Lisp의 힘

### 개발 속도

Lisp의 장점:
- 빠른 프로토타이핑
- 적은 코드량
- 유연한 수정

### 구체적 이점

Viaweb에서의 경험:
- 기능 추가가 빠름
- 버그 수정이 빠름
- **경쟁사가 6개월 걸릴 일을 2주에**

### 매크로의 힘

Lisp 매크로:
- 언어 자체를 확장
- 반복 패턴 추상화
- 도메인 특화 언어(DSL) 생성

## Blub Paradox

### 개념

"Blub"이라는 가상의 언어:
- 어셈블리보다는 강력
- Lisp보다는 약함
- **중간 언어의 프로그래머**

### 위아래를 못 본다

Blub 프로그래머의 시야:
- 아래 (어셈블리): "왜 저걸 쓰지? 이건 더 쉬운데"
- 위 (Lisp): "왜 저런 기능이 필요하지? 난 필요 없는데"

**자기가 모르는 것을 모른다.**

### 적용

현실 세계:
- Java 프로그래머 → 클로저? 왜 필요?
- C 프로그래머 → 가비지 컬렉션? 필요 없어
- **더 강력한 언어를 이해 못함**

## 언어 연속체

### 언어의 힘

언어는 평등하지 않다:
- 어떤 언어는 더 강력
- 어떤 언어는 더 표현적
- 측정 가능한 차이

### 순위

Graham의 (암묵적) 순위:
1. Lisp (최상)
2. Python, Ruby
3. Java, C++
4. C
5. 어셈블리

## 비밀 무기 전략

### 남들이 안 쓰는 것

경쟁 우위:
- 남들이 모르는 기술
- 남들이 "이상하다"고 하는 것
- **평균에서 벗어나기**

### 리스크

이 전략의 위험:
- 채용 어려움
- 학습 곡선
- 생태계 부족

### 보상

그러나:
- 진짜 차별화
- 작은 팀으로 큰 일
- **속도가 곧 생존**

## 교훈

### 언어 선택을 진지하게

스타트업에게:
- 언어 선택은 전략적 결정
- "모두가 쓰는 것"이 최선 아님
- 장점과 단점 따져보라

### 열린 마음

프로그래머에게:
- 새 언어 배우라
- 다른 패러다임 경험하라
- Blub 프로그래머 되지 마라

## 반론에 대한 응답

### "그건 특수한 경우"

비판:
- Viaweb은 특별한 상황
- 대부분은 Java/Python이 충분

Graham:
- 그래도 가능성은 존재
- 차이를 만들 수 있다면 그게 경쟁력

### "팀 채용이 어렵다"

비판:
- Lisp 개발자 구하기 어려움

Graham:
- 좋은 개발자는 뭐든 배움
- 오히려 필터링 효과
- **Lisp 쓰고 싶어하는 사람 = 좋은 개발자**

## 시사점

1. **언어는 경쟁력**: 올바른 선택이 승패를 가름
2. **Blub Paradox**: 자기가 모르는 것을 모른다
3. **평균에서 벗어나라**: 남들과 같으면 경쟁력 없음
4. **열린 마음**: 새로운 것을 배워라
