## Architecture


## FPP — F-Prime Prompt 모델 언어

```text
# MyComponent.fpp
module MyModule {
  
  active component MyComp {
    
    # Input port
    sync input port Run: Svc.Sched
    
    # Output port
    output port Out: MyModule.MyPort
    
    # Command
    sync command DO_THING(
      arg: U32
    )
    
    # Event
    event SOMETHING_HAPPENED(
      value: U32
    ) severity activity high \
      format "value was {}"
    
    # Telemetry
    telemetry MyChannel: U32 update on change
    
    # Parameter
    param MyParam: F32 default 1.0
  }
}
```

XML 시절보다 *훨씬 가독성*. 컴파일 시 *type-safe 코드 생성*.

## Autocoder Output

```cpp
// Autogenerated: MyCompImplCfg.hpp
namespace MyModule {
    class MyCompImpl : public MyCompComponentBase {
    public:
        // Port handlers — user implements
        void Run_handler(...) override;
        
        // Command handlers
        void DO_THING_cmdHandler(U32 arg) override;
        
        // Helper functions
        void log_ACTIVITY_HI_SOMETHING_HAPPENED(U32 value);
        void tlmWrite_MyChannel(U32 val);
    };
}
```

사용자는 *handler만 채움*. Boilerplate 자동.

## Component Implementation

```cpp
// MyCompImpl.cpp
#include "MyCompImpl.hpp"

namespace MyModule {

void MyCompImpl::Run_handler(
    const FwIndexType portNum,
    U32 context
) {
    // Scheduled to run
    U32 value = readSensor();
    
    // Send telemetry
    this->tlmWrite_MyChannel(value);
    
    // Optional event
    if (value > threshold) {
        this->log_ACTIVITY_HI_SOMETHING_HAPPENED(value);
    }
    
    // Output port call
    this->Out_out(0, output_data);
}

void MyCompImpl::DO_THING_cmdHandler(
    const FwOpcodeType opCode,
    const U32 cmdSeq,
    U32 arg
) {
    // Handle command
    process(arg);
    
    // Acknowledge
    this->cmdResponse_out(opCode, cmdSeq, Fw::CmdResponse::OK);
}

} // namespace MyModule
```

C++ 인터페이스 — *strong typed*.

## Topology — Component 연결

```text
# Topology.fpp
module Ref {
  topology MyTopology {
    
    instance comp1: MyComp \
      base id 0x1000 \
      queue size 10 \
      stack size 4096 \
      priority 10
    
    instance comp2: AnotherComp \
      base id 0x2000
    
    connections:
      # comp1.Out → comp2.In
      comp1.Out -> comp2.In
  }
}
```

Static topology — runtime *connection 변경 안 됨*. 검증 쉬움.

## Port Types

```text
sync port:    동기 호출 (function call)
async port:   비동기 (queued, message)
data port:    data transfer
serial port:  generic serialization
```

```text
sync input port Run: Svc.Sched
  → 동기, 호출자가 wait
  
async input port BufferSend: Fw.BufferSend
  → queue·async, returns immediately
```

Designer가 *명시 선택*. cFS에서는 *모두 message*.

## cFS vs F-Prime 비교

| 항목 | cFS | F-Prime |
|---|---|---|
| 언어 | C | C++14 |
| Model | XML·구두 | FPP |
| Comm | Software Bus (msg) | Typed ports (sync·async) |
| Topology | Dynamic (runtime) | Static (compile) |
| Type | Weak (CCSDS msg) | Strong (C++ types) |
| 학습 | 가벼움 | 모던 (C++ 필요) |
| Mission | NASA GSFC (LADEE·MMS) | NASA JPL (Ingenuity) |
| 채택 | 더 큰 base | growing |

둘 다 NASA — *상보적*. Mission 성격에 따라.

## 자주 하는 실수

> ⚠️ FPP 없이 직접 C++

```text
"Component 직접 만들고 싶다"
→ topology 통합 안 됨
→ autocoder 기능 사용 못함
```

→ FPP 활용.

> ⚠️ Static topology에 dynamic logic

```text
runtime component creation 시도
→ F-Prime은 *static* 설계
```

→ topology 재설계.

> ⚠️ Cross-version XML+FPP

```text
Legacy XML + new FPP 혼용 → confusion
```

→ project 시작부터 *FPP* 통일.

> ⚠️ Async port abuse

```text
모든 port async → queue overflow
```

→ sync·async 명확 선택.
