> Outline — *왜 도구도 qualify*하는가 — verification·development 도구의 *오류가 사용자 SW에 silent으로 들어감*. *TQL-1~5* — 도구 사용 목적 + DAL 조합으로 등급. TQL-1 (가장 엄격, code 직접 생성) → TQL-5 (단순 verification helper). 도구 *사용 시 신뢰*를 어떻게 입증하는가 — TOR (Tool Operational Requirements) + 개발자 산출물.
