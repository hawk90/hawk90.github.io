## 한 줄 요약

> **"RS-485는 산업 현장의 표준 시리얼입니다."** 차동 신호로 1.2 km 거리, multi-drop으로 32개 노드를 묶을 수 있습니다.

## 어떤 상황에서 쓰나

- 공장 PLC와 sensor/actuator 통신 (Modbus RTU)
- 건물 자동화(BACnet MS/TP)
- 태양광 inverter 모니터링
- 의료 장비 간 데이터 링크
- 노이즈 강한 환경의 보드 간 통신

## 핵심 개념

### 1) RS-422 vs RS-485

| 항목 | RS-422 | RS-485 |
| --- | --- | --- |
| Duplex | full-duplex | half-duplex(보통) |
| Driver | 1개 (point-to-point 또는 multi-drop) | 다수 (multi-master) |
| Receiver | 최대 10개 | 최대 32 unit load |
| Line | 4선 (TX±, RX±) | 2선 |
| 거리 | 1.2 km @ 100 kbit | 1.2 km @ 100 kbit |

RS-422는 한 driver가 여러 receiver에 broadcast 합니다. RS-485는 여러 driver가 같은 line을 share 합니다(turn-around).

### 2) 차동 전압 정의

```text
   A line ── B line  → 신호
   A > B + 200 mV → 1 (mark)
   A < B - 200 mV → 0 (space)
   |A - B| < 200 mV → 불확정 (idle 시 fail-safe 필요)
```

UART 신호의 1/0이 그대로 차동 A/B로 매핑됩니다.

### 3) Bus 구조와 종단

```text
       Driver/Receiver Node
            │
     ┌──────┴───────┐
     │ MAX485 등    │
     └──┬───────┬───┘
        │A      │B
        │       │
       ─┴───────┴─ Bus (twisted pair)
        │       │
       ┌┴───────┴┐
       │ Driver  │ ... 노드 32개
       └─────────┘
        
   양 끝에 120Ω 종단
```

종단은 CAN과 마찬가지로 양 끝에만, 중간 노드는 종단 금지. 노드 분기(stub)는 가능한 짧게.

### 4) Fail-safe biasing

idle 상태(모든 driver disable)에서 line이 unknown이 되면 receiver가 random 데이터를 출력합니다. 이를 막기 위해 외부 bias로 line을 살짝 1로 끌어둡니다.

```text
   +5V ── 680Ω ── A line
                  B line ── 680Ω ── GND
```

이렇게 하면 idle 시 A > B로 살짝 양수가 되어 receiver가 1을 출력합니다(UART idle 상태).

### 5) Half-duplex와 turn-around time

RS-485는 한 시점에 한 driver만 송신합니다. RX/TX 전환에 시간이 걸리므로 protocol에서 이를 고려해야 합니다.

```c
// MAX485 — DE/RE pin으로 송수신 전환
gpio_set_high(DE_RE);      // TX enable
uart_send(data, len);
while (!(USART1->ISR & USART_ISR_TC));   // wait last bit out
gpio_set_low(DE_RE);       // RX enable
```

마지막 비트가 line을 떠난 *후* DE를 내려야 합니다. 너무 빨리 내리면 마지막 비트가 잘립니다.

## 코드 / 실제 사용 예

Modbus RTU master 전송 예시입니다.

```c
// Modbus RTU — 9600 baud, 8N1
// silent interval 3.5 char time = 4 ms @ 9600

#define DE_RE_PIN  5

void modbus_send(uint8_t *frame, int len) {
    GPIOA->BSRR = (1 << DE_RE_PIN);          // TX enable
    delay_us(10);                              // settle
    
    for (int i = 0; i < len; i++) {
        while (!(USART1->ISR & USART_ISR_TXE_TXFNF));
        USART1->TDR = frame[i];
    }
    
    while (!(USART1->ISR & USART_ISR_TC));   // wait stop bit
    GPIOA->BSRR = (1 << (DE_RE_PIN + 16));   // RX enable
}

// 슬레이브 응답 대기 — silent interval로 frame end 감지
void modbus_receive(uint8_t *buf, int *len) {
    int idx = 0;
    uint32_t last_byte = get_ms();
    
    while ((get_ms() - last_byte) < 4) {
        if (USART1->ISR & USART_ISR_RXNE_RXFNE) {
            buf[idx++] = USART1->RDR;
            last_byte = get_ms();
        }
    }
    *len = idx;
}
```

CRC-16 (Modbus CRC) 계산 코드는 별도로 필요합니다.

## 자주 보는 함정

> ⚠️ Fail-safe bias 누락

idle 시 garbage data가 들어옵니다. master가 송신 후 RX로 전환된 직후 첫 byte를 매번 잘못 읽습니다. 680Ω + 680Ω bias가 표준 해법입니다.

> ⚠️ DE 너무 빨리 내림

마지막 byte의 stop bit가 line을 떠나기 전에 DE를 내리면 마지막 bit가 잘립니다. `TC` (Transmission Complete) flag를 기다린 후 내립니다.

> ⚠️ Twisted pair 미사용

평행 wire는 common-mode 노이즈가 차이로 들어옵니다. 반드시 twisted pair를 씁니다.

> ⚠️ GND 연결 누락

차동이라도 common-mode 범위는 -7 ~ +12 V로 제한됩니다. 노드 간 ground 전위차가 크면 transceiver가 죽습니다. 별도 ground wire를 한 줄 더 보내거나 isolation 사용.

> ⚠️ 중간 노드 종단

bus 중간의 노드에 120Ω을 달면 신호 진폭이 절반으로 줄어 receiver가 못 잡습니다. 양 끝만 종단.

> ⚠️ Address 충돌(Modbus)

같은 slave 주소를 두 노드에 설정하면 응답 시 collision. master는 garbage를 받습니다.

## 정리

- RS-485는 산업 현장의 표준 차동 시리얼입니다. 32 unit load, 1.2 km 거리.
- 양 끝 120Ω 종단, fail-safe bias(680Ω), twisted pair가 표준 구성입니다.
- DE/RE 전환은 송신 완료 후에만 합니다. `TC` flag로 마지막 비트 종료를 확인합니다.
- GND 연결 없이는 동작하지 않습니다. common-mode 범위를 넘으면 transceiver가 죽습니다.
- 중간 노드 종단, twisted pair 누락이 흔한 디버깅 원인입니다.
