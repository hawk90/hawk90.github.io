구두점은 *문장의 호흡과 의미*를 결정합니다. 같은 단어 묶음도 구두점에 따라 다른 뜻이 됩니다. 이 챕터는 각 구두점의 표준 용법을 정리합니다.

## Period (마침표) — `.`

### 기본 용도

```
1. 평서문 끝:
   The dog barked.

2. 명령문 끝 (강하지 않으면):
   Please close the door.

3. 약어:
   Dr., Mr., Mrs., Ph.D., a.m., etc.
```

### 약어와 마침표

```
미국식: Mr., Mrs., Dr., Ph.D.
영국식: Mr, Mrs, Dr, PhD     (마침표 없음)
```

자기 글이 어느 스타일을 따를지 결정하고 *일관성*을 유지합니다.

### 약어가 문장 끝에 올 때

```
He earned a Ph.D.   (마침표 하나)
회피: He earned a Ph.D..  (두 개)
```

## Comma (쉼표) — `,`

가장 자주 쓰이고 가장 자주 잘못 쓰이는 구두점입니다. Strunk의 Chapter 1 *Rules 2–4*를 함께 보면 좋습니다.

### 주요 용도

#### 1. 직렬 (Series)

```
red, white, and blue
gold, silver, or copper
```

*Oxford comma*(마지막 항목 앞 쉼표)는 학술 글에서 표준.

#### 2. 등위접속사 앞에서 독립절 연결

```
The dog barked, and the cat ran away.
She studied hard, but she failed the test.
```

#### 3. 종속절이 앞에 올 때

```
When the dog barked, the cat ran away.
If you have time, please come early.
```

종속절이 *뒤에 오면* 쉼표 보통 생략.

```
The cat ran away when the dog barked.   (쉼표 없음)
```

#### 4. 비제한 관계절 / 동격

```
My brother, who lives in Seoul, called me.
The capital of France, Paris, is beautiful.
```

#### 5. 도입 구·절·단어

```
However, the results were surprising.
In the morning, she went for a walk.
First, we need to define our terms.
Yes, I agree.
```

#### 6. 호명·동격

```
John, please close the door.
Susan, my old friend, called yesterday.
```

#### 7. 직접 인용 도입

```
He said, "I will be there."
"I will be there," he said.
```

### 쉼표를 *쓰지 않는* 자리

```
잘못: She said, that she would come.
바름: She said that she would come.   (간접 인용, 쉼표 없음)

잘못: The book, that I read was excellent.
바름: The book that I read was excellent.   (제한 관계절, 쉼표 없음)

잘못: The dog, barked loudly.
바름: The dog barked loudly.   (주어와 동사 사이 쉼표 없음)
```

## Semicolon (세미콜론) — `;`

### 주요 용도

#### 1. 두 독립절을 연결 (접속사 없이)

```
Mary likes apples; John likes oranges.
The road was long; we were tired.
```

두 절이 *의미적으로 가까울 때*. 마침표보다 약하고 쉼표보다 강한 휴식.

#### 2. 접속 부사 뒤

```
The plan was risky; however, we agreed.
She was tired; nevertheless, she finished the work.
```

*However, nevertheless, therefore, moreover, furthermore* 같은 부사는 *접속사가 아닙니다*. 쉼표만으로 두 절을 잇지 못합니다.

#### 3. 내부에 쉼표가 있는 리스트

```
We visited Seoul, Korea; Tokyo, Japan; and Beijing, China.
```

각 항목 안에 *이미 쉼표*가 있어서 *항목 구분*을 세미콜론으로.

## Colon (콜론) — `:`

### 주요 용도

#### 1. 목록 도입

```
He had three goals: wealth, power, and fame.
We needed three things: time, money, and patience.
```

콜론 앞은 *완전한 문장*이어야 합니다.

```
잘못: My favorite fruits are: apples, bananas, and grapes.
바름: My favorite fruits are apples, bananas, and grapes.
바름: I love three fruits: apples, bananas, and grapes.
```

#### 2. 부연·설명

```
The reason is clear: we need more time.
There is only one solution: hard work.
```

#### 3. 인용 도입 (형식적)

```
Lincoln said: "Four score and seven years ago..."
```

#### 4. 시간·비율·책 부제

```
3:45 PM
The ratio is 2:1.
Title: Subtitle
```

#### 5. 편지의 인사말 (미국식)

```
Dear Mr. Smith:
```

## Dash — `—`

대시는 *em dash*(긴 대시 —)와 *en dash*(짧은 대시 –)가 다릅니다.

### Em Dash — `—`

#### 1. 갑작스러운 단절

```
She turned to leave—then she saw him.
The answer is—well, complicated.
```

#### 2. 강조된 동격

```
Three of his colleagues—Smith, Brown, and Lee—signed the petition.
```

#### 3. 강한 강조

```
Only one thing matters—truth.
```

### En Dash — `–`

#### 범위·연결

```
pages 10–20
1990–2000
the New York–London flight
```

### 한글 글의 대시

한글 글에서는 *em dash*를 자주 *—* (양옆 공백 있음)으로 씁니다.

## Quotation Marks (따옴표) — `" "` / `' '`

### 직접 인용

```
He said, "I will be there at noon."
"I will be there at noon," he said.
```

### 짧은 작품 제목 (시, 단편, 노래)

```
Frost's "The Road Not Taken"
Joyce's "The Dead" from Dubliners
```

긴 작품(책, 영화, 저널)은 *이탤릭체*.

### 강조·의심·기술적 용어

```
The "experts" couldn't agree.   (반어)
The word "set" has many meanings.
She used the term "post-truth".
```

### 미국식 vs 영국식

```
미국식:
- 큰따옴표 외부: "I said, 'No,'" she replied.
- 인용 끝의 쉼표/마침표: 따옴표 *안에*
   "I will be there," he said.

영국식:
- 작은따옴표 외부: 'I said, "No,"' she replied.
- 인용 끝의 쉼표/마침표: 인용의 *원문에 속할 때만* 안에
   'I will be there', he said.
```

자기 글의 *스타일*을 결정하고 *일관성*을 유지합니다.

## Apostrophe (어포스트로피) — `'`

### 1. 소유격

```
the dog's tail              (단수 's)
the dogs' tails             (복수 s 뒤 ')
the children's school       (복수 자체가 다른 형태일 때 's)
Charles's friend            (s로 끝나는 단수 — 's)
```

### 2. 축약

```
do not → don't
cannot → can't
it is → it's
they are → they're
```

### 함정 — *its* vs *it's*

```
its     소유격: The dog wagged its tail.
it's    축약형 (it is, it has): It's raining.
```

이 둘의 혼동이 *영어 글쓰기의 가장 흔한 실수*입니다.

### 어포스트로피를 쓰지 않는 자리

```
잘못: The dog's are barking.   (복수 표시에 어포스트로피 X)
바름: The dogs are barking.

잘못: The 1990's were ...
바름: The 1990s were ...        (연대 표시)
```

## Parentheses (괄호) — `( )`

### 부가 정보

```
The conference (held in Seoul) was successful.
She earned a Ph.D. (in linguistics) at Stanford.
```

### 약어 도입

```
The European Union (EU) was founded in 1993.
```

### 인용 출처

```
The data show a significant trend (Smith, 2020).
```

## Brackets (대괄호) — `[ ]`

### 인용 안의 편집

```
He said, "I will [definitely] be there."   (편집자가 추가)
She wrote, "[They] don't understand."     (대명사 명확화)
```

### 괄호 안의 괄호

```
(See Smith [2020] for details.)
```

## Hyphen (하이픈) — `-`

### 1. 복합 형용사 (명사 앞)

```
a well-known author
a five-year plan
a high-quality product
```

### 2. 복합 명사

```
mother-in-law
self-control
twenty-one
```

### 하이픈을 *쓰지 않는* 자리

```
잘못: well known author (명사 앞에서는 하이픈)
바름: well-known author

잘못: The author is well-known.   (명사 앞이 아니므로 하이픈 X)
바름: The author is well known.
```

```
잘못: very-careful analysis
바름: very careful analysis    (-ly 부사 뒤는 하이픈 X)
```

## Ellipsis (말줄임표) — `...`

### 인용의 일부 생략

```
원문: "The quick brown fox jumps over the lazy dog."
인용: "The quick ... fox jumps over the lazy dog."
```

### 미완결·휴지

```
He whispered, "I can't believe it..."
She paused, then said, "But ..."
```

## Question Mark (물음표) — `?`

### 직접 의문문

```
What time is it?
Why did you leave?
```

간접 의문문에는 *물음표 없음*.

```
잘못: She asked what time it was?
바름: She asked what time it was.
```

## Exclamation Mark (느낌표) — `!`

진짜 감탄·명령에만. 평범한 진술에 강조를 위해 쓰는 것은 회피.

```
회피: It was a wonderful show!
바름: It was a wonderful show.

OK:   What a wonderful show!
OK:   Stop!
```

학술 글에서는 *거의 등장하지 않습니다*.

## 자기 글 점검

```
□ 직렬에 Oxford comma가 있는가?
□ Comma splice가 없는가?
□ Its / It's 혼동이 없는가?
□ 약어의 마침표가 일관되는가?
□ 따옴표 안/밖의 구두점이 스타일과 맞는가?
□ 하이픈이 명사 앞 복합 형용사에만 있는가?
```

## 한국 학생의 구두점 함정

```
1. *세미콜론* 한국어에 없음 → 영어로 옮길 때 어색
   해결: 두 독립절을 *마침표*로 분리

2. *어포스트로피* 한국어에 없음 → Its / It's 혼동
   해결: It is로 풀어 보아 점검

3. *직렬 쉼표* (Oxford comma) 한국어 가운뎃점과 다름
   해결: 영어는 마지막 and 앞 쉼표 (학술 표준)

4. *대시* 과용 — 번역체 신호
   해결: 쉼표 / 마침표로 대체 가능한지 확인

5. 따옴표 *부호* 차이
   한국: " ", 「 」, 《 》
   영어: " " (미국), ' ' (영국)
   해결: 한 글에 일관성
```

## 정리

- 마침표·쉼표·세미콜론·콜론·대시 — 각각의 자리
- 쉼표의 7대 용도와 *쓰지 않는* 자리
- 세미콜론 — 두 독립절 + 접속 부사
- 콜론 — 목록·부연 도입 (앞은 완전한 문장)
- 따옴표 — 미국식과 영국식의 차이
- 어포스트로피 — 소유격·축약 (Its / It's 혼동 주의)
- 하이픈 — 명사 앞 복합 형용사만
