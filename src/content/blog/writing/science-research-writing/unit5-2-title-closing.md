논문의 *제목*은 가장 짧고 가장 중요한 부분입니다. 모든 잠재 독자가 *처음 보는* 것이고, 인용·검색·소개에서 반복 등장합니다.

## Title의 역할

```
1. 검색 가능성 — 데이터베이스 키워드 매칭
2. 정확성 — 논문 내용을 정확히 반영
3. 매력 — 독자가 클릭 / 다운로드하게
4. 인용 가능성 — 다른 연구자가 자기 글에 인용
```

## 좋은 Title의 조건

```
□ 논문의 *핵심 주제*가 명확
□ 핵심 *변수 / 대상*이 포함됨
□ *방법론*의 신호가 있음 (가능하면)
□ *너무 길지 않음* (보통 15-20 단어)
□ *너무 짧지 않음* (5 단어 미만은 흔히 모호)
□ 검색에 노출될 *키워드*가 포함됨
```

## 좋은 제목의 패턴

### 1. Topic + Method

```
"Effect of caffeine timing on sleep efficiency: A multilevel analysis"
"Synthesis of biodegradable polymer blends from polylactide and rubber"
"Detection of dark matter halos using gravitational lensing"
```

*무엇*을 *어떻게* 연구했는지가 모두 포함됩니다.

### 2. Topic + Finding

```
"Caffeine timing reduces sleep efficiency in young adults"
"Rubber modification of PLA increases impact resistance"
```

핵심 *발견*을 제목에 직접. 결정적 발견이 있을 때 효과적.

### 3. Question 형식

```
"Does caffeine timing affect sleep quality? Evidence from a multilevel study"
"Can deep learning predict protein folding?"
```

질문이 *주의*를 끕니다. 학술 글에 점차 흔해지고 있습니다.

### 4. Subtitle 사용

```
"Caffeine and sleep: A multilevel analysis of timing effects"
"Polymer blends: From synthesis to mechanical characterization"
```

콜론(:)으로 *큰 주제 + 구체적 초점*을 분리.

## 함정 — 피해야 할 제목

### 1. 너무 일반적

```
회피: A study on polymers
회피: Investigation of sleep
회피: Research on AI
```

검색에서 묻힙니다. *어떤 polymers? 어떤 sleep? 어떤 AI?*

### 2. 너무 길음

```
회피: An investigation into the effects of various concentrations of
       caffeine consumed at different times of day on the sleep
       efficiency of college students at three universities

선호: Caffeine timing and sleep efficiency in college students
```

20 단어를 넘으면 *검색에서 잘림*. 핵심을 추려야 합니다.

### 3. Jargon 과다

```
회피: SOTA SSL methods for downstream task evaluation in NLP

선호: Self-supervised learning methods for natural language tasks:
       A benchmark comparison
```

분야 *외부 독자*도 이해할 수 있어야 합니다. 약어는 *논문 안에서 정의*되지만 *제목에서는 풀어 쓰기*.

### 4. 과장

```
회피: Revolutionary new approach to AI

선호: A new approach to AI alignment via constitutional methods
```

*Revolutionary*, *Breakthrough*, *Novel*(과사용) 같은 단어는 *심사*에서 깎입니다.

### 5. 약속 위반

제목이 *약속한 것*과 본문이 다르면 심사·인용에서 부정적입니다.

```
제목: A comprehensive review of X
본문: 실제로는 X의 일부만 다룸

제목: Causal effects of A on B
본문: 실제로는 상관만 보임
```

본문 완성 *후* 제목을 *마지막에 다시* 점검.

## 제목 작성 절차

### Step 1 — 키워드 추출

자기 논문에서 *가장 중요한 단어 5-8개*를 적습니다.

```
caffeine, timing, sleep, college students, multilevel, actigraphy
```

### Step 2 — 조합

핵심 키워드를 조합해 *세 가지 후보*를 만듭니다.

```
후보 1: Caffeine timing and sleep quality in college students
후보 2: Effects of evening caffeine on sleep efficiency: A multilevel
        study
후보 3: When you drink matters: Timing of caffeine consumption and
        sleep
```

### Step 3 — 동료 점검

후보들을 *분야의 동료*에게 보입니다. 어느 것이 *가장 클릭하고 싶은가*.

### Step 4 — 검색 시뮬레이션

Google Scholar에 자기 후보 제목으로 검색해 봅니다. *비슷한 제목이 너무 많으면* 차별화가 필요합니다.

### Step 5 — Abstract와의 일관성

제목과 abstract의 *주된 단어*가 겹쳐야 합니다. 다르면 검색에서 노출이 줄어듭니다.

## 학위 논문의 제목

학위 논문(thesis)은 저널 논문과 다른 관습이 있습니다.

```
저널 (짧음, 결정적):
"Caffeine timing reduces sleep efficiency"

Thesis (길음, 묘사적):
"Caffeine consumption patterns and sleep quality: An investigation
of timing effects in college students using actigraphic measurement"
```

학위 논문은 *범위와 방법*을 모두 명시하는 긴 제목이 흔합니다.

## 자기 점검

```
□ 제목이 *15-20 단어* 이내인가?
□ *핵심 변수 / 대상*이 포함되었는가?
□ Google Scholar 검색에서 *너무 흔하지* 않은가?
□ 약어가 *분야 외부*에도 이해되는가?
□ 본문이 *제목이 약속한 것*을 다 하는가?
□ Abstract와 제목의 *핵심어*가 일치하는가?
```

## 시리즈 마무리 — Science Research Writing 회고

이 시리즈에서 다룬 것:

```
Unit 1. How to Write an Introduction
   1.1 구조와 4-Move 모델
   1.2 문법과 어휘 (시제, signalling, passive, vocabulary)

Unit 2. Writing about Methodology
   2.1 구조와 어휘 (재료, 절차, 분석)
   2.2 문법 (passive, articles, adverbs)

Unit 3. Writing about Results
   3.1 구조와 어휘 (sequence, frequency, quantity, causality)
   3.2 문법 (시제, 비교급, hedging의 도입)

Unit 4. Writing the Discussion / Conclusion
   4.1 구조와 어휘 (발견 → 비교 → 한계 → 함의)
   4.2 문법 (hedging, 거리 두기, 인과 언어)

Unit 5. Writing the Abstract (and Title)
   5.1 Abstract의 4 부분 구조
   5.2 Title의 작성과 시리즈 마무리 (이 글)
```

## 책의 핵심 메시지

Glasman-Deal의 *Science Research Writing*의 핵심은 단순합니다.

> Science writing is much easier than it looks.

*과학 글쓰기는 보이는 것보다 훨씬 쉽다*. 이유 — 구조와 어휘가 *관습적*이기 때문입니다.

```
1. 자기 분야의 *모범 논문*을 자세히 본다
2. 구조 / 문법 / 어휘의 *패턴*을 발견한다
3. 그 패턴을 자기 *template*으로 만든다
4. 자기 연구에 *적용*한다
```

이 절차가 *영어 능력 보다 더 중요*합니다. 영어가 모국어가 아니어도 *관습을 따르면* 충분히 좋은 과학 글을 쓸 수 있습니다.

## AWG와의 통합

이 시리즈는 *Academic Writing for Graduate Students*의 짝으로 작성되었습니다.

```
AWG가 가르치는 것:
- 학술 영어의 원리
- Audience, Purpose, Organization
- 정의, 데이터 commentary
- 요약, 비평
- IMRD와 CARS 모델

SRW가 가르치는 것:
- 즉시 적용 가능한 template
- 과학 논문의 즉시적 패턴
- 시제 / signalling / passive의 결정
- Abstract와 Title 특별 강조
```

두 책을 *함께* 활용하면 학술 영어의 *이론과 실용*이 모두 채워집니다.

## 한국 연구자의 Title 함정

```
1. 너무 김 (20+ 단어) — 12-15 표준
2. 너무 모호 ("A Study on X")
3. 전문 용어 부족 — 검색 안 됨
4. 클리셰 ("Toward a New Understanding of...")
5. 한국어 "~에 관한 연구" 직역
```

## 시리즈 완결

12 글의 여정을 끝까지 함께해 주셔서 감사합니다. 자기 분야의 모범 논문을 자세히 보고, 자기 template을 만들고, 자기 연구로 적용하는 *반복의 과정*이 평생의 글쓰기 동반자가 되기를 바랍니다.
