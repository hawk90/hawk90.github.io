Discussion의 문법은 *Hedging*(헤지)에 집중됩니다. 자기 결론의 강도를 데이터가 옹호할 만큼만 표현하는 것이 학술 영어의 어조입니다.

> Be confidently uncertain. — Skelton (1988)

## Discussion의 시제

Discussion은 시제가 가장 자주 바뀌는 자리입니다.

```
Past — 자기 연구의 결과:
   Our analysis revealed a strong effect.
   We found that group A scored higher.

Present — 일반적 해석 / 함의:
   These results suggest that timing matters.
   The findings have important implications.

Present Perfect — 기존 연구 흐름:
   Previous studies have shown similar effects.
   Research has consistently found ...

Past — 다른 특정 연구:
   Smith (2020) reported a comparable pattern.
```

한 단락에 네 시제가 모두 등장할 수 있습니다.

```
Our analysis revealed [Past] a strong relationship. This finding
is [Present] consistent with previous reports (Smith, 2020). Smith
found [Past] a similar pattern in adults. More recent studies have
extended [Present Perfect] this work to younger populations.
```

각 시제가 다른 시간이나 맥락을 신호합니다.

## Hedging — 핵심 도구

### Modal Auxiliaries

```
강함:    will / does
중간:    may / can
부드러움: might / could
```

```
강함 (회피):
This pattern proves that caffeine causes insomnia.

부드러움:
This pattern suggests that caffeine may contribute to insomnia.

매우 부드러움:
This pattern could indicate that caffeine might contribute to insomnia
in some individuals.
```

자기 데이터의 결정성에 맞춰 modal을 선택합니다.

### Hedging 동사

```
강함:    show, demonstrate, prove, establish
중간:    indicate, reveal, find
부드러움: suggest, imply, point to
매우 부드러움: hint at, appear to suggest
```

```
The data demonstrate that ...   ← 강함
The data indicate that ...       ← 중간
The data suggest that ...        ← 부드러움
```

### Hedging 부사 / 형용사

```
강함:    clearly, certainly, undoubtedly
중간:    probably, likely
부드러움: possibly, perhaps, arguably
```

```
이 결과는 *명확히* 보인다.
   → The results clearly show ...

이 결과는 *아마도* 의미한다.
   → The results probably indicate ...

이 결과는 *어쩌면* 시사한다.
   → The results possibly suggest ...
```

### Hedging 명사 / 표현

```
강함:    a strong indication, clear evidence
중간:    indication, evidence
부드러움: a suggestion, a tendency, a possibility
```

## Distance — 거리 두기

자기 주장과 데이터 사이에 거리를 만드는 기법입니다.

### Appear / Seem

```
강함: The intervention reduces stress.
부드러움: The intervention appears to reduce stress.
매우 부드러움: It appears that the intervention may reduce stress.
```

### Based on / According to

```
Based on these results, we suggest that ...
According to our analysis, it may be that ...
In light of these findings, we propose ...
```

이런 표현이 조건부를 명시합니다.

### Limit (한정)

```
Within the scope of this study, ...
Under the conditions tested, ...
For the population examined, ...
```

주장의 적용 범위를 명시합니다.

## Generalization — 일반화 약화

자기 데이터에서 얼마나 일반화할지 결정해야 합니다.

### 약화 동사 — Appear / Seem / Tend

```
강함: Children living in poverty do poorly in school.
부드러움: Children living in poverty tend to do poorly in school.
        Children living in poverty appear to do poorly in school.
        Children living in poverty seem to do poorly in school.
```

*Tend to*, *appear to*, *seem to*가 "모든 ~"을 경향성으로 약화시킵니다.

### 주어 한정

```
일반화 (강함): Children living in poverty do poorly.

부분 한정: Many children living in poverty do poorly.
         A majority of children ...
         Some children ...
         In most parts of the world, children ...
```

### 예외 추가

```
With the exception of those enrolled in specialized programs,
children living in poverty tend to do poorly.

Apart from / Except for ...
```

## Combined Hedging — 조합

여러 헤지를 동시에 쓸 수 있습니다.

```
강한 주장:
Caffeine causes insomnia.

조합 헤지:
+ "according to some studies" (거리 두기)
+ "in some cases" (일반화 약화)
+ "may" (modal)
+ "contribute to" (약한 동사)

결과:
According to some studies, in some cases caffeine may contribute
to insomnia.
```

### 과한 헤지 주의

```
과도 (대부분의 단어가 헤지):
It could be tentatively concluded that some evidence may suggest
that certain individuals might possibly experience reduced sleep
quality.

→ 거의 아무 말도 하지 않음
```

너무 신중하면 책임 회피가 됩니다. 적절한 강도가 필요합니다.

## Causal Language — 인과 vs 상관

Discussion에서 상관과 인과를 구분하는 것이 학술적 정직성입니다.

### Strong (인과)

```
A causes B.
A leads to B.
A results in B.
A produces B.
```

### Moderate (인과 시사)

```
A may contribute to B.
A is associated with B.
A appears to influence B.
```

### Weak (상관만)

```
A is correlated with B.
A and B co-occur.
A is linked to B.
```

자기 연구가 상관만 측정했다면 *causes*가 아니라 *is associated with*나 *may contribute to*를 씁니다.

## 한계 인정의 문법

### 시제 — 보통 Present

```
This study has several limitations.
The cross-sectional design precludes causal inference.
Our findings may not generalize beyond the tested population.
```

한계는 지금도 유효하므로 Present를 씁니다.

### Modal 사용

```
The results may not generalize to ...
Findings could be biased by ...
The conclusions might be limited to ...
```

한계 자체에 적절한 modal을 사용합니다.

## Implication의 문법

### 시제 — 보통 Present

```
These findings have important implications for ...
The results suggest that policy makers should consider ...
This work contributes to our understanding of ...
```

함의는 현재와 미래에 유효하므로 Present를 씁니다.

### Future 권장

```
Future research should examine ...
Replication studies are needed to ...
Longitudinal designs could clarify ...
```

다음 연구 권장에 Future나 *should*를 씁니다.

## 사례 — 한 단락

```
Our findings indicate that consumption of caffeine within 6 hours
of bedtime is associated with reduced sleep efficiency in young
adults. This pattern is consistent with previous reports in older
populations (Park, 2018), suggesting that the underlying mechanism
operates across the lifespan. However, contrary to our hypothesis,
caffeine metabolism did not moderate the effect. This unexpected
finding may indicate that timing effects exceed individual variation
once a threshold is crossed, although alternative interpretations
warrant consideration. Several limitations should be acknowledged.
First, the cross-sectional design precludes causal inference.
Second, self-reported caffeine intake may have introduced recall
bias. Despite these limitations, our results have practical
implications for public health messaging. Future studies should
employ objective measurement and longitudinal designs to clarify
the temporal relationship.
```

분석:
- *indicate*, *is associated with* — 적절한 강도(중간)입니다.
- *suggesting that* — 부드러운 인과입니다.
- *may indicate* — modal과 약한 동사를 결합한 이중 헤지입니다.
- *should be acknowledged* — 한계 도입의 신호입니다.
- *precludes* — 강한 표현으로 방법론적 사실을 단언합니다.
- *Despite these limitations* — 균형을 잡아 줍니다.
- *should employ* — 다음 연구 권장입니다.

여러 헤지가 섞여 자연스럽게 흐릅니다.

## 자기 점검

- [ ] Modal verb가 데이터의 강도와 맞는가?
- [ ] 동사가 데이터의 결정성과 맞는가?
- [ ] 상관과 인과가 적절히 구분되었는가?
- [ ] 헤지가 과도하지 않은가?
- [ ] 한계가 명확한 시제로 표현되었는가?
- [ ] 함의가 적절한 범위에 한정되었는가?

## 정리

- Discussion의 시제는 자주 바뀝니다. Past(자기 결과), Present(일반), Present Perfect(기존 흐름)가 섞입니다.
- Hedging의 네 도구는 Modal, 동사, 부사, 명사입니다.
- Distance 표현으로 *appear*, *seem*, *based on*, *within the scope*를 씁니다.
- 일반화 약화에는 *tend to*, 주어 한정, 예외 추가가 있습니다.
- Causal과 Correlational 언어를 구분합니다.
- 한계는 Present로, 함의는 Present와 Future로 씁니다.
- 과한 헤지는 책임 회피가 됩니다. 적절한 강도가 필요합니다.

## 한국 연구자의 Discussion Grammar 함정

```
1. 헤지 부족 — 강한 단언
2. 헤지 과다 — 여러 헤지 중첩
3. 시제 흔들림
4. 인과와 상관 표현 혼동
5. 함의 시제가 약함
```
