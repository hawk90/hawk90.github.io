Introduction을 쓸 때 자주 마주치는 문법 결정과 어휘 선택을 정리합니다. 시제 쌍, signalling language, passive/active의 사용이 모두 작가의 의도를 표시합니다.

## Tense Pairs — 시제 쌍

### Present Simple vs Present Continuous

```
(a) PLA is a polymer obtained from corn.   ← Present Simple (확립된 사실)
(b) Researchers are exploring new methods.  ← Present Continuous (진행 중)
```

*Present Simple*은 받아들여진 사실에 씁니다. 무엇이 fact인지의 결정은 작가의 판단입니다. 자기 발견을 fact로 격상시키면 강한 주장이 됩니다.

```
약함 (Past Simple):
We found that the pressure increased as the temperature rose, which
indicated that temperature played a significant role.

강함 (Present Simple):
We found that the pressure increases as the temperature rises, which
indicates that temperature plays a significant role.
```

(b) 버전은 자기 발견이 일반 진리라는 신호입니다. 같은 데이터지만 더 강한 주장이 됩니다.

### Past Simple vs Present Perfect

```
Past Simple:     I lived in Tokyo for five years.
                 (but I don't live there anymore)
Present Perfect: I have lived in Tokyo for five years.
                 (and I still live there NOW)

Past Simple:     I broke my glasses.
                 (but it doesn't matter / I repaired them)
Present Perfect: I have broken my glasses.
                 (and so I can't see properly NOW)
```

두 시제의 차이는 시간이 아니라 현재와의 관련성입니다. 같은 사건이라도 Present Perfect면 지금도 영향이 남아 있다는 뜻이고, Past Simple이면 그때로 끝난 일이라는 뜻입니다.

### 실제 Introduction에서의 시제 변화

PLA 예시의 일부를 다시 봅니다.

```
For example, Penney et al. showed that PLA composites could be
prepared using blending techniques and more recently, Hillier
established the toughness of such composites. However, although the
effect of the rubber particles on the mechanical properties of
copolymer systems was demonstrated over two years ago, little
attention has been paid to the selection of an appropriate rubber
component.
```

시제 분포는 다음과 같습니다.

- *showed, established, was demonstrated* — Past Simple로 과거의 특정 연구를 가리킵니다.
- *has been paid* — Present Perfect로 그 부족함이 지금까지 이어진다는 점을 알립니다.

시제 변경이 의미를 신호합니다. *little attention has been paid*는 지금도 부족함이 남아 있음을 알리는 표현입니다.

만약 끝까지 Past Simple로 썼다면 의미가 달라집니다.

```
... little attention was paid to the selection of an appropriate
rubber component.
```

이 문장은 과거에는 부족했지만 지금은 어쩌면 충분하다는 함의를 가질 수 있습니다. 시제 결정이 연구의 동기를 약화시킵니다.

## Signalling Language — 신호어

문장 사이의 연결을 명시하는 것이 신호어입니다. 다섯 그룹으로 정리됩니다.

### Cause (원인)

```
The experiment was unsuccessful ___ the measuring instruments
were inaccurate.

선택지:
명사구 도입:               절 도입:
due to (the fact that)     as
on account of (the fact that)  because
in view of (the fact that)  since
```

*Since*는 *from that time*의 의미도 있어 혼동될 수 있습니다. 모호하면 *because*나 *as*를 선택합니다.

*Because*도 문장 시작에 쓸 수 있습니다.

```
Because the measuring instruments were inaccurate, the experiment
was unsuccessful.
```

### Result (결과)

```
The instruments were calibrated accurately, ___ the experiment
was successful.

선택지:
therefore                  as a result (of which)
consequently               which is why
hence                      so
```

*So*는 비격식이라 학술 글의 문장 시작에는 쓰지 않습니다.

### Contrast / Difference (대비)

```
British students are all vegetarians, ___ Norwegian students eat
meat every day.

선택지:
however                    on the other hand
whereas                    while
but                        by contrast
```

*On the contrary*와 *conversely*는 정반대를 강조합니다. 단순한 차이에는 부적절합니다.

### Unexpectedness (예상 외)

```
(a) ___ it was difficult, a solution was eventually found.
(b) ___ the difficulty, a solution was eventually found.
(c) It was difficult; ___ a solution was eventually found.

(a) Although / Even though / Though
(b) Despite / In spite of / Regardless of / Notwithstanding
(c) nevertheless / however / yet / nonetheless / even so
```

(a)는 절을 도입하고, (b)는 명사구를 도입하며, (c)는 두 독립절을 연결합니다.

### Addition (추가)

```
We used a batch processing system because it was more effective;
___ it was faster.

선택지:
in addition           also
moreover              secondly (etc.)
furthermore           in the second place (etc.)
apart from that/which what is more
```

*Besides*는 더 강력하고 설득적인 맥락에 적합합니다.

## Passive vs Active

Introduction에서 *we*의 사용에 대해 학생이 자주 질문합니다.

```
Use of "we":
✓ 연구 그룹 / 팀을 가리킬 때:
   We measured / We propose

✗ 사람들 일반을 가리킬 때 — 회피:
   대신: It is known/thought that ...

Use of "It" (formal):
   It is generally accepted that ...
   It has been suggested that ...
```

논문이 아닌 thesis에서는 *I*를 쓰기 어려우므로 passive나 dummy subject를 씁니다.

```
This article describes an algorithm for clustering sequences.
The present paper presents a set of criteria for selecting such
a component.
```

*This article*, *The present paper*가 dummy subject 역할을 합니다.

### Passive의 함정 — Agent 누락

```
The compound was synthesised, isolated and purified.
```

이 문장은 agent(누가 했는지)를 명시하지 않습니다. *We synthesised it*인지 *Brown synthesised it*인지 모호합니다.

해결책은 명시입니다. 자기 작업이면 *we*나 *the present authors*로, 다른 연구자의 작업이면 *Brown synthesised it*으로 적습니다.

## Vocabulary — Introduction의 핵심 동사

Glasman-Deal이 정리한 Introduction에 자주 등장하는 동사들입니다.

### 중요성 / 주목

```
receive (much) attention
gain (considerable) interest
become important
emerge as
play a key role in
```

### 기존 연구 묘사

```
show, demonstrate, establish, report
suggest, propose, indicate
find, observe, identify
examine, investigate, explore, study
```

이 동사들의 미묘한 차이는 다음과 같습니다.

```
demonstrate — 강한 증명
establish   — 분야의 합의
report      — 중립적 보고
show        — 가장 일반적
suggest     — 약한 주장
indicate    — 시사
investigate — 조사 행위 (결과 X)
```

### 부족함 / 미해결

```
remain unclear
be poorly understood
receive little attention
need further investigation
be largely unexplored
warrant further study
```

이 표현들이 Move 3의 핵심입니다.

### 자기 논문 소개 (Move 4)

```
present, describe, propose, develop
examine, investigate, demonstrate
characterize, evaluate, compare
```

```
The present paper presents ...
This study examines ...
We develop a model ...
```

### 방법 / 결과

```
prepare, synthesize
measure, calculate
model, simulate, derive
analyze, observe
```

각 동사는 문장에서 자리가 어느 정도 정해져 있습니다. *establish*는 기존 연구를, *propose*는 자기의 새 주장을 가리키는 식입니다.

## Write Your Introduction — 통합 과제

Unit 1의 마지막 단계는 자기 연구로 직접 Introduction을 작성하는 것입니다.

```
1. 자기 연구의 4-Move 답을 한 문장씩 적기
   - Move 1: 우리 분야가 왜 중요한가?
   - Move 2: 무엇이 알려져 있나?
   - Move 3: 무엇이 빠져 있나?
   - Move 4: 이 논문은 무엇을 하는가?

2. Unit 1.1의 template을 자기 답으로 채우기

3. 시제 점검:
   - 분야의 사실 → Present Simple
   - 특정 과거 연구 → Past Simple
   - 부족함의 지속 → Present Perfect

4. Signalling language 점검:
   - However가 Move 3 도입을 신호하는가?
   - 다른 연결어로 다양화 가능한가?

5. Passive / Active 점검:
   - We의 사용이 적절한가?
   - Thesis라면 dummy subject로 대체?

6. 자기 분야 모범 논문과 비교

7. 수정
```

이 절차로 첫 draft가 만들어집니다. 완벽하지 않아도 됩니다. Unit 2의 Methodology를 작성한 뒤 다시 Introduction을 보면 더 명확해집니다.

## 자가 점검 체크리스트

```
□ 4-Move가 모두 있는가? (Importance / Background / Gap / Present work)
□ 첫 문장이 너무 광범위하지 않은가? (Throughout history ...)
□ Move 3에 적절한 연결어가 있는가? (However, Despite, ...)
□ 시제가 의미에 맞게 사용되었는가?
□ 마지막 문장이 자기 논문의 핵심을 보이는가?
□ 단락 분할이 적절한가?
□ 인용이 자기 분야 표준 형식인가?
```

## 정리

- 시제 결정은 Present Simple(사실), Present Perfect(현재 관련성), Past Simple(과거 사건)로 갈립니다.
- Signalling language는 다섯 그룹으로 정리됩니다. Cause, Result, Contrast, Unexpectedness, Addition입니다.
- *We*는 연구 그룹을 가리킬 때 쓰고, thesis에서는 passive나 dummy subject로 대체합니다.
- Passive의 함정인 agent 누락에 주의합니다.
- Introduction의 핵심 동사로 *receive attention*, *demonstrate*, *remain unclear*, *present*가 자주 쓰입니다.
- Write 과제는 4-Move 답과 template을 결합한 뒤 시제·signalling·passive를 점검하는 절차입니다.

## 한국 연구자의 Grammar 함정

```
1. Present Perfect의 어려움 (한국어와 매핑이 안 됨)
2. 시제 흔들림
3. Passive 남용
4. Signalling이 한쪽에 치우침
5. Article(a/the) — 한국어에 없음
```
