Swales와 Feak가 Unit 1에서 출발점으로 잡은 세 가지는 *Audience*, *Purpose*, *Organization*입니다. 이 셋은 *글의 내용을 결정하기 전에* 미리 정해 두어야 하는 결정들입니다. 이 글은 그중 *Audience*와 *Purpose / Strategy*를 다룹니다.

## 왜 *Audience*와 *Purpose*가 먼저인가

대학원생이 자주 빠지는 함정 — *바로 쓰기 시작*. 자료를 모으고, 결과를 분석하고, 그대로 글을 *써 내려가는* 것. 결과는 *방향 없는 글*. 

```
회피 (방향 없음):
1. 데이터를 모은다
2. 분석한다
3. 결과를 본다
4. 그것을 글로 옮긴다
→ 누가 읽을지, 무엇을 강조할지 *결정 없이* 시작
→ 글이 *흩어짐*

선호 (방향 있음):
1. 누가 읽을지 결정 (audience)
2. 어떤 효과를 원하는지 결정 (purpose)
3. 어떤 구조로 갈지 결정 (organization)
4. 이제 글의 *모양*이 정해짐
5. 글을 쓰기 시작
```

이 *방향 결정*이 *글쓰기 시간의 30%*를 차지해도 *전체 효율*은 더 높습니다.

## Audience — 독자가 누구인가

학술 글쓰기에서 가장 먼저 던져야 할 질문은 *"누가 읽을 것인가"*입니다. 대학원생의 일반적 독자.

- 강의 담당 교수 (과제)
- 지도 교수와 논문 심사위원 (학위 논문)
- 학회 청중 (발표)
- 저널 심사위원 (논문)
- 같은 분야의 다른 연구자 (출판)
- 분야 밖의 학자 (학제 간 글)
- 일반 대중 (대중 글)

각 독자는 *기대하는 것*이 다릅니다.

```
강의 담당 교수:
- 학생이 *수업 내용을 이해*했는지 평가
- 정해진 *분량 / 형식*에 맞췄는지
- 자기 의견이 들어가는 것 환영

저널 심사위원:
- *학문적 기여*가 있는지 평가
- *방법론*이 엄밀한지
- 분야의 *기존 연구*와 연결되는지

학회 청중:
- *듣는 동안* 따라잡을 수 있어야
- *시각 자료* 의존
- 핵심 메시지가 *반복*되어야

분야 밖 학자:
- *기본 개념* 설명 필요
- *전문 용어 최소화*
- 분야 간 *연결*을 만들어야
```

*같은 결과*를 가지고도 *독자에 따라* 글의 모양이 완전히 *달라집니다*.

## 두 텍스트의 비교 — Desalination

Swales와 Feak는 *desalination* (담수화)에 관한 두 짧은 텍스트를 비교 예시로 듭니다.

### 텍스트 A — 일반 독자용

```
"People have been pulling freshwater out of the oceans for
centuries using technologies that involve evaporation, which
leaves the salts and other unwanted constituents behind.
Salty source water is heated to speed evaporation, and the
evaporated water is then trapped and distilled. ...
A potentially cheaper technology called membrane desalination
may expand the role of desalination worldwide, ..."
```

### 텍스트 B — 공학 전문가용

```
"Reverse osmosis (RO) membrane systems are often used for
seawater and brackish water desalination. The systems are
typically installed as a network of modules that must be
designed to meet the technical, environmental, and economic
requirements of the separation process. The complete
optimization of an RO network includes the optimal design of
both the individual module structure and the network
configuration."
```

### 비교

| 요소 | 텍스트 A (일반) | 텍스트 B (전문) |
|------|---------|---------|
| 어휘 | 일상적, 비유 활용 | 전문 용어 ("brackish water", "RO network") |
| 문장 | 다양한 길이, 비유적 | 일관된 길이, 명사 중심 |
| 정의 | 풀어서 설명 (reverse osmosis가 무엇인지 설명) | 정의 없이 사용 |
| 인용 | 단일 출처 | 명시적 인용 패턴 (Maskan et al., 2000) |
| 인칭 | "People have been ..." | 비인칭 / 수동 |
| 시제 | 다양 (현재 완료 등) | 현재 (학술 표준) |
| 톤 | 친근, 설명 | 객관, 분석 |

자기 글의 독자가 어디에 가까운지 *글을 쓰기 전에* 결정. 이 결정이 *어휘 선택 / 정의 방식 / 문장 길이 / 인용 빈도*를 모두 좌우합니다.

## *Insider* vs *Outsider*

학술 분야의 독자는 *insider*와 *outsider*로 나뉩니다.

```
Insider (내부자):
- 분야의 *기본 개념을 안다*
- *전문 용어*에 익숙
- 분야의 *논쟁사*를 안다
- *간결한 글*에 익숙

Outsider (외부자):
- *기본 개념 설명* 필요
- *용어 정의* 필요
- *맥락 설명* 필요
- *비유 / 일상 예시* 도움
```

```
독자에 따른 결정:

분야 내 동료 (insider):
"RO membrane systems..." (정의 없이)

학제 간 학회 (mixed):
"Reverse osmosis (RO), a membrane-based desalination
technique, ..." (간단 정의)

일반 대중 (outsider):
"Reverse osmosis works like a coffee filter, but for salt
water instead of coffee grounds." (비유)
```

자기 글의 *예상 독자가 누구*인지 *질문*. 다른 *수준의 독자*가 섞여 있다면 *주된 독자*를 정합니다.

## Purpose and Strategy — 목적과 전략

같은 주제라도 *목적이 다르면* 글의 *결*도 달라집니다. 학술 글쓰기에서 자주 마주치는 목적.

```
설명 (Explanation) — 무언가가 어떻게 작동하는지 보이기
   "이 알고리즘은 다음과 같이 동작한다 ..."

분석 (Analysis) — 부분과 관계를 풀어 보이기
   "이 결과를 세 요소로 나눠 보면 ..."

비교 (Comparison) — 둘 이상의 대상을 대비
   "방법 A와 방법 B를 비교하면 ..."

논증 (Argumentation) — 자기 주장을 근거로 옹호
   "이 방법이 우월하다. 그 근거는 ..."

종합 (Synthesis) — 여러 자료를 자기 관점으로 묶기
   "기존 연구의 세 흐름을 종합하면 ..."

평가 (Evaluation) — 가치 판단 (서평, 비평)
   "이 책의 강점은 ..., 약점은 ..."
```

대부분의 학술 글은 *둘 이상의 목적*을 섞습니다.

```
복합 목적의 예:

문헌 리뷰: 종합 + 평가
- "여러 자료를 묶고 + 그 가치를 판단"

토론 (Discussion) 섹션: 분석 + 논증
- "결과를 분석하고 + 자기 해석을 옹호"

서론 (Introduction): 설명 + 종합 + 동기 부여
- "배경 + 기존 연구 + 본 연구의 의의"
```

글을 *쓰기 전*에 *주요 목적*과 *부차적 목적*을 *자기 안에서* 정리해 두면 글의 *방향이 흔들리지 않습니다*.

## Task One — 자기 전략 점검

Swales와 Feak는 학생들에게 *자기 글쓰기 전략*을 점검해 보라고 합니다. 다음 항목 중 *자주 쓰는 것*에 표시.

```
□ 모국어로 먼저 생각하고 번역하기 (영어가 모국어 아닐 때)
□ 정보 수집과 조사에 시간을 많이 들이고 한 번에 작성
□ 자기 분야의 *모범 논문 참조* — 구성, 표현, 예시 활용 방식
□ "사정을 아는" 멘토에게 의존 — 어느 저널이 적절한지 등
□ 자기 분야 밖의 친구에게 표현 자문
□ *예상 독자*를 의식 — 무엇을 말하고 무엇을 빼야 하는지
□ *문체 변형*의 필요를 인식, 그 자원을 익히기
□ 다른 (출판된) 논문에서 *유용한 문구를 모아* 자기 생각을 엮기
□ 적절한 *저자 페르소나* 구축 — 학문 공동체의 구성원으로 보이기
□ *문장 수준 문법*에 집중 — 생각 전달의 가장 중요한 요소
□ *영어 원어민 동료*에게 교정 요청
□ *AI 도구* 활용 (문법 점검 / 표현 제안)
□ *여러 번 수정* — 한 번에 완성하지 않음
□ *큰 소리로 읽기* — 흐름 점검
```

학생마다 *패턴이 다릅니다*. 어떤 학생은 *모범 논문을 분해하고 모방*하는 데 시간을 많이 들입니다. 어떤 학생은 *문법보다 전체 구성*에 신경 씁니다.

*자기 강점과 약점*을 알면 *어디에 더 노력*해야 할지 보입니다.

### 한국 / 비영어권 학생의 전형 전략

```
강점:
- 모범 논문 *상세 분석* (한국 학습 문화)
- 문법 규칙 *명시적 학습*
- 모방 / 패턴 따라 하기

약점:
- *영어 원어민 톤*에 대한 직감 부족
- *번역체* 함정
- 자기 분야의 *관용 표현* 익히기 어려움
- *학술 영어의 미묘함* (헤지, 정중함 등)

권장 전략:
1. 자기 분야 *모범 논문 5편*을 *복사*해 두기
2. 그 안에서 *반복되는 표현 / 패턴* 추출
3. *Academic Phrasebank*(맨체스터 대학) 활용
4. *원어민 동료 / 교수*에게 *전체 흐름* 점검 요청
5. *AI 교정 도구*는 *최종 점검*에만 (생각의 흐름은 자기가)
```

## 전략 점검의 두 질문

```
1. 어떤 전략이 *자기에게 가장 유용*한가? 왜?
   같은 글에 대해 다른 전략으로 *바꿔 본 적*이 있는가?

2. 자기가 *사용하지 않는* 전략 중 *가장 키우고 싶은 것*은?
   어떻게 *시작*할 수 있을까?
```

이 질문에 대한 답은 *자기 분야*마다 다릅니다. 물리학자의 글쓰기 전략과 역사학자의 전략은 *같지 않습니다*.

자기 분야의 *모범 논문 두세 편*을 정해 두고, 그것을 *거듭 읽으면서* *어떤 결정이 어디서 내려졌는지* 관찰하는 것이 *가장 좋은 학습*입니다.

## 학술 글쓰기의 *여섯 가지 고려 사항*

Swales와 Feak는 학술 글쓰기가 *여섯 가지 고려 사항*의 결합이라고 정리합니다.

```
Audience ↕ Purpose ↕ Organization ↕ Style ↕ Flow ↕ Presentation
```

이 여섯은 *위에서 아래로* 내려가는 *고정된 순서*가 아닙니다. *모두 양방향 화살표*로 연결.

```
한 항목 결정 → 다른 항목 결정에 영향

예시:
- 독자가 *전문가*로 정해지면 → *어휘*가 좁혀짐
- 어휘가 *전문 용어*로 좁혀지면 → *흐름* 만드는 방법 바뀜
- 흐름이 *명사 중심*이면 → *발표 형식*도 달라짐
- 발표가 *글*이면 → *문장 길이* 유연
- 발표가 *말*이면 → *짧은 문장* + *반복* 필요
```

이 글에서는 *Audience*와 *Purpose*를 다뤘습니다. *Organization*부터는 다음 글에서 이어집니다.

## 자기 글 시작 전 점검 — Audience / Purpose

```
□ 주된 독자는 누구인가? (한 명을 떠올려 보기)
□ 그 독자가 *이미 아는 것*은? *모르는 것*은?
□ 그 독자가 *기대하는 형식*은? (학회 / 저널 / 일반)
□ 주된 목적은 무엇인가? (설명 / 분석 / 비교 / 논증 / 종합 / 평가)
□ 부차적 목적은? (대부분의 글은 *복합 목적*)
□ 자기 *글쓰기 전략*의 강점과 약점은?
□ 모범 논문 *2-3편*을 정해 두었는가?
□ 시간 배분: *방향 결정 30% + 작성 50% + 수정 20%*
```

## 한국 학술 글쓰기의 특수성

```
1. 영어 vs 한국어 결정
   - 자기 분야 *논문 90%가 영어*면 → 영어로
   - 자기 분야 *한국어 학술지*가 표준이면 → 한국어로

2. 영어로 쓸 때 한국 학생의 흔한 함정
   - *주어 생략* 직역 (한국식)
   - *수동태 남용* (격식의 환상)
   - *추상 명사* 과다 (-tion, -ity, -ness)
   - *과잉 헤지* (might possibly perhaps)

3. 한국어로 쓸 때 영어 직역의 함정
   - "~에 있어서", "~에 의해서" 같은 번역체
   - *명사화 과다*
   - 영어식 *세미콜론 / em-dash* 직역
   - *3인칭 객관성*의 강박 (한국어는 1인칭도 자연)
```

자기 글의 *주된 언어*에 맞는 *관용 표현*을 익히는 것이 *학술 글쓰기의 출발*.

## 정리

- **Audience** — 누가 읽는지가 어휘 / 정의 / 문장 길이 / 인용 빈도를 결정
- **Purpose** — 설명 / 분석 / 비교 / 논증 / 종합 / 평가 — 글을 쓰기 전에 주된 목적과 부차적 목적을 정리
- 자기 글쓰기 *전략*을 점검 — 강점과 약점을 알면 노력의 방향이 보임
- 여섯 가지 고려 사항(Audience–Purpose–Organization–Style–Flow–Presentation)은 *서로 영향*을 주고받음
- *Insider / outsider* 독자 구분이 어휘 / 정의를 결정
- 한국 학생은 *모범 논문 분석*에 강점, *영어 톤 / 관용*에 약점
