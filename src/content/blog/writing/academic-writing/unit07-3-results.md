*Results* 섹션은 *발견*을 보고하는 자리입니다. *해석*은 *Discussion*에서 — Results는 *무엇을 봤는지*에 집중합니다. 이 글은 Results의 표현 패턴, 통계 보고 형식, Data Commentary와의 관계를 다룹니다.

## Results의 목적과 어조

```
목적:    무엇을 발견했는지를 명확히 보고
어조:    객관적·기술적 — 해석 최소
시제:    과거 (관찰된 사실)
voice:   능동·수동 둘 다
```

Results는 *발견*을 *데이터로 뒷받침*해서 보고합니다. *왜 그런지*나 *무엇을 의미하는지*는 Discussion의 일입니다.

```
Results: Group A scored significantly higher than Group B (t = ..., p = ...).
Discussion: This difference may reflect the additional training that
            Group A received.
```

## Results의 표준 구조

```
1. 데이터 설명 (descriptive statistics) — 큰 그림
2. 주요 발견 1 — 가장 중요한 발견부터
3. 주요 발견 2
4. ...
5. 보조 발견 / 추가 분석
```

가장 중요한 발견을 *앞*에 둡니다. 일부 분야는 *연구 질문 순서*로 구성하기도 합니다.

## 첫 단락 — Descriptive Statistics

대부분의 Results는 *기본 통계*로 시작합니다.

```
The sample consisted of 245 participants (M_age = 22.3 years, SD = 3.1;
54% female). Demographic and baseline characteristics are presented
in Table 1. Groups did not differ on any baseline variable (all p > .15).
```

핵심 정보:
- 표본 크기, 인구 통계
- 기본 변수의 분포 (평균, SD)
- 그룹 간 *baseline 차이 없음* (실험 연구의 경우)
- Table 1 또는 첫 그림 참조

## 그림·표 참조

Results에서 *그림·표를 본문에서 참조*해야 합니다.

### 능동 참조

```
Figure 1 shows the relationship between caffeine intake and sleep
efficiency.
Table 2 presents the regression coefficients for each predictor.
Figure 3 illustrates the time course of the response.
```

### 수동 참조

```
The relationship between caffeine intake and sleep efficiency is shown
in Figure 1.
The regression coefficients are presented in Table 2.
```

### 괄호 참조

```
Caffeine intake was significantly correlated with sleep efficiency
(r = -.34, p < .001; see Figure 1).
```

### `As shown` 참조

```
As shown in Figure 1, sleep efficiency decreases linearly with caffeine
intake.
```

세 형태를 *섞어 쓰는 것*이 자연스럽습니다. 같은 형태가 반복되면 단조롭습니다.

## 통계 결과 보고

분야와 저널마다 형식이 다르지만, 다음이 일반적입니다.

### t-test

```
Group A scored higher than Group B, t(243) = 3.45, p = .001, d = 0.44.
```

핵심:
- t 값
- 자유도 (괄호 안)
- p 값
- 효과 크기 (Cohen's d, η², r 등)

### ANOVA

```
There was a significant main effect of treatment, F(2, 240) = 8.92,
p < .001, η²_p = .07. Post hoc tests revealed that ...
```

### Correlation

```
Caffeine intake was negatively correlated with sleep efficiency,
r(243) = -.34, p < .001, 95% CI [-.45, -.22].
```

### Regression

```
The model significantly predicted sleep efficiency, F(3, 241) = 12.45,
p < .001, R² = .13. Caffeine intake was a significant negative predictor
(β = -.32, p < .001), while age (β = .04, p = .52) and sex (β = -.02,
p = .76) were not.
```

### 신뢰 구간 (Confidence Intervals)

```
The mean difference was 2.3 points (95% CI [1.4, 3.2], p < .001).
```

현대 학술 영어에서 *p-value만이 아니라 효과 크기와 신뢰 구간*도 보고하는 추세입니다.

## 발견을 *말로* 표현

통계 표기 외에 발견을 *문장으로* 표현합니다.

### Highlighting 동사

```
The data revealed ...
The analysis showed ...
The results indicated ...
The data demonstrated ...
A significant difference emerged ...
The findings suggest ...
```

*Suggest*는 *Discussion에 더 가까운* 동사 — Results에서는 *demonstrate, indicate, show*가 더 흔합니다.

### 발견의 강도

```
강함: The data demonstrate that ...
중간: The data indicate that ...
부드러움: The data suggest that ...
```

자기 데이터의 *결정적 정도*에 맞춰 동사를 선택합니다.

### Negative Findings — 부정 결과

```
No significant difference was found between groups (t = ..., p = .42).
The correlation between X and Y was not significant (r = .08, p = .21).
Contrary to expectations, the intervention did not affect Y.
```

*부정 결과*도 *결과*입니다. *발견하지 못한 것*도 명확히 보고합니다.

## Data Commentary와 Results의 관계

Unit 4의 *Data Commentary*가 Results 작성의 *세부 도구함*입니다.

```
Unit 4 도구:                       Results에서:
- Location statement              표/그림 참조
- Indicative vs Informative      어느 형태로?
- 동사 선택                       shows, presents, indicates, demonstrates
- Hedging                         발견의 강도 조절
- Comparisons                     그룹/조건 간 비교
- Highlighting statements        무엇을 부각할지
```

Results 섹션 전체가 *연속된 Data Commentary*라고 봐도 됩니다.

## 발견의 순서

### 옵션 1 — 가장 중요한 발견부터

```
가장 중요한 발견 (보통 주된 가설)
   ↓
보조적 발견
   ↓
탐색적 / 추가 발견
```

자연과학·공학에 흔합니다. *결정적 발견*을 *앞*에 둡니다.

### 옵션 2 — 연구 질문 순서로

```
Q1의 결과
Q2의 결과
Q3의 결과
```

질문이 *논리적 순서*면 이 구조가 자연스럽습니다. 사회과학에 흔합니다.

### 옵션 3 — 분석의 흐름대로

```
기초 분석 (descriptive)
   ↓
주된 검정
   ↓
추가 / 보충 분석 (sensitivity, robustness checks)
```

데이터 분석의 *논리적 순서*를 따릅니다.

## 자주 쓰는 표현

### 발견을 *연결*

```
In addition, ...
Furthermore, ...
Similarly, ...
Consistent with [previous finding], ...
A similar pattern emerged for ...
This pattern held across all subgroups.
```

### 발견을 *대조*

```
However, ...
In contrast, ...
The opposite pattern was observed for ...
A different effect emerged when ...
Unlike X, Y did not show ...
```

### 양적 변화 표현

```
increased by 23%
decreased from X to Y
nearly doubled
reduced by half
remained stable
fluctuated between X and Y
```

### 통계적 유의성

```
significantly differed
significantly higher than
significantly correlated with
did not significantly differ
approached significance (p = .07)
```

*Approached significance*는 *학술 비판*을 받기 쉬운 표현입니다. *not significant but in the predicted direction* 같은 더 정확한 표현이 안전합니다.

## Results의 함정

### 1. 해석 섞임

```
회피:
Group A scored higher, which suggests that the training was effective.

수정:
Group A scored higher than Group B.   ← Results
[Discussion에서] The higher score in Group A may reflect the
effectiveness of the training.
```

*해석*은 Discussion으로 옮깁니다.

### 2. 모든 숫자 반복

```
회피:
Group A had a mean of 45.32 (SD = 5.61, range = 32–58), while
Group B had a mean of 41.87 (SD = 6.23, range = 28–55) ...
```

표가 모든 숫자를 보여 줍니다. 본문은 *핵심*만.

```
수정:
Group A scored higher on average than Group B (t = 3.45, p = .001;
see Table 1 for details).
```

### 3. 부정 결과 숨김

```
회피 (생략):
[가설 1의 결과만 보고하고 가설 2-4의 부정 결과는 생략]

수정:
[모든 결과 보고, 부정 결과도 명확히]
```

*Publication bias*를 만드는 행위. 학술 정직성에 어긋납니다.

### 4. 추가 분석의 *post hoc* 라벨

```
정직:
Exploratory analyses suggested that ...
A post hoc test revealed ...

회피 (정직하지 않음):
Additional analyses showed ...   ← post hoc임을 숨김
```

*가설을 세운 검정*과 *데이터를 본 후의 탐색*을 구분합니다.

## Unit 7 마무리

Unit 7에서 다룬 것:

```
7.1 출판 유형과 IMRD 구조
7.2 Methods Section 작성
7.3 Results Section 작성 (이 글)
```

핵심:

- 학술 출판의 다양한 형식, **IMRD**의 *모래시계* 모양
- Methods — *재현 가능성*, 수동·과거
- Results — *발견 자체*, 해석은 Discussion으로
- 통계 표기 형식의 *분야 표준*
- Data Commentary 도구가 Results의 세부 도구

## 한국 학생의 Results 함정

```
1. 해석을 Results에 — Discussion에 두기
2. 모든 데이터 본문에 — 표 / 그림 활용
3. 통계 표기 분야 표준 무시
4. 시제 흔들림
```

## 자기 글 점검

```
□ Results가 발견 자체?
□ 해석은 Discussion?
□ 통계 표기 분야 표준?
□ 표 / 그림과 상호 보완?
```
