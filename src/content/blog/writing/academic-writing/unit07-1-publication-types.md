Unit 7은 *연구 논문 작성*의 첫 절반입니다. 출판 유형의 이해부터 Methods, Results 섹션까지. 첫 글은 *어떤 형식의 논문을 쓸지*의 결정과, 자연과학·공학 논문의 표준 *IMRD 구조*를 다룹니다.

## 학술 출판의 유형

연구 결과를 발표할 수 있는 형식이 여러 가지입니다.

### Full Research Papers — 정식 연구 논문

```
길이:    4,000–10,000 단어 (분야에 따라 다름)
구조:    IMRD (Introduction, Methods, Results, Discussion)
독자:    같은 분야의 연구자
심사:    Peer review (보통 2-3명의 심사자)
```

학술 출판의 *기본 형식*입니다. 자연과학·공학·사회과학 대부분에서 표준입니다.

### Short Communications / Letters

```
길이:    1,500–3,000 단어
구조:    압축된 IMRD 또는 다른 형식
독자:    같은 분야 연구자
목적:    빠른 결과 보고, 학문적 우선권 확보
```

*Nature*, *Science* 같은 저널의 *Letters*가 이 형식입니다. *짧지만 영향력 큰* 발견을 빠르게 발표합니다.

```
특징:
- 도입이 매우 압축적 (1–2 단락)
- Methods가 supplementary materials에 자주
- Results와 Discussion이 통합되기도
- 그림과 표가 핵심
```

### Review Articles

```
길이:    5,000–15,000 단어
구조:    분야 / 주제별 구조 (IMRD 아님)
독자:    같은 분야 + 인접 분야 연구자
목적:    분야의 현재 상태 정리, 미래 방향 제시
```

*Annual Review of ...*, *Trends in ...*, *Reviews of Modern Physics* 같은 저널의 글입니다. 분야의 *권위자*가 보통 청탁받아 작성합니다.

### Conference Papers / Proceedings

```
길이:    4–10 페이지 (저널 논문보다 짧음)
심사:    Peer review (보통 더 빠름)
목적:    학회 발표 + 출판
```

공학·컴퓨터 과학에서 특히 중요한 형식. *ICML*, *NeurIPS*, *SIGGRAPH* 같은 학회는 *저널만큼 권위 있는* 발표 자리입니다.

### Workshop / Position Papers

```
길이:    2–4 페이지
심사:    가벼운 심사 또는 없음
목적:    새 아이디어 / 진행 중 연구 발표
```

학회의 *부속 workshop*에서 발표하는 짧은 글. 본격 연구 *전 단계*에서 *피드백*을 받기 위함.

## Short Communications의 자리

특히 *fieldwork 보고*가 있는 분야(생태학, 지질학, 고고학, 천문학)에서 *Short Communications*가 흔합니다.

```
보통의 구조:
1. 매우 짧은 Introduction (1–2 단락) — context와 gap
2. 압축된 Methods — 핵심 절차만
3. Results — 한두 가지 핵심 관찰
4. Brief Discussion — 의미와 다음 단계
```

```
예 — Astronomy:
"We report the detection of a Jupiter-mass planet orbiting Kepler-186
at a period of 287 days..."

1 단락 Introduction, 1 단락 Methods, 그림 한두 개, 1 단락 Discussion.
전체 1,500단어.
```

## IMRD 구조

자연과학·공학 *Full Research Paper*의 표준 구조입니다.

```
I - Introduction
M - Methods
R - Results
D - Discussion
```

각 섹션이 *다른 시간 흐름*과 *다른 어조*를 가집니다.

### Introduction — 도입

```
무엇:    연구 문제와 그 중요성, 기존 연구, 자기 연구의 위치, 질문/가설
시간:    과거 + 현재
어조:    설득적 — *이 연구가 왜 필요한가*
패턴:    Move 1 (Territory) → Move 2 (Gap) → Move 3 (Niche)
```

Unit 8의 *CARS 모델*에서 자세히.

### Methods — 방법

```
무엇:    어떻게 했는지 - 재료, 절차, 분석
시간:    과거 (이미 한 일)
어조:    중립적·기술적 — *그대로 재현 가능하게*
패턴:    재료/장비 → 절차 → 분석 방법
```

### Results — 결과

```
무엇:    무엇을 발견했는지 - 데이터, 통계, 그림/표
시간:    과거 (관찰된 사실)
어조:    객관적 — *해석 최소*
패턴:    그림/표 + Data Commentary (Unit 4)
```

### Discussion — 토론

```
무엇:    무엇을 의미하는지, 기존 연구와의 관계, 한계, 함의
시간:    현재 + 과거
어조:    해석적·평가적 — *결과를 맥락에 둠*
패턴:    핵심 발견 → 의미 → 비교 → 한계 → 함의 → 다음 연구
```

### IMRD의 *모래시계* 모양

```
넓음 ← Introduction (일반 → 구체)
   ↓
좁음 ← Methods (이 연구만)
좁음 ← Results (이 연구의 결과만)
   ↓
넓음 ← Discussion (구체 → 일반, 함의)
```

I와 D가 *넓게*, M과 R이 *좁게*. 글 전체가 모래시계 모양입니다.

## IMRD의 변형

분야와 저널에 따라 변형이 있습니다.

### IMRaD with Conclusion

```
Introduction
Methods
Results and Discussion (통합)
Conclusion
```

공학·응용과학에 흔합니다. Results와 Discussion을 *결과별로* 통합 — 한 결과 → 그 결과의 해석 → 다음 결과 → ...

### ILMRD with Literature Review

```
Introduction
Literature Review (별도 섹션)
Methods
Results
Discussion
```

사회과학·교육학에 흔합니다. 도입을 *Literature Review*로 확장해 별도 섹션으로.

### 인문학·역사학의 *비-IMRD* 구조

```
Introduction
Body (주제별 섹션 — 시기별, 사례별, 개념별)
Conclusion
```

인문학·역사학은 IMRD를 거의 쓰지 않습니다. *주제별 구조*가 표준입니다.

## 자기 분야의 표준 확인

자기 분야의 표준 구조를 *직접 확인*해야 합니다.

```
□ 자기 분야의 *모범 저널* 두세 곳 선택
□ 최근 1년 정도 *5–10편*의 논문을 펴서
□ 각 논문의 *섹션 제목* 정리
□ 공통 구조 식별
```

이 작업이 자기 분야의 *논문 모양*을 익히는 가장 빠른 방법입니다.

## 섹션 제목의 다양성

같은 IMRD라도 분야에 따라 *섹션 제목*이 다릅니다.

### 자연과학 / 공학 — 짧고 단순

```
1. Introduction
2. Methods (또는 Methodology, Experimental)
3. Results
4. Discussion
5. Conclusions
```

### 의학 / 보건과학

```
1. Introduction
2. Materials and Methods
3. Results
4. Discussion
5. Conclusion(s)
```

### 사회과학

```
1. Introduction
2. Literature Review (또는 Background, Theoretical Framework)
3. Research Design / Methodology
4. Results / Findings
5. Discussion
6. Conclusion / Implications
```

### 응용 분야 / 정책

```
1. Introduction
2. Background
3. Approach / Framework
4. Analysis
5. Recommendations
6. Conclusion
```

## 분량의 일반적 분포

```
20% — Introduction
15% — Methods
30% — Results
30% — Discussion
5%  — Conclusion / References
```

분야와 논문의 *성격*에 따라 다릅니다. 새 방법론을 제시하는 논문은 *Methods*가 더 많고, 실증 결과 중심 논문은 *Results*가 더 많습니다.

## Abstract — 따로 다룰 만함

논문의 첫 자리에는 *Abstract*가 옵니다. AWG는 abstract를 *별도 챕터*로 다루지 않지만, 짧게 짚어 둘 만합니다.

### Abstract의 4 부분

```
1. Background — 연구 문제 (1–2 문장)
2. Methods    — 어떻게 (1–2 문장)
3. Results    — 발견 (2–3 문장)
4. Implications — 의미 (1–2 문장)
```

200–300 단어가 일반적. 자기 논문의 *전체*를 한 단락으로 압축합니다.

### Abstract의 시제

```
Background:   현재 (분야의 현재 상태)
Methods:      과거 (이미 한 일)
Results:      과거
Implications: 현재 / 미래 (현재의 의미)
```

## 정리

- 학술 출판의 다양한 형식 — Full Paper, Short Communication, Review, Conference, Workshop
- 자연과학·공학의 표준 구조 — **IMRD** (Introduction, Methods, Results, Discussion)
- 분야에 따라 *변형* — IMRaD with Conclusion, ILMRD, 인문학의 주제별 구조
- 섹션 제목과 분량은 분야 / 저널마다 다름
- 자기 분야의 *모범 논문 5–10편*을 분해해 표준을 익힘
- **Abstract** — 200–300 단어로 논문 전체 압축

## 한국 학생의 출판 함정

```
1. 분야 / 저널 관습 무시 — 일반 IMRD만
2. Abstract 비대 / 약함
3. 제목 / 키워드 전문성 부족
4. 한국어 직역 abstract
```

## 자기 글 점검

```
□ 자기 분야 *표준 구조*?
□ Abstract 5 요소?
□ 키워드 검색 가능?
□ 길이 저널 기준?
```
