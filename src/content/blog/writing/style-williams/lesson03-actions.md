> **Style: Lessons in Clarity and Grace** Lesson 3 요약

## 이 챕터의 메시지

Williams의 가장 강력한 통찰 중 하나.

> **동사가 명사로 변하면 글이 흐려진다.**

이 패턴 — **nominalization** (명사화) — 이 거의 모든 나쁜 글의 핵심에 있다. 이걸 찾아 되돌리는 게 명확한 글의 첫 단계.

## Nominalization이란

동사를 명사로 만든 것.

```
discover (동사) → discovery (명사)
investigate → investigation
move → movement
decide → decision
analyze → analysis
react → reaction
fail → failure
```

문법적으로 잘못된 게 아니다. 그러나 **남용하면 글이 무거워진다**.

## 비교 — 동사 vs Nominalization

```
✅ The mayor decided to act.
   (동사: decided)

❌ The mayor's decision was to take action.
   (nominalization: decision, action — 동사가 사라짐)
```

같은 의미. 그러나 두 번째는 **명사가 가득**하고 동사가 약하다.

```
✅ We discovered that pollution affects health.
❌ Our discovery was that pollution has an effect on health.

✅ The teacher analyzed the poem.
❌ The teacher conducted an analysis of the poem.
```

패턴이 보인다. **약한 동사 + nominalization → 강한 동사**.

## 왜 Nominalization을 쓰나

좋은 의도에서 시작한다.

- "분석을 수행했다"가 "분석했다"보다 학술적으로 들림
- 동사를 명사로 만들면 **객관적**으로 느낌 ("decision was made" vs "I decided")
- 격식 있는 글에는 nominalization이 많아야 한다는 학습된 패턴

결과 — 짧고 명확한 문장 대신 길고 흐릿한 문장.

## 진단 절차

Williams의 단계.

**1. 명사를 찾는다** (특히 -tion, -ment, -ence, -ance, -ity로 끝나는)

```
"The implementation of the plan resulted in the improvement of efficiency."
                    ↑                          ↑
            implementation              improvement
```

**2. 그 명사를 동사로 되돌린다**

```
implementation → implement
improvement → improve
```

**3. 행위자를 찾는다** — 누가 그 행동을 하는가

```
"우리가 implement했다" → "we implemented"
"무엇이 improve됐다" → "X improved"
```

**4. 재구성**

```
원본: "The implementation of the plan resulted in the improvement of efficiency."

수정: "Implementing the plan improved efficiency."

또는: "We implemented the plan, which improved efficiency."
```

3분의 1 길이. 두 배 강한 문장.

## 흔한 패턴 — Nominalization + 약한 동사

```
"There is X" / "There are X"
   ↓
"There was the suggestion that..."
✅ "Someone suggested..."

"It is the opinion of...that"
   ↓
"It is the opinion of the committee that..."
✅ "The committee thinks that..."

"...had a tendency to..."
   ↓
"He had a tendency to forget."
✅ "He often forgot."

"Make a decision / move / improvement"
   ↓
"We need to make a decision."
✅ "We need to decide."

"Conduct an analysis / investigation"
   ↓
"They conducted an investigation."
✅ "They investigated."
```

이런 패턴이 보이면 — 거의 항상 더 강한 동사로 재작성 가능.

## "There be" 구문

```
"There is X that..."
"There are X who..."
```

이런 구문은 자주 진짜 동사를 숨긴다.

```
❌ There are several reasons why we should reconsider.
✅ We should reconsider for several reasons.

❌ There is a problem with the design.
✅ The design has a problem.   (또는 The design is flawed.)
```

## 격식 있는 글에서도 — 동사가 강하다

학술 / 비즈니스 글에서도 동사 중심이 더 강하다.

```
❌ "The investigation conducted by the research team revealed the 
    existence of significant discrepancies."

✅ "The research team found significant discrepancies."

❌ "It is our recommendation that immediate action be taken."

✅ "We recommend immediate action."

❌ "The implementation of these changes will result in the 
    improvement of overall system performance."

✅ "These changes will improve system performance."
```

학술이라도 — 짧고 동사 중심이 거의 항상 더 강하다.

## 예외 — Nominalization이 좋을 때

모든 nominalization을 없애야 하는 건 아니다. 다음 경우 — 그대로 두는 게 좋다.

**1. 그게 진짜 주제일 때**

```
"His decision shocked everyone."
   ↑
He의 결정이 주제. nominalization OK.
```

**2. 이전 문장에 동사 형태가 있을 때**

```
"He decided to leave. His decision shocked everyone."
                       ↑
이전 문장의 "decided"를 가리킴 — cohesion 위해 nominalization 사용.
```

**3. 학문적 개념이 명사로 정착된 경우**

```
"Globalization is changing the world economy."
"Inflation reached 5% last year."
```

이런 단어는 nominalization이지만 — **개념 자체가 명사**.

## 진단 연습

다음 문장을 수정해 보자.

```
❌ "The decision of the committee was to make a recommendation for 
   the postponement of the meeting."

수정 단계:
1. 명사: decision, recommendation, postponement
2. 동사로: decided, recommended, postpone
3. 행위자: committee
4. 재구성:

✅ "The committee decided to recommend postponing the meeting."
   (3 nominalizations → 2)

✅ "The committee recommended postponing the meeting."
   (decision은 implicit — 더 간결)
```

## 정리

- **Nominalization** — 동사가 명사로 변한 것
- 동사 → 명사로 가면 **글이 흐려진다**
- 진단 — -tion, -ment, -ence, -ance, -ity 명사 찾기
- 수정 — 명사 → 동사, 행위자 찾기, 재구성
- "There is/are" 구문은 자주 진짜 동사를 숨김
- 학술 / 격식 글에서도 동사가 강하다
- 예외 — 그게 진짜 주제이거나, 개념이 명사로 정착된 경우
