문장 흐름에 부가 정보로 끼어드는 어구는 양옆에 쉼표.

```
The best way to see a country, unless you are pressed for time, is to travel on foot.
```

문장의 흐름이 약간만 끊어진다면 쉼표를 둘 다 생략할 수도 있습니다. 그러나 한쪽만 두는 일은 없습니다. 다음은 잘못된 예입니다.

```
Marjorie's husband, Colonel Nelson paid us a visit yesterday.   // 잘못
My brother you will be pleased to hear, is now in perfect health. // 잘못
```

## 날짜와 삽입 표현

날짜에 들어가는 요소도 삽입 표현으로 봅니다.

```
February to July, 1992
April 6, 1986
Wednesday, November 14, 1990
```

단, 일·월·연 순서면 쉼표를 생략합니다.

```
6 April 1988
```

호명, 약어(*etc.*, *i.e.*, *e.g.*), 직위 약자도 삽입 표현으로 처리합니다.

```
If, Sir, you refuse, I cannot predict what will happen.
Horace Fulsome, Ph.D., presided.
Rachel Simonds, Attorney
```

## 비제한적 관계절

비제한적 관계절(*nonrestrictive*)은 쉼표로 감쌉니다. 핵심 정보를 더할 뿐 한정하지 않는 절입니다.

```
The audience, which had at first been indifferent, became more and more interested.
```

제한적 관계절은 쉼표를 쓰지 않습니다. 의미가 달라집니다.

```
My brother, who lives in Seoul, called me.  ← brother가 한 명, 부가 정보
My brother who lives in Seoul called me.    ← 여러 brother 중 서울 사는 그
```

## 핵심

- 삽입 표현은 양옆에 쉼표, 한쪽만 두지 말 것
- 비제한적 관계절(부가 정보)은 쉼표로 감싸기
- 제한적 관계절(한정)은 쉼표 없이
