## G

### Get

`Have got`은 *회화체*. 형식적 글에서는 *have*. 과거분사는 *got*이 표준(미국식)이지만, *gotten*도 미국 회화에서 사용됩니다.

```
회화: "I have got a new car."
선호 (격식): "I have a new car."

미국 회화: "I have gotten a new car."  ← gotten 사용
영국: "I have got a new car."  ← gotten 안 씀
```

### Gratuitous

*받지 않은, 부당한, 정당화 안 되는*의 의미. *공짜*가 아닙니다.

```
"The insult seemed gratuitous."
   ← undeserved (정당화될 이유가 없는)

"Gratuitous violence in films."
   ← 불필요한 / 정당화 안 되는 폭력

회피 (혼동):
"Gratuitous coffee at the meeting" → "Free coffee"
```

### Guess

*Suppose*, *believe*, *think*의 의미로 *과용*. 회화는 OK, 격식 글에서는 적절한 동사로.

```
회피 (격식): "I guess he is right."
선호: "I believe he is right." / "I think he is right."

선호 (회화):
"I guess we should go now."
```

### He is a man who

*군더더기* 표현.

```
회피: "He is a man who is very ambitious."
선호: "He is very ambitious."

회피: "She is a person who likes books."
선호: "She likes books."
```

*Is a man who / Is a person who* 패턴은 거의 항상 *제거 가능*.

## H

### Hopefully

원래 *with hope*. 지금은 *I hope* 또는 *it is to be hoped*의 의미로 *흔히* 쓰이지만, Strunk는 이 새 용법을 *nonsense*라고 부릅니다. *보수적 글*에서는 *옛 의미*만 씁니다.

```
회피 (현대 회화):
"Hopefully I'll leave on the noon plane."

선호 (격식):
"I hope to leave on the noon plane."
"It is to be hoped that I shall leave on the noon plane."

선호 (옛 의미, 부사로):
"She gazed at the door hopefully."
   ← 희망에 차서
```

현대 영어는 *변화 중*. 격식 / 학술 글에서는 *주의*.

### How ever / However

```
한 단어 (However) = nevertheless / in whatever way:
"However, we succeeded."
"However you do it, ..."

두 단어 (How ever) = how, with emphasis:
"How ever did you manage that?"
```

### However

*Nevertheless*의 의미로 *문장 첫머리에 쓰지 마세요*. 보통 *다른 자리*에 두면 더 자연스럽습니다.

```
어색:
"However, we succeeded in reaching camp."

자연:
"We succeeded, however, in reaching camp."
```

문장 *첫머리의 however*는 *in whatever way*, *to whatever extent*의 의미일 때만.

```
선호 (in whatever way):
"However you advise him, he will probably do as he thinks best."

선호 (to whatever extent):
"However hard he tries, he will not succeed."
```

현대 영어에서는 *however = nevertheless*가 *문장 첫머리*에 흔하지만, *Strunk-White 학파*는 *반대*.

## I

### Imply / Infer

`Imply` — 말하지 않고 *암시함*. `Infer` — 증거에서 *추론함*. *화자가 imply, 청자가 infer*.

```
"Farming implies early rising."
   ← 농사는 일찍 일어남을 *암시한다*

"Since she was a farmer, we inferred that she got up early."
   ← 우리는 그녀가 일찍 일어난다고 *추론했다*
```

```
규칙:
- Speaker / writer / fact → imply (암시)
- Listener / reader → infer (추론)

회피 (혼동):
"Are you inferring that I'm lying?" (잘못)
선호: "Are you implying that I'm lying?"
```

이 *구분*이 자주 *흐려지지만* 격식 글에서는 *지킵니다*.

### In regard to / As regards

`In regards to`는 *잘못*. `In regard to` 또는 `as regards`가 맞습니다.

```
회피: "In regards to your request, ..."
선호: "In regard to your request, ..."
또는: "As regards your request, ..."
또는: "Regarding your request, ..." (가장 간결)
```

### In terms of

대개 *군더더기*.

```
회피:
"The job was unattractive in terms of salary."

선호:
"The salary made the job unattractive."
또는: "The salary was low."
```

*In terms of*가 정당한 자리: *비교의 기준*이 다른 차원일 때만.

```
선호 (정당):
"Express the length in terms of meters."
   ← 측정 기준
```

### Insightful

*Perceptive*의 *과장된 대체어*. 정말 *비범한 통찰*일 때만. 평범한 발언에는 *perceptive*로 충분.

```
회피 (과장):
"Her insightful observation was that the meeting was long."
   ← 평범한 발언

선호:
"She observed that the meeting was long."
```

```
선호 (적절):
"Einstein's insightful theory transformed physics."
   ← 진짜 비범한 통찰
```

비즈니스 / 마케팅 영어에 *과도하게* 등장. *학술 / 격식 글*에서는 *주의*.

### Interesting

*소개의 도구로 쓰지 마세요*. 무엇을 *말한 다음* 독자가 흥미를 느끼게 해야지, *앞서서 흥미롭다고 알려서*는 안 됩니다.

```
회피:
"An interesting story is told of ..."
"It is interesting to note that ..."
"Interestingly, ..."

선호:
(단순히 그 이야기 / 사실을 들려주세요.)
```

*Show, don't tell*. 흥미로움은 *독자가 느끼는 것*. *작가가 선언*하는 것이 아닙니다.

### Irregardless

*Regardless*가 맞습니다. *Irregardless*는 *이중 부정* (ir- + -less)으로 *비표준*.

```
회피: "Irregardless of the weather, we will proceed."
선호: "Regardless of the weather, we will proceed."
```

### It is

*문장 시작 it is*는 *약한 도입*. *진짜 주어*가 있다면 그것으로 시작.

```
회피 (약함):
"It is important that we consider this."
"It is the case that the data shows ..."

선호 (강함):
"We must consider this."
"The data shows ..."
```

```
선호 (정당한 it is):
"It is raining." (날씨)
"It is five o'clock." (시간)
"It is John who called." (강조)
```

### Its / It's

가장 흔한 *영어 글쓰기 실수*. Rule 1에서 자세히 다룸.

```
its  = 소유격 (어포스트로피 없음)
it's = it is / it has (축약)

점검: 풀어 쓰면 it is / it has가 자연스러운가?
- 예 → it's
- 아니오 → its
```

### -ize

`-ize` 접미사로 *동사를 만드는 습관*에 주의. *Containerize*, *prioritize*, *finalize* 같은 단어들이 늘고 있지만, *이미 같은 의미의 좋은 동사가 있는지* 먼저 확인하세요.

```
회피                       선호
utilize                    use
prioritize                 rank / order
finalize                   complete / conclude
operationalize             carry out / put into practice
containerize               pack
formalize                  formalize (이 단어는 OK, 대체 없음)
optimize                   improve (또는 그대로)
```

```
회피:
"We need to utilize this resource to optimize our productivity
and finalize the project."

선호:
"We need to use this resource to improve our productivity and
complete the project."
```

비즈니스 영어의 *최악 함정*. *-ize*가 *전문성* 같지만 *모호함*과 *추상*을 만듭니다.

### -ization

*-ize*에서 *-ization*으로 *명사화*되면 *더 약해집니다*.

```
회피:
"The utilization of the resource for the optimization of
productivity required the finalization of the plan."

선호:
"We used the resource to improve productivity, which required
completing the plan."
```

## 한국 학습자의 G~I 흔한 함정

```
영어                  한국 학습자 실수
─────────────────────────────────────
have got              → 격식 글에 사용
hopefully             → 문장 시작에 과용
however               → 문장 첫머리에 직역적 사용
imply / infer         → 둘을 같은 의미로 혼동
in regards to         → "in regard to"가 표준
insightful            → 평범한 발언에 과장 사용
interesting           → "Interestingly, ..." 도입 과용
irregardless          → 비표준 단어
it is ... that        → 약한 도입 패턴
utilize/-ize          → use가 더 직접적
```

## 자기 글 점검

```
□ "Have got" → 격식 글에서 "have"
□ "Hopefully" → "I hope" / "It is to be hoped"
□ "However"가 *문장 중간*에 있는가? 첫머리 자제
□ "Imply"(화자) / "Infer"(청자) 구분?
□ "In regards to" → "In regard to" / "Regarding"
□ "In terms of" 군더더기 없는가?
□ "Insightful" 과장 없는가?
□ "Interesting" 도입 자제?
□ "Irregardless" → "Regardless"
□ "It is ... that" 약한 패턴 없는가?
□ "-ize / -ization" → 더 직접적 단어로?
```
