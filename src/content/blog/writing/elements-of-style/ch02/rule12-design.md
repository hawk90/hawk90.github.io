> Choose a suitable design and hold to it.

Chapter 2의 첫 번째 규칙입니다. Chapter 1이 문장 단위의 규칙이라면, Chapter 2는 글 전체의 구성에 대한 규칙입니다.

## 모든 글에는 골격이 있다

작가가 골격을 미리 그려 두고 쓰면 글의 모양이 분명해집니다. 사랑 편지처럼 골격이 없는 즉흥적 글도 있지만, 대부분의 글은 시작 전에 설계가 필요한 행위입니다.

```
소네트       → 14줄, 각 줄 5음보 (확정된 골격)
에세이       → 도입 → 본문 → 결말 (유연한 골격)
서평         → 책 요약 → 평가 → 추천 (관습적 골격)
학술 논문    → IMRaD (Introduction, Methods, Results, Discussion)
뉴스 기사    → 역피라미드 (가장 중요한 사실 → 세부)
소설         → 발단 → 전개 → 절정 → 결말
```

작가가 글의 형태를 또렷이 그릴수록 글이 잘 흘러갑니다.

## 디자인이 내용을 결정한다

같은 주제라도 디자인이 다르면 내용이 달라집니다.

```
주제: "내 첫 직장 경험"

설계 A — 시간 순서:
1. 입사 첫날
2. 첫 한 달
3. 첫 위기
4. 한 해 후
5. 떠나는 날

설계 B — 주제 별:
1. 동료 관계
2. 업무 학습
3. 회사 문화
4. 자기 성장

설계 C — 인과 연결:
1. 직장의 결과 (지금의 나)
2. 결정적 사건들 (이유)
3. 깨달음 (의미)
```

세 설계가 같은 사실에서 다른 글을 만듭니다. "어느 것이 맞다"가 아니라 목적에 맞는 것을 선택합니다.

## 디자인 결정의 질문

```
1. 누가 읽을 글인가? (독자)
2. 어떤 효과를 원하는가? (목적)
3. 시간이 흐르는 글인가? (시간 vs 주제)
4. 결론으로 향하는가, 결론에서 출발하는가?
5. 얼마나 긴 글인가? (분량)
6. 어떤 매체인가? (책 / 논문 / 블로그 / 이메일)
```

이 질문들에 답하면 디자인이 자연스럽게 결정됩니다.

## 끝까지 지킬 것

White의 강조는 "끝까지 지킬 것"입니다. 시작은 한 디자인, 끝은 다른 디자인은 최악의 글입니다.

```
회피 (디자인 흔들림):
"이 글은 내 첫 직장 경험을 시간 순서로 다룬다. 처음에 입사하고
... [중간에 갑자기 주제별로 바뀜] ... 동료 관계 측면에서는 ...
[다시 시간 순서로 돌아옴] ... 그래서 떠나기로 결정했다."

선호 (한 디자인 유지):
처음부터 끝까지 한 디자인. 디자인을 바꾸려면 명확한 신호와
함께. (예: "지금까지 시간 순서를 따랐다. 이제 주제 별로 정리하자.")
```

## 글의 큰 구조 — 3단 구성

가장 흔한 골격입니다.

```
1. 도입 (Introduction):
   - 무엇에 대한 글인가
   - 왜 이 글을 쓰는가
   - 어떤 결론을 향하는가 (선택)

2. 본문 (Body):
   - 사실 / 논리 / 예시
   - 디자인에 따라 구성 (시간 / 주제 / 인과 등)

3. 결말 (Conclusion):
   - 정리
   - 의미
   - 다음 단계 (선택)
```

학술 글, 비즈니스 글, 블로그, 에세이 — 대부분이 이 골격의 변형입니다.

## 학술 논문의 IMRaD

```
I - Introduction (배경 + 문제 + 가설)
M - Methods (어떻게 연구했는가)
R - Results (무엇을 발견했는가)
D - Discussion (의미와 한계)

장점: 정해진 골격이므로 *디자인 결정 부담 없음*
단점: 창의성이 제한
```

학술 글의 표준 디자인입니다. 자기가 따라야 할 분야 표준인지 점검합니다.

## 비즈니스 글의 피라미드

Barbara Minto의 *Pyramid Principle*:

```
1. 결론 (가장 중요한 메시지)
2. 핵심 근거 (3-5개 지지 논점)
3. 세부 (각 논점의 데이터)

장점: 바쁜 독자에 *상위 메시지 먼저*
단점: 결말의 *극적 효과* 없음
```

이메일·보고서·제안서에 적합합니다. 학술·에세이에는 덜 맞습니다.

## 뉴스의 역피라미드

```
1. 가장 중요한 사실 (5W1H)
2. 부가 사실
3. 배경
4. 인용 / 의견
5. 덜 중요한 세부

장점: 독자가 *어디서든 읽기를 멈춰도* 핵심을 얻음
```

뉴스·보도 자료의 표준입니다.

## 블로그 글의 디자인

블로그는 자유도가 높아 디자인 선택이 다양합니다.

```
설계 A — 문제 / 해결:
1. 문제 제시
2. 분석
3. 해결책
4. 결과

설계 B — 어떻게 하나 (How-to):
1. 무엇을 다룰지
2. 준비
3. 단계 1, 2, 3, ...
4. 점검

설계 C — 의견 / 에세이:
1. 주장
2. 근거
3. 반론 / 한계
4. 결론

설계 D — 학습 노트:
1. 무엇을 배웠나
2. 핵심 개념
3. 예시 / 코드
4. 자기 점검
```

자기 블로그의 스타일을 정해 일관성을 두면 독자에게 친숙한 패턴이 됩니다.

## 한국 글의 전통 디자인 — 기승전결

한국·동아시아 문학의 기-승-전-결입니다.

```
기 (起) — 시작, 주제 도입
승 (承) — 발전, 사실 / 사례
전 (轉) — 전환, 새 시각 / 반전
결 (結) — 마무리, 결론
```

4단 구성입니다. 서양 3단보다 극적입니다. 에세이·단편 소설에 자주 쓰입니다.

## 디자인을 결정하기 전 — Outline

White의 권고는 outline 작성입니다.

```
1. 머릿속 아이디어를 *나열*
2. 비슷한 것끼리 *그룹화*
3. 그룹을 *순서로 배열*
4. 각 그룹 안의 *세부 순서*
5. 도입 / 결말 *고민*
6. 글을 *시작*
```

Outline 없이 시작하면 중간에 길을 잃습니다. 짧은 글에서도 3분의 outline이 큰 차이를 만듭니다.

## 디자인이 실패하는 신호

- 같은 주제가 여러 자리에 흩어짐
- 시간 순서가 뒤섞임
- 도입과 결말이 서로 다른 글인 듯함
- 단락 사이 연결이 약함
- 독자가 어디로 가는지 모름

이런 신호가 보이면 디자인을 재검토합니다. 글을 재작성하는 편이 수정보다 빠를 수 있습니다.

## 자기 글 점검

- [ ] 글을 쓰기 전에 outline을 만들었는가?
- [ ] 한 명확한 디자인을 선택했는가?
- [ ] 디자인이 처음부터 끝까지 일관되는가?
- [ ] 디자인이 목적과 독자에 맞는가?
- [ ] 도입·본문·결말이 서로 연결되는가?
- [ ] 한 단락이 한 주제에 집중하는가? (Rule 13에서 다룸)
