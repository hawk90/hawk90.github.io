> Put statements in positive form.

분명히 단언하세요. 시들하고 색 없는 미적지근한 표현을 피하세요. *Not*은 부정을 표현할 때만 쓰고, 회피 수단으로 쓰지 마세요.

## 기본 대조

```
미적지근:
"He was not very often on time."

단언 (긍정 형식):
"He usually came late."
```

```
미적지근:
"She did not think that studying Latin was a sensible way to
use one's time."

단언:
"She thought the study of Latin a waste of time."
```

*Not*을 생각의 회피에 사용하면 글이 흐릿해집니다.

## 긍정 형식으로 바꾸기

*Not*으로 부정하면 독자는 무엇이 아닌지만 알 뿐 무엇인지는 모릅니다. 가능한 한 긍정 형식으로 같은 의미를 표현합니다.

```
not honest          → dishonest
not important       → trifling
not many            → few
not different       → alike / similar
not the same        → different
not often           → rarely / seldom
not happy           → sad / unhappy

did not remember               → forgot
did not pay any attention to   → ignored
did not have much confidence in → distrusted
did not allow                  → forbade
did not succeed                → failed
did not stay                   → left
```

한 단어 동사가 부정 + 일반 동사 조합보다 강합니다.

## *Not* 사용이 정당한 자리

부정을 진짜 강조할 때 씁니다.

```
선호 (부정 강조):
"This is not a drill."  ← 강한 부정
"He did not say that."  ← 부인
"They were not amused." ← 격식 부정

부정 + 긍정 대비:
"Not charity, but simple justice."
"Not that I loved Caesar less, but that I loved Rome more."
"Ask not what your country can do for you—ask what you can do
for your country."  ← Kennedy의 유명 문장
```

*Not A but B* 구조는 극적 효과를 만듭니다. 대비가 강조를 강화합니다.

## 조동사 남발 금지

`Would`, `should`, `could`, `may`, `might`, `can` 같은 조동사를 남발하면 글이 머뭇거립니다. 진짜 불확실할 때만 씁니다.

```
회피 (조동사 남발):
"I would suggest that we might consider that this could
possibly be an issue that may need our attention."

선호 (단언):
"This is an issue. We must address it."
```

### 학술·격식 글의 헤지

학술 글에서는 적절한 헤지가 과학적 정직성입니다. 그러나 과도하면 글이 약해집니다.

```
적절한 헤지:
"The results suggest that ..."
"This appears to indicate ..."
"It is possible that ..."

과도한 헤지:
"It would seem that the results might possibly suggest that
there could perhaps be ..."
```

한 문장에 한 헤지가 한도입니다.

## 약한 동사 → 강한 동사

흔히 약한 동사를 수식어로 보강하려 합니다. 강한 동사 하나가 더 좋습니다.

```
회피 (약함):
"He walked quickly down the street."
"She spoke in a loud voice."
"The team played very well."

선호 (강함):
"He hurried down the street."
"She shouted."
"The team excelled."
```

## Negative-Sounding 표현

부정적 인상의 표현은 긍정 등가물로 바꿉니다.

| 회피 | 선호 |
|------|------|
| not large | small / modest |
| not few | many / numerous |
| not unlike | similar |
| not unusual | common / typical |
| not impossible | possible |
| not unknown | familiar / known |
| not lacking | full / abundant |

이중 부정은 애매합니다. 직접 긍정이 명료합니다.

## *Not Un-* 패턴 — Orwell의 비판

Orwell의 *"Politics and the English Language"*:
> *Not un-* 패턴은 작가가 자기 견해를 숨기는 도구다.

```
회피:
"His response was not unfriendly."
"The result was not unexpected."
"Her work is not unimpressive."

선호:
"His response was friendly."  / "He responded warmly."
"The result was expected."    / "We expected this result."
"Her work is impressive."     / "Her work impresses."
```

*Not un-*은 학자의 가짜 정확성입니다. 명료한 글은 직접 말합니다.

## 비즈니스 글의 약한 단언

비즈니스·학술 글에 자주 등장하는 약한 표현입니다.

```
회피 (약함):
"It is somewhat important to consider that ..."
"Perhaps we might want to think about ..."
"I would like to suggest that maybe ..."

선호 (단언):
"We must consider ..."
"Consider ..."
"I suggest ..."
```

간결성과 결단력은 글에 권위를 줍니다.

## 한국어의 미적지근한 단언

한국어에도 비슷한 함정이 있습니다.

```
미적지근:
"그것은 좋지 않다."        → "그것은 나쁘다."
"많지 않다."               → "적다."
"안 정확하다."             → "부정확하다."
"잘 모르겠다."             → "모른다."  (또는 정직히)

조심스러운 표현:
"~인 것 같다."             → 종종 단언 가능
"~할 것 같다."             → 종종 단언 가능
"~라고 할 수 있다."        → "~이다."
```

### 한국 학술 글의 헤지

한국 학술 글도 과도한 헤지가 흔합니다.

```
회피 (과도):
"본 연구의 결과는 X가 Y에 영향을 미치는 것으로 보인다고 할 수
있을 것 같다."

선호:
"본 연구의 결과는 X가 Y에 영향을 미친다는 것을 보여 준다."
또는: "X는 Y에 영향을 미친다 (p < 0.05)."
```

근거가 있을 때 단언하세요. 헤지의 다층화는 책임 회피에 가깝습니다.

## 단언의 심리적 효과

```
강한 단언 → 작가가 자기 견해에 자신
약한 단언 → 작가가 책임 회피 / 자기 견해를 반쯤 숨김

독자의 신뢰:
- 단언하는 작가: "이 사람은 주제를 안다"
- 미적지근한 작가: "이 사람은 확신이 없다"
```

단언이 틀릴 위험은 있지만, 미적지근함의 지속 비용이 더 큽니다.

## *Don't Hedge* — White의 강조

White의 핵심:
> 두려워하지 마세요. 자기 견해를 분명히 말하세요. 틀리면 고치면 됩니다.

```
회피 (두려운 글):
"Some might argue that perhaps, in certain contexts, it could
be the case that ..."

선호 (용기 있는 글):
"I argue that ..."
"This is true because ..."
"The evidence shows ..."
```

## 자기 글 점검

- [ ] *Not*이 부정 진술에 진짜 필요한가, 아니면 회피인가?
- [ ] Not + 일반어를 한 단어로 바꿀 수 있는가? (*not honest* → *dishonest*)
- [ ] 조동사가 남발되지 않았는가?
- [ ] *Not un-* 패턴은 없는가?
- [ ] 약한 동사 + 부사를 강한 동사 하나로 바꿀 수 있는가?
- [ ] 한국 글에 미적지근한 헤지가 과다하지 않은가?
- [ ] 자기 견해를 분명히 말했는가?
