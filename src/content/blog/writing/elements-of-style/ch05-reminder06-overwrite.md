> Do not overwrite.

화려하고 장식이 많은 산문은 *소화하기 어렵습니다*. 한 단락에 비유 셋, 형용사 다섯이 들어가면 독자가 *과식 상태*가 됩니다.

## 과잉 작문의 신호

```
- 한 문장에 형용사 4개 이상
- 한 단락에 비유 3개 이상
- 같은 의미를 두 번 표현
- 평범한 사실에 *극적 어조*
- 매 단락에 "정말, 매우, 진짜" 같은 강조 부사
```

이런 글은 *작가가 자기에게 도취*된 신호입니다.

## 워드 프로세서의 유혹

E.B. White가 1979년에 이미 진단한 위험:

> 단어 컴퓨터 키보드의 *클릭과 흐름*이 작가를 유혹할 수 있습니다.

손가락이 키 위를 달리고 글자가 화면에 차오르는 *쾌감* 때문에 작가는 *불필요한 단어와 문단*을 더하게 됩니다.

오늘날 (2026년) 이 위험은 더 큽니다. AI가 *추가 단어*를 제안합니다. 클릭 한 번으로 단락이 *확장*됩니다.

```
원본:
"The meeting was productive."

AI 확장 (회피):
"The meeting was incredibly productive, with significant
breakthroughs achieved through robust discussions, leading to
actionable insights that will undoubtedly catalyze our future
endeavors."
```

확장된 버전은 *원본의 모든 의미를 잃었습니다*.

## 방어 — 가차 없는 자르기

방어는 *하나*입니다.

> 나중에 다시 읽고 *가차 없이 잉여를 지우세요*.

자기 글에 *사랑*을 두지 마세요. 한 문장이 *예쁘게 보여도* 글 전체에 *기여하지 않으면* 잘라냅니다.

### "Kill your darlings"

William Faulkner의 격언:

> In writing, you must kill all your darlings.

자기가 *가장 좋아하는 문장*이 종종 글에 가장 *해로운* 문장입니다. 그것이 자기 도취의 표시입니다.

## 과잉의 가시화

자기 글이 *과잉인지* 점검하는 도구.

### 1. 단어 수 세기

```
이 단락의 의도된 의미를 한 줄로 적기 → 단어 수
실제 단락의 단어 수
```

두 수가 *극적으로 다르면* 과잉입니다.

### 2. 형용사 / 부사 비율

```
한 단락에 (형용사 + 부사) / 명사 + 동사
   ≤ 30% → 정상
   30-50% → 약간 과잉
   > 50% → 명백한 과잉
```

### 3. 비유 빈도

```
한 단락에 비유 1개 이하 → 정상
2개 → 조심
3개 이상 → 과잉
```

## 자기 점검의 절차

```
1. 초안 완성
2. 24시간 대기
3. 다시 읽기 — *불필요한 단어*에 표시
4. 한 줄에 *5단어 이상*을 잘라낼 수 있나 점검
5. 한 단락에 *한 문장*을 잘라낼 수 있나 점검
6. 한 글에 *한 단락*을 잘라낼 수 있나 점검
```

이 절차가 글을 *25-50% 줄입니다*. 줄어든 글이 보통 *더 강합니다*.

## 한국어 글의 과잉

한국어 글에도 과잉의 신호가 있습니다.

```
- "~라고 할 수 있다" (그냥 ~이다)
- "~인 것 같다" (그냥 ~이다, 헤지 필요 없으면)
- "정말로, 진짜로, 매우" (강조 부사 남발)
- "~을 통해서" (~으로)
- 같은 의미의 형용사 나열 ("크고 거대한", "오래되고 낡은")
```

이런 *한국어 과잉*을 잘라내면 글이 산다.

## 마무리

자기 글이 *과잉인지* 의심되면:

```
□ 단어 수 / 의도된 의미를 한 줄로 → 비율 확인
□ 형용사와 부사가 명사와 동사를 가리지 않는가
□ 비유가 너무 많지 않은가
□ 자기 도취의 *예쁜 문장*이 글 전체에 기여하는가
□ 가차 없이 잘라낼 수 있는가
```
