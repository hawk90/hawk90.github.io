이 챕터는 짧습니다. 규칙은 11개 항목이지만 하나하나가 작은 결정입니다. 글의 형식 관습을 정리합니다.

## 왜 형식이 중요한가

학생들은 종종 내용만 좋으면 형식은 부차적이라고 생각합니다. 그러나 다음과 같은 문제가 있습니다.

형식이 잘못되면:
- 편집자나 교수가 내용에 집중하지 못합니다.
- 출판·학회 거절의 흔한 사유가 됩니다.
- 작가의 전문성에 의심이 갑니다.

형식이 잘 되면:
- 독자가 내용에 집중합니다.
- 글이 전문가의 작품으로 보입니다.
- 작가의 세심함이 신호로 전달됩니다.

Strunk 시대(1918년)부터 지금까지 형식의 디테일이 글의 신뢰를 만듭니다.

## Colloquialisms — 구어체

구어체 표현이나 슬랭을 쓸 때는 그냥 씁니다. 따옴표로 감싸 "나는 이걸 안다, 하지만 알면서도 쓴다" 같은 신호를 보내지 마세요. 그것은 아는 사람들끼리의 동아리에 독자를 초대하는 듯한 거만함을 풍깁니다.

```
회피:
"He was a real 'wise guy' about the whole thing."

선호:
"He was a real wise guy about the whole thing."
```

따옴표는 인용에만 씁니다. 작가의 자의식 표현에 쓰지 않습니다.

### 슬랭의 정당한 자리

1. 대화 인용 (소설, 회고)
2. 직접 인용 (인터뷰, 발화 기록)
3. 의도적 톤 (회화적 에세이)

회피:

4. 과시적 슬랭 ("I'm not gonna go there.")
5. 신조어 / 인터넷 슬랭의 격식 글 사용

## Exclamations — 느낌표

평범한 진술에 느낌표를 붙여 강조를 만들어 내려고 하지 마세요. 느낌표는 진짜 감탄이나 명령에만 씁니다.

```
회피:
"It was a wonderful show!"
"The data clearly shows!"
"This is important!"

선호:
"It was a wonderful show."
"The data clearly shows X."
"This matters because ..."

선호 (정당한 느낌표):
"What a wonderful show!"  ← 진짜 감탄
"Halt!"                    ← 명령
"Help!"                    ← 외침
"Indeed!"                  ← 강한 동의
```

### F. Scott Fitzgerald의 충고

> "느낌표는 자기가 던진 농담에 자기가 웃는 격이다."

학술·격식 글에서 느낌표는 드물게 쓰거나 아예 쓰지 않습니다.

## Headings — 머리글 / 제목

원고를 출판용으로 제출할 때는 첫 페이지 위에 공간을 충분히 둡니다.

표준 형식:
- 첫 페이지 위 1/4 지점에 제목
- 제목 한 줄 뒤 본문 시작
- 다음 페이지부터 위쪽 가까이
- 제목 끝 마침표 생략 (물음표나 느낌표가 필요한 제목이면 둠)

### 현대 학술 형식

저널 논문:
- 제목 (대문자 첫 글자, 약 12-15 단어)
- 저자 / 소속
- 초록 (200-250 단어)
- 키워드 (3-5개)
- 본문 (IMRaD)
- 참고문헌

학위 논문:
- 표지 (제목, 저자, 학교, 연도)
- 초록 (300-500 단어)
- 목차
- 본문 (장, 절, 항)
- 참고문헌
- 부록

### 제목 작성의 원칙

좋은 제목:
- 명확한 핵심
- 키워드 포함 (검색에 도움)
- 12-15 단어 이내 (저널 표준)
- 콜론으로 부제 분리 가능

회피:
- 너무 모호
- 너무 길음 (20단어+)
- 클리셰 ("Toward a New Understanding of ...")
- 의문문 (대부분의 분야)

```
선호:
"Caffeine Consumption and Sleep Quality in University Students"
"The Effects of Climate Change on Coffee Production in
Ethiopia: A 30-Year Analysis"

회피:
"A Study"
"Some Observations on Climate"
"Caffeine? An Inquiry"
```

## Hyphen — 하이픈

두 단어가 합쳐져 복합 형용사를 만들면 하이픈이 보통 필요합니다.

```
선호:
"He belonged to the leisure class and enjoyed leisure-class
pursuits."
"She entered her boat in the round-the-island race."
"A well-known author."
"A state-of-the-art system."
"High-quality data."
```

### 복합 형용사 규칙

```
명사 앞 (수식어 위치) → 하이픈:
"a well-known book"
"a state-of-the-art lab"
"a 20-year-old man"

명사 뒤 (서술어 위치) → 하이픈 없음:
"The book is well known."
"The lab is state of the art."
"The man is 20 years old."
```

### 하이픈 없이 한 단어 — 굳어진 합성어

```
회피                 선호
water-fowl       →   waterfowl
wild-life        →   wildlife
bell-boy         →   bellboy
foot-ball        →   football
e-mail           →   email (현대)
on-line          →   online
data-base        →   database
```

영어 단어의 진화는 두 단어 → 하이픈 → 한 단어의 방향을 따릅니다.

```
bed chamber → bed-chamber → bedchamber
data base → data-base → database
on line → on-line → online
```

새 합성어는 하이픈으로 시작했다가 시간이 지나면 한 단어가 되는 경향이 있습니다. 의심이면 사전을 확인합니다.

### *News-Free Press*의 유머

하이픈을 잘못 넣어 우습게 만든 유명한 예가 있습니다. Chattanooga의 두 신문 *News*와 *Free Press*가 합병되어 *Chattanooga News-Free Press*가 되자, 마치 "뉴스 없는 신문"처럼 읽혔습니다.

```
의도: News + Free Press의 합병 신문
읽힘: News-Free (뉴스 없는) Press
```

하이픈의 위치가 의미를 완전히 바꿉니다.

### 한국어의 하이픈

한국어 표준은 하이픈을 거의 쓰지 않습니다. 띄어쓰기로 구분합니다.

```
한국어:
- "출판사 편집자" (띄어쓰기)
- 하이픈 사용: 외래어 표기, 학술 용어
   "데이터-기반" "사용자-중심"

영어 직역 시 주의:
- 영어 하이픈 → 한국어는 일반적으로 *띄어쓰기*
- 영어 한 단어 → 한국어 한 단어
```

## Margins — 여백

좌우 여백을 같은 폭으로 둡니다. 편집과 주석이 많이 들어갈 원고는 왼쪽 여백을 더 넓게 둡니다.

### 현대 표준 (대학 / 저널)

표준 여백 (US Letter / A4):
- 위 / 아래: 1 inch (2.54 cm)
- 좌 / 우: 1 inch (2.54 cm)
- 들여쓰기: 0.5 inch (1.27 cm)
- 행간: 더블 스페이싱 (초안), 1.15-1.5 (출판)

수정용 원고:
- 좌측 여백: 1.5 inch (편집자 메모용)

폰트:
- Times New Roman 12pt (전통)
- Arial / Calibri 11pt (현대)
- 페이지 번호: 우상단 또는 하단 중앙

## Numerals — 숫자

날짜와 일련번호는 풀어 쓰지 말고 숫자나 로마 숫자로 표기합니다.

```
선호:
"August 9, 1988"
"Part XII"
"Rule 3"
"352d Infantry"
"page 247"
"Figure 5"
"Chapter 7"
```

### 대화에서는 풀어 쓰기

```
선호 (대화):
"I arrived home on August ninth."
"In the year 1990, I turned twenty-one."
"Read Chapter Twelve."
```

### 학술 글의 숫자 규칙 (APA / MLA)

```
APA Style:
- 10 이하: 풀어 쓰기 (one, two, ..., nine)
- 10 이상: 숫자 (10, 11, ...)
- 예외: 측정 / 통계는 항상 숫자 (3 mg, 5%, p < 0.05)
- 문장 시작: 항상 풀어 쓰기

선호 (APA):
"Three students attended."
"The sample included 25 participants."
"Fifteen out of 25 agreed."  (문장 시작이면 풀어 쓰기)
"The mean was 3.7."
```

```
MLA Style:
- 100 이하: 풀어 쓰기
- 100 이상: 숫자
- 분야별 변형 많음
```

## Parentheses — 괄호

괄호를 포함한 문장의 구두점은 괄호가 없는 것처럼 처리합니다. 괄호 안의 내용은 자기 안에서 따로 구두점을 갖되, 끝의 마침표는 생략합니다 (괄호 안이 물음표나 느낌표라면 둡니다).

```
선호:
"I went to her house yesterday (my third attempt to see her),
but she had left town."

"He declares (and why should we doubt his good faith?) that he
is now certain of success."
```

### 괄호가 문장 전체를 감쌀 때

완전히 독립된 표현이나 문장이 통째로 괄호 안에 들어가면 마침표를 괄호 안에 둡니다.

```
선호:
"(He had already decided to leave when the letter arrived.)"

"The result was decisive. (See Figure 3 for details.)"
```

### 괄호의 종류

| 기호 | 이름 | 용도 |
|------|------|------|
| ( ) | parentheses | 본문에 자주 사용 |
| [ ] | brackets | 인용 안의 편집자 주, 수정 |
| { } | braces | 수학 / 컴퓨터, 본문 드뭄 |
| 〈 〉 | angle brackets | 학술적, 격식 |

선호: "The author writes that 'he [the prince] was wise.'" -- 편집자가 추가한 명시 [the prince]

## Quotations — 인용

증거로 인용하는 공식 인용은 콜론 뒤에 따옴표로 묶습니다.

```
선호 (공식 / 긴 인용):
"The United States Coast Pilot has this to say of the place:
'Bracy Cove, 0.5 mile eastward of Bear Island, is exposed to
southeast winds, has a rocky and uneven bottom, and is unfit
for anchorage.'"
```

### 짧은 인용 — 쉼표

동격이나 동사의 직접 목적어로 들어가는 인용은 쉼표 + 따옴표로 묶습니다.

```
선호:
"I am reminded of the advice of my neighbor, 'Never worry
about your heart till it stops beating.'"

"Mark Twain says, 'A classic is something that everybody wants
to have read and nobody wants to read.'"
```

### Attributive 위치 (누가 말했다)

인용 뒤에 attributive(누가 말했다)가 따라오면 쉼표를 따옴표 안에 둡니다 (미국식).

```
선호 (미국식):
"'I can't attend,' she said."
"'The data shows an increase,' the report states."

영국식:
"'I can't attend', she said."
   → 영국은 쉼표를 따옴표 밖에
```

### 긴 인용 — Block Quotation

여러 줄짜리 인용은 새 줄에서 시작하고 들여쓰기로 본문과 구분합니다. 이때 따옴표는 보통 생략합니다 (원문에 따옴표가 있는 경우는 살립니다).

```
선호:
"Wordsworth's enthusiasm for the French Revolution was at first
unbounded:

    Bliss was it in that dawn to be alive,
    But to be young was very heaven!"
```

APA 규칙 (block quotation):
- 40단어 이상 인용 -- 새 줄, 들여쓰기
- 따옴표 없음
- 행간 더블 스페이싱

### 간접 인용 — 따옴표 없음

`that`으로 이끄는 간접 인용은 따옴표를 쓰지 않습니다.

```
선호:
"Keats declares that beauty is truth, truth beauty."
"Dickinson states that a coffin is a small domain."

→ 원문 인용 아님, 그러므로 따옴표 없음
```

### 격언 / 문학적 표현

널리 알려진 격언이나 문학적 표현도 따옴표 없이 씁니다.

```
선호:
"These are the times that try men's souls."
"He lives far from the madding crowd."
"To be or not to be."
```

이런 표현은 영어 문화의 공유 자산입니다. 따옴표가 거꾸로 거만하게 보입니다.

### 한국어 인용의 차이

한국어 인용 부호:
- 큰 따옴표 " " (대화, 직접 인용)
- 작은 따옴표 ' ' (인용 안의 인용, 강조)
- 「 」 (일본식, 한국에서도 사용)
- 《 》 (작품 제목)

영어:
- " " (미국식 큰)
- ' ' (미국식 작은, 영국식 큰)
- 이탤릭 (작품 제목)

영어 인용을 한국어로 옮길 때 부호 변환에 주의하세요.

## References — 참조 / 인용 표기

학술적 글에서 자주 등장하는 작품의 제목은 약어로 표기하고, 약어 일람을 글 끝에 알파벳순으로 정리합니다.

```
선호 (셰익스피어):
"in the second scene of the third act"  →  "in III.ii"
"After the killing of Polonius, Hamlet is placed under guard
(IV.ii.14)."

선호 (성경):
"2 Samuel i:17-27"

선호 (셰익스피어 세부):
"Othello II.iii.264-267, III.iii.155-161"
```

### 현대 학술 인용 스타일

```
APA (심리학, 사회과학):
"Smith (2020) argued that ..."
"Recent research has shown ... (Smith, 2020)."
"In multiple studies (Smith, 2020; Jones, 2021), ..."

MLA (인문학):
"Smith argues that ... (45)."
"... (Smith 45)."

Chicago (역사, 인문학):
"... (Smith 2020, 45)."
또는 각주:
"¹ Smith, History of X, 45."

Vancouver (의학):
"... [1, 2]."
"Smith [1] showed that ..."

IEEE (공학):
"... [1]."
```

자기 분야의 표준 스타일을 확인하고 일관성을 두는 것이 형식의 핵심입니다.

## Syllabication — 음절 분리

줄 끝에서 단어를 분리해야 할 때는 사전에서 음절 구분을 확인합니다.

```
선호:
beau-ti-ful  → 줄 끝에 "beau-" 또는 "beauti-" OK
sep-a-rate  → "sep-arate" 또는 "separa-te" OK

회피:
- 한 음절 단어 분리 ("th-rough", "wor-d")
- 한 글자 분리 ("e-vent", "a-bout")
- 약어 분리 ("US-A", "UK-IP")
```

### 현대 디지털 글

디지털 글에서:
- 자동 줄바꿈 -- 수동 분리 불필요
- 모바일 / 반응형 -- 분리 자체 회피
- PDF 출판 -- 자동 hyphenation 기능 사용

현대는 음절 분리 결정이 덜 중요합니다.

## Titles — 작품 제목

문학 작품의 제목은 보통 이탤릭체 + 단어 첫 글자 대문자로 표기합니다. 출판사와 편집자마다 관습이 약간씩 다릅니다.

```
선호 (이탤릭):
"A Tale of Two Cities"  (이탤릭으로)
"Dickens's Tale of Two Cities"  ← 소유격 앞에서는 A/The 생략
"The Age of Innocence"
"Wharton's Age of Innocence"
```

### 작품 종류별 형식

**이탤릭:**
- 책 (소설, 논픽션)
- 학술지 (저널)
- 영화
- 연극
- TV 시리즈
- 음악 앨범
- 그림 / 조각
- 신문

**따옴표:**
- 단편 소설
- 시 (짧은 것)
- 학술 논문 제목
- 신문 / 잡지 기사
- TV 시리즈의 개별 에피소드
- 노래
- 책의 장 제목

**로마체 (이탤릭 / 따옴표 없음):**
- 성경
- 종교 경전 (Qur'an, Talmud)
- 법률 문서 (US Constitution)
- 일반 명사화된 작품

### 한국어 작품 제목

한국어:
- 책 / 큰 작품: 《 》 또는 『 』 -- 《클린 코드》, 『토지』
- 짧은 작품: 〈 〉 또는 「 」 -- 〈진달래꽃〉, 「운수 좋은 날」
- 영어 / 외국어 작품: 이탤릭 또는 한국 부호 -- *Hamlet* 또는 《햄릿》

## 디지털 / 현대 출판 형식

Strunk 시대(1918)와 현대(2026)의 형식 차이는 다음과 같습니다.

| 시대 | 특징 |
|------|------|
| 타이프라이터 (Strunk) | 핸드 타자, 줄 끝 음절 분리, 더블 스페이싱, 종이 원고 제출 |
| 디지털 (현대) | 워드 프로세서 / LaTeX, 자동 줄바꿈, 1.5 스페이싱, PDF / 온라인 제출, DOI / URL 인용 |

### 인터넷 출처의 인용

```
APA 7판:
Author, A. (Year, Month Day). Title of article.
   Site Name. URL

예시:
Smith, J. (2024, March 15). The future of AI.
   TechReview. https://techreview.com/ai-future
```

### 한국 학술 형식 (현대)

한국연구재단 표준:
- A4 용지
- 글꼴: 신명조 / 함초롬바탕 11pt
- 행간: 160-180%
- 여백: 위 30mm, 아래 25mm, 좌우 25mm
- 인용: 분야 표준 (APA / MLA / IEEE)
- 참고문헌: 분야 표준

자기 학교나 학회의 제출 가이드라인을 확인하는 것이 첫 단계입니다.

## 자기 글 점검 — 형식

- [ ] 구어체에 따옴표를 안 붙였는가?
- [ ] 느낌표가 진짜 감탄·명령에만 있는가?
- [ ] 제목·머리글 형식이 분야 표준인가?
- [ ] 복합 형용사에 하이픈이 맞는가? (명사 앞 위치)
- [ ] 굳어진 합성어가 한 단어인가? (waterfowl, online)
- [ ] 여백·폰트·행간이 제출 가이드에 맞는가?
- [ ] 숫자·풀어 쓰기가 분야 표준인가?
- [ ] 괄호 안과 밖의 구두점이 맞는가?
- [ ] 인용 부호와 attributive 위치가 맞는가?
- [ ] 작품 제목이 이탤릭·따옴표 적절한가?
- [ ] 참고문헌 스타일이 일관되는가?
- [ ] 한국어·영어 인용 부호가 섞이지 않았는가?

## 정리

- 구어체는 따옴표 *없이* 그대로
- 느낌표는 진짜 감탄 · 명령에만
- 머리글 형식, 여백, 들여쓰기의 기본 관습
- 복합 형용사에는 하이픈, 굳어진 합성어에는 한 단어
- 날짜 · 일련번호는 숫자, 대화에서는 풀어 쓰기
- 괄호 안 구두점, 인용의 따옴표 관습
- 참조 약어, 작품 제목의 이탤릭
- 분야 표준 스타일 (APA / MLA / Chicago / Vancouver / IEEE) 일관성
