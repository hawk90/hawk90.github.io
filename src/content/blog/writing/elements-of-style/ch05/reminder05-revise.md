> Revise and rewrite.

처음 쓴 글이 완성형인 작가는 거의 없습니다. 완성된 원고를 다시 들여다보면 거의 항상 *재배치할 자리*가 보입니다.

## 고쳐 쓰기의 종류

**거시적 수정:**
- 단락 순서 바꾸기
- 한 단락을 다른 자리로
- 큰 섹션 통합 / 분할
- 도입과 결론의 위치 재고

**중간 수정:**
- 단락의 첫 문장 재작성
- 단락 안 문장 순서
- 길이 조정
- transition 추가

**미시적 수정:**
- 단어 선택
- 구두점
- 문장 길이
- 어순

각 단계가 *다른 마인드*를 요구합니다.

## 워드 프로세서의 가치

손글씨 시대의 고쳐 쓰기:

1. 첫 초안을 종이에 씀
2. 펜으로 표시
3. 깨끗하게 다시 옮겨 적음 (시간 많이 듦)
4. 다시 표시
5. 또 옮겨 적음

워드 프로세서 시대:

1. 첫 초안 입력
2. 컴퓨터에서 직접 수정 (자르고, 옮기고, 추가)
3. 원본 / 수정본 모두 저장 가능
4. 실험 자유로움

워드 프로세서가 *고쳐 쓰기의 비용*을 극적으로 낮췄습니다. 이 도구를 *사용하지 않는 것*은 손해입니다.

## 원본과 수정본 함께 저장

자주 실수하는 것 — *원본을 덮어쓰는 것*. 그러면 *이전 표현이 더 좋았는지* 비교가 불가능합니다.

Better practice:
- 매 큰 수정마다 *별도 파일* 저장
- v1, v2, v3 등으로 버전 관리
- 또는 Git으로 관리 (블로그 글에도 가능)

이렇게 하면 *수정의 흔적*이 남고, 필요하면 *되돌리기*가 가능합니다.

## 대수술의 빈도

원고가 *대수술*에 들어가는 것이 *약함의 신호*가 아닙니다.

가장 좋은 글에서도 흔히 일어나는 일:

- 첫 초안에서 30-50% 잘림
- 단락 순서가 완전히 바뀜
- 도입과 결론이 통째로 다시 쓰임
- 핵심 메시지가 *수정 중*에 발견됨

좋은 작가는 *수정을 두려워하지 않습니다*. 첫 초안에 매이지 않습니다.

## 고쳐 쓰기의 횟수

| 글 유형 | 수정 횟수 |
|---------|-----------|
| 일기 / 메모 | 0회 (그대로 OK) |
| 이메일 | 1-2회 |
| 블로그 글 | 2-3회 |
| 에세이 | 3-5회 |
| 연구 논문 | 5-10회 (peer review 포함) |
| 책 | 10회 이상 |

글의 *야망과 수명*에 비례해 수정 횟수가 늘어납니다.

## 시간 두기

고쳐 쓰기 사이에 *시간을 두는 것*이 중요합니다.

1. 첫 초안 작성
2. 24시간 대기 (또는 그 이상)
3. 다시 읽기 — 어색한 자리가 *분명히* 보임
4. 수정
5. 다시 시간 두기
6. 또 수정

자기 글에 너무 가까이 있을 때는 *문제가 보이지 않습니다*. 거리를 두어야 객관성이 생깁니다.

## 외부 reader

자기가 자기 글의 마지막 reader가 되면 안 됩니다.

외부 reader의 가치:

- 자기는 안 보이는 어색함 발견
- 정보가 부족한 자리 발견
- 흐름이 끊긴 자리 식별
- 자기 가정이 *공유되지 않음* 인식

좋은 reader를 *한두 명* 두는 것이 좋은 작가의 핵심 자원입니다.

## 한국어 글에서의 고쳐 쓰기

한국어 글에서 자주 등장하는 *수정 패턴*:

- 번역체 정리 ("~의 ~의 ~", "있어왔다" 등)
- 영문 구조 → 한국어 자연 어순
- 너무 긴 문장 → 두세 문장으로 분할
- 외래어 → 한국어 단어로 (가능한 경우)
- 강조 부사 (매우, 정말, 진짜) 제거

이런 수정이 한국어 글을 *한국어다운 글*로 만듭니다.

## 적용

- [ ] 매 글마다 *최소 2회* 수정
- [ ] 수정 사이에 *24시간 이상* 시간
- [ ] *원본 별도 저장*
- [ ] 외부 reader 한 명 정함
- [ ] 큰 수정도 *두려워하지 않기*
