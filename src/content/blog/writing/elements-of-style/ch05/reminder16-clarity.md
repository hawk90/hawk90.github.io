> Be clear.

명료함이 글의 *최고 미덕*은 아닙니다. 어떤 작가는 *의도적 모호함*을 추구합니다. 그러나 글은 *전달*이므로 명료함은 거의 항상 미덕입니다.

## 명료함의 자리

```
명료함이 *필수*인 글:
- 학술 논문
- 비즈니스 문서
- 기술 매뉴얼
- 뉴스 기사
- 교과서

명료함이 *희생되어도 좋을 자리*:
- 시
- 일부 문학 (의도된 효과)
- 광고 (감성적 영향)
```

대부분의 글에서 *명료함*은 첫 번째 미덕입니다.

## 의도적 모호함

어떤 작가는 의도적으로 모호하게 씁니다.

```
T.S. Eliot의 시
Borges의 단편소설
Kafka의 산문
James Joyce의 *Ulysses*
```

그러나 White의 통찰:

> 의도적 모호함을 *추구*하는 작가도 *명료하게 모호*해야 합니다.

*무엇이* 모호한지는 *명료*해야 합니다. 작가가 *무엇을 안 알리려*는지, *어떤 효과*를 원하는지는 분명해야 합니다. 그래야 독자가 *모호함을 즐길 수 있습니다*.

## 문장 안에서 길을 잃을 때

자기 글을 쓰다 문장이 *너무 복잡*해진 자리. 흔합니다.

```
원본 (길을 잃음):
The phenomenon, which was discovered in 1995 by a researcher
working in collaboration with a team at the institute that was at
the time supported by funding from agencies that have since
restructured, suggests that ...
```

길을 잃었을 때 *가장 좋은 해결*은:

> 새로 시작합니다.

망가진 구조와 *싸우지 마세요*. 보통은 문장이 *너무 복잡해진 것*이 원인입니다. 두세 문장으로 쪼개세요.

```
수정 (쪼갬):
The phenomenon was discovered in 1995. A researcher working with
the institute team made the discovery. The team was funded by
agencies that have since restructured. The phenomenon suggests
that ...
```

같은 정보지만 *읽기 쉽게* 됩니다.

## 명료함은 *삶의 미덕*

White가 가장 깊이 쓰는 단락:

> 명료함은 단순히 글의 미덕이 아니라 *삶의 미덕*입니다. 길에서의 잘못 쓰인 표지판, 잘못 표현된 편지로 비롯된 연인 사이의 비참함, 잘못 배달된 전보로 인한 기차역에서의 고독 — 이 모든 비극이 *모호함*에서 옵니다.

명료함은 *글쓰기 기술*만이 아닙니다. *소통의 윤리*입니다. 자기 글이 흐릿하면 다른 사람의 *시간과 정서*가 손해를 봅니다.

## 명료함의 도구

```
1. 단어 — 평범하고 정확한 단어
2. 문장 — 너무 길지 않게
3. 단락 — 한 주제
4. 구조 — 도입 / 본문 / 마무리
5. 연결 — signalling language
```

이 다섯 단계 모두에 *명료함의 결정*이 있습니다.

## 명료함을 *깨뜨리는* 흔한 함정

```
- 너무 긴 문장 (50단어 이상)
- 모호한 대명사 ("this", "that"이 무엇을 가리키는가)
- 누락된 정보 (가정된 지식)
- 잘못된 어순
- 중복 / 군더더기
- 헤지 남발
- jargon
```

각 함정에 대한 *수정 방법*은 다른 Rule / Reminder에서 다룹니다.

## 명료함 vs 단순함

```
단순함 (simplicity) ≠ 명료함 (clarity)

단순한 글이 모호할 수 있음.
복잡한 글이 명료할 수 있음.

좋은 글의 목표는 — *그 주제에 필요한 정도의 복잡성*으로,
*명료하게* 쓰는 것.
```

복잡한 과학을 다룬다면 글이 단순할 수 없습니다. 그러나 *복잡한 글도 명료할 수 있습니다*.

## 자기 글 점검 — 명료함

```
□ 외부 reader에게 보여 "어디가 헷갈리는가" 묻기
□ 자기 글을 *큰 소리로 읽기* — 막히는 자리가 모호한 자리
□ 한 단락에 *한 주제*인가?
□ 문장이 *50단어를 넘지 않는가*?
□ 대명사가 *명확한 대상*을 가리키는가?
□ 가정된 지식이 있다면 *간단히 설명*했는가?
```
