> **On Writing Well** Chapter 15 요약

## 이 챕터의 메시지

과학 / 기술 글은 어렵다. **추상**과 **전문 용어**가 가득.

> **핵심 원칙 — 한 단계씩 (step by step).**

A에서 B로, B에서 C로. 점프하지 마라.

## "Logical chain"

좋은 과학 글의 구조.

```
1. 기본 개념 (모두 알 만한)
2. 그 위에 새 개념
3. 그 위에 또
4. ...
```

각 단계가 — 이전 단계를 전제로. 점프하면 — 독자가 떨어진다.

```
❌ "Quantum entanglement requires understanding tensor products..."
   (사전 지식 가정)

✅ "First, imagine two coins. If you flip them, ... [점진적]"
```

## 비유 (Analogy)

추상 → 구체로 변환.

```
"DNA는 책의 페이지처럼..."
"원자는 태양계처럼..."
"메모리는 책상처럼..."
```

좋은 비유 — 독자가 이미 아는 것에서 출발.

다만 — 비유의 한계도 명시. "단, X는 책과 달리..." 같이.

## 전문 용어 (Jargon)

**정확히 정의 후 사용**.

```
✅ "Compile (the process of translating source code to machine code)..."
✅ "Compile" — 정의 후 — 자유롭게 사용
```

정의 없이 jargon — 일반 독자 떨어짐. 같은 분야 독자라도 — 정의가 도움.

## 모방할 작가

좋은 과학 글의 거장.

- **Carl Sagan** — *Cosmos* — 천문학 / 인생
- **Stephen Jay Gould** — 진화생물학
- **Oliver Sacks** — 신경학 / 환자 이야기
- **Richard Feynman** — 물리학 강의

이들의 공통점 — **인간적 voice**, **구체적 사례**, **단계적 설명**.

## "Who" — 사람을 잊지 마라

과학 / 기술 글에 — 종종 사람이 사라진다.

```
❌ "The phenomenon was discovered in 1995."
✅ "In 1995, a researcher named Alice Chen discovered..."
```

사람이 들어가면 — 추상이 구체화. 독자가 따라옴.

## 정리

- 과학 / 기술 = **단계적 설명**
- **비유**로 추상을 구체로
- 전문 용어 — **정의 후 사용**
- 사람을 잊지 마라 — 누가 발견 / 누가 영향
- Carl Sagan, Oliver Sacks 등 모범
