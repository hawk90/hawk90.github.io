과학과 기술 글은 어렵습니다. *추상*과 *전문 용어*가 가득합니다.

> 핵심 원칙은 *한 단계씩 (step by step)*입니다.

A에서 B로, B에서 C로. 점프하지 않습니다.

## Logical Chain

좋은 과학 글의 구조입니다.

```
1. 기본 개념 (모두가 알 만한)
2. 그 위에 새 개념
3. 그 위에 또 새 개념
4. ...
```

각 단계가 이전 단계를 *전제*로 합니다. 점프하면 독자가 떨어집니다.

```
회피: Quantum entanglement requires understanding tensor products ...
       (사전 지식을 가정)

선호: First, imagine two coins. If you flip them, ... [점진적]
```

## 비유 (Analogy)

추상을 구체로 변환하는 도구입니다.

```
"DNA는 책의 페이지처럼 ..."
"원자는 태양계처럼 ..."
"메모리는 책상처럼 ..."
```

좋은 비유는 *독자가 이미 아는 것*에서 출발합니다. 다만 비유의 한계도 명시합니다 — *"단, X는 책과 달리 ..."* 같이.

## 전문 용어 (Jargon)

*정확히 정의한 뒤*에 사용합니다.

```
선호: Compile (the process of translating source code to machine code) ...
       이후 "compile"을 자유롭게 사용
```

정의 없이 jargon을 쓰면 일반 독자가 떨어집니다. 같은 분야 독자라도 정의가 도움이 됩니다.

## 모방할 작가들

좋은 과학 글의 거장들:

- **Carl Sagan** — *Cosmos*, 천문학과 인생
- **Stephen Jay Gould** — 진화생물학
- **Oliver Sacks** — 신경학, 환자 이야기
- **Richard Feynman** — 물리학 강의

이들의 공통점은 *인간적 voice*, *구체적 사례*, *단계적 설명*입니다.

## "Who" — 사람을 잊지 마세요

과학과 기술 글에는 종종 사람이 사라집니다.

```
회피: The phenomenon was discovered in 1995.
선호: In 1995, a researcher named Alice Chen discovered ...
```

사람이 들어가면 추상이 구체화되고 독자가 따라옵니다.

## 정리

- 과학과 기술 글은 *단계적 설명*
- *비유*로 추상을 구체로
- 전문 용어는 *정의 후 사용*
- 사람을 잊지 말 것 — 누가 발견했는가
- Carl Sagan, Oliver Sacks 등 모범
