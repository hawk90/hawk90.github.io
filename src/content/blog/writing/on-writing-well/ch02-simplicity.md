Zinsser의 가장 유명한 문장이 이 챕터에 있습니다.

> The secret of good writing is to strip every sentence to its cleanest components.

*좋은 글쓰기의 비밀은 모든 문장을 가장 깨끗한 구성 요소로 벗기는 것*. 이 한 줄이 책 전체의 한 줄 요약입니다. Strunk와 White의 *Omit needless words*가 한 차원 더 발전한 형태입니다.

## 미국 산문의 위기

Zinsser가 1970년대 미국 산문의 상태를 진단합니다.

- 비즈니스 보고서는 이해할 수 없습니다
- 정부 문서는 의도적으로 흐릿합니다
- 학술 논문은 가식적입니다
- 광고는 거짓의 어조를 가집니다

이 모든 문제의 공통 원인은 *단순함에 대한 두려움*입니다. 작가들은 단순한 글이 *어리석어 보일까* 걱정하고, 복잡한 글이 *더 똑똑해 보일 것*이라고 가정합니다.

## 잘못된 가정

복잡한 글은 똑똑한 글이 아닙니다. 오히려 반대입니다. 정말 명확하게 생각하는 사람만이 단순하게 쓸 수 있습니다.

> Clear thinking becomes clear writing.

명확한 사고가 명확한 글이 됩니다. 흐릿한 사고는 흐릿한 글로 드러납니다. 복잡한 문장은 보통 작가 자신이 무엇을 말하려는지 모른다는 신호입니다.

## 두 종류의 어려움

복잡함의 두 원천이 있습니다.

### 1. 주제가 본질적으로 복잡한 경우

양자역학, 분산 시스템, 법률처럼 *본질이 복잡한* 주제가 있습니다. 이 경우 어쩔 수 없이 복잡합니다. 그러나 *가능한 한 단순하게* 쓰는 것이 좋은 작가의 책임입니다.

### 2. 작가가 만든 복잡함

단순한 내용을 가식적 어휘로 부풀린 경우. 이것이 진짜 문제입니다.

| 회피 | 바름 |
|------|------|
| We conducted an investigation into the matter. | We investigated. |
| It is currently our utilization of ... | We use ... |

대부분의 나쁜 글은 두 번째 종류입니다.

## 단순함은 노력의 결과

쉬워 보이지만 어렵습니다. 첫 초안은 거의 항상 부풀어 있습니다. 단순화는 *수정*의 산물입니다. 좋은 글은 다섯 번, 열 번 수정된 글입니다.

> Writing improves in direct ratio to the number of things we can keep out of it.

*글은 우리가 빼낼 수 있는 것의 양에 비례해 좋아집니다*.

## "Decide what you want to say"

단순함의 첫 단계는 *결정*입니다.

- 무엇을 말할 것인가
- 무엇을 말하지 않을 것인가
- 가장 핵심이 되는 한 가지는 무엇인가

이 결정이 서면 어떤 단어를 쓸지가 자연스럽게 정해집니다. 결정이 없으면 글이 흐려집니다.

## 흔한 흐림의 원천

Zinsser가 든 짝들입니다. Strunk와 Williams의 유사한 목록과 겹칩니다.

| 회피 | 바름 |
|------|------|
| at this point in time | now |
| for the purpose of | to / for |
| in light of the fact that | since |
| due to the fact that | because |
| has the ability to | can |
| make a recommendation | recommend |
| utilize | use |
| implement | do / carry out |
| facilitate | help |
| in the event that | if |

모두 짧은 단어가 더 강합니다.

## 단순함의 적 — Jargon

전문 분야 용어가 단순함의 가장 큰 적입니다.

- 의사, 변호사, 학자가 자기 분야 용어로 일반 독자를 막습니다
- 비즈니스 jargon — *synergy*, *leverage*, *deliverable*
- 기술 jargon — 같은 의미를 어려운 단어로

좋은 작가는 jargon을 정확히 필요한 자리에만 쓰고, 처음 등장할 때는 풀어 설명합니다.

## "Less is more"

건축가 Mies van der Rohe의 원칙을 Zinsser는 글쓰기에 적용합니다.

> 긴 문장 / 어려운 단어 / 가식적 어조
> ↓
> 짧은 문장 / 평범한 단어 / 자연스러운 어조

이 방향이 거의 항상 더 좋은 글을 만듭니다.

## 단순함이 어려운 이유

우리는 본능적으로 *복잡하게* 쓰고 싶어 합니다.

- 학교 작문이 가르친 습관
- 직장 문서의 패턴
- *전문가답게 보이려는* 욕구

이 본능을 거스르려면 의식적 노력이 필요합니다.

## 자기 글의 단순화 점검

다음 다섯 가지가 단순화의 기본입니다.

1. 긴 단어를 짧은 단어로
2. 여러 단어 표현을 한 단어로
3. 불필요한 수식어를 제거
4. 수동태를 능동태로
5. 추상을 구체로

## "Editing" — 글의 90퍼센트

Zinsser의 통찰 — 좋은 글의 90퍼센트는 *편집*입니다.

1. **1차 초안**: 자유롭게 쏟아 내기
2. **2차**: 30퍼센트 잘라내기
3. **3차**: 다시 20퍼센트 잘라내기
4. **4차**: 단어와 구두점 다듬기

매 단계마다 단순해집니다. 최종 글은 첫 초안의 절반 정도가 됩니다.

## 단순함은 정직

복잡한 글은 종종 *자기가 이해하지 못한 것*을 가립니다.

- 어려운 단어로 쓰면 → 아는 척이 가능
- 짧고 명확하게 쓰면 → 모르는 부분이 드러남

따라서 명확하게 쓰는 것은 *정직하게 쓰는 것*과 같습니다.

## 한국어 글의 단순화

한국어 글에도 Zinsser 단순화가 그대로 적용됩니다.

| 회피 | 바름 |
|------|------|
| ~에 있어서 | ~에서, 에 |
| ~을 통해서 | ~로 |
| ~한다는 것이다 | ~한다 |
| ~할 수 있을 것으로 보인다 | ~할 것이다 |
| 주지하다시피 | 보통 제거 |
| 역사적으로 볼 때 | 역사적으로 |

한국어 jargon:
- 학술 용어 과시 (인지부조화 → 마음의 갈등)
- 경영 영어 (game changer, paradigm shift)
- 한자어 과시 (cognitive load → 인지 부담)

## 자기 글 점검

- [ ] 모든 문장이 *가장 깨끗한 구성*인가?
- [ ] Jargon이 *필요한 자리*에만 있는가?
- [ ] 자기 글을 *수정*하는 데 시간을 쓰는가?
- [ ] 한국어 글에 *번역체 군더더기*가 없는가?
- [ ] 어려운 한자어를 *쉬운 한국어*로 바꿀 수 있는가?

## 정리

- 좋은 글은 *모든 문장을 가장 깨끗한 구성으로*
- 복잡한 글은 똑똑한 글이 아니라 *흐릿한 사고의 결과*
- 단순함은 *노력의 결과* — 자연스럽지 않음
- *결정*이 단순화의 시작
- Jargon은 단순함의 가장 큰 적
- *편집*이 글의 90퍼센트
- 단순함은 *정직*
