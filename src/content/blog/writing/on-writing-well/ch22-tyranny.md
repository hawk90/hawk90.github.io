학교와 직장이 가르치는 것은 *최종 결과물(final product)이 중요*하다는 것입니다. 시험에서 답안지가 평가됩니다. 직장에서 발표 자료가 평가됩니다. 과정은 *보이지 않습니다*.

Zinsser는 이 가정을 뒤집습니다.

> Process가 중요합니다. Final product는 결과일 뿐입니다.

좋은 글은 *좋은 process*에서 나옵니다.

## "The tyranny of the final product"

완벽한 결과에 대한 강박은 작가를 *마비*시킵니다.

```
"이 글이 완벽해야 해."
→ 손이 안 움직임
→ 첫 줄을 못 씀
→ 화면이 비어 있음
→ 시간이 흘러감
→ 더 압박감
→ 더 못 씀
```

이것이 *폭정(tyranny)*입니다. 완벽함이 *시작 자체*를 막습니다.

### 학생의 작가 차단(writer's block)

작가 차단의 흔한 원인은 *지식 부족*이 아닙니다. *완벽주의*입니다.

```
완벽주의 작가:
"이 첫 문장이 완벽하지 않으면 ..."
"이 단어가 정확하지 않으면 ..."
"이 구조가 맞지 않으면 ..."
→ 결국 글이 안 시작됨

좋은 작가:
"일단 쓰자. 어차피 고칠 거다."
→ 첫 draft가 빠르게 나옴
→ 수정에서 좋아짐
```

## 해법 — Process 우선

```
1. 자유롭게 첫 초안 (perfectionism 끄기)
2. 비판적으로 수정 (perfectionism 켜기)
3. 다듬기 (세부 수정)
4. 발표 (또는 보류)
```

각 단계가 *다른 마인드*를 요구합니다. *동시에 하지 않습니다*.

### 1단계 — 자유로운 첫 초안

```
- 문법 신경 안 씀
- 단어 선택 고민 안 함
- 한 단락이 어색해도 다음으로 진행
- 후에 자를 부분도 일단 다 씀
- "이거 쓰레기야"라는 생각 무시
```

이 단계에서는 *내면의 비평가를 끕니다*. 비평가가 첫 단계에서 활동하면 글이 시작되지 않습니다.

### 2단계 — 비판적 수정

이제 비평가를 *켭니다*.

```
- 전체 구조 점검
- 흐름이 끊긴 자리 식별
- 너무 길거나 짧은 부분 조정
- 부족한 디테일 추가
- 과한 디테일 잘라냄
```

이 단계에서 *50%까지 잘라내는 것*이 흔합니다.

### 3단계 — 다듬기

```
- 단어 하나하나 점검
- 문장 길이 다양화
- 단락 첫 문장 정리
- 구두점 점검
- 인용 / 참조 확인
```

세부적 작업입니다. 거시적 변경은 이 단계에서 거의 없습니다.

### 4단계 — 발표 (또는 보류)

```
- 24시간 두고 다시 보기
- 외부 reader에게 보이기
- 피드백 반영
- 최종 점검
- 발표
```

## "Bad First Draft"의 가치

좋은 작가는 *나쁜 첫 초안*을 환영합니다.

```
Anne Lamott (in Bird by Bird):
   "Shitty first drafts."

Hemingway:
   "The first draft of anything is shit."
```

첫 초안의 *임무*는 단순합니다 — *존재하는 것*. 좋은 것이 아닙니다. 수정에서 좋아집니다.

### 첫 초안의 함수

```
첫 초안 = 진흙 한 덩어리
수정 = 그 진흙으로 조각하기

진흙이 없으면 조각도 없음.
```

학생이 *완벽한 첫 초안*을 만들려는 것은 *조각용 진흙이 완벽하게 모양 잡혀 있기*를 바라는 것과 같습니다.

### "쓰레기 같은 글"을 두려워하지 말 것

자기 첫 초안이 *쓰레기*처럼 느껴지면 그건 *정상*입니다. 모든 작가가 그렇게 느낍니다.

```
유명 작가들이 자기 첫 초안에 대해 한 말:
- 무엇이 살아 있는지 모르겠다
- 부끄럽다
- 정말 못 썼다
- 이게 나의 best였나
```

그래도 그들은 *수정*해서 출판합니다.

## 학교가 만든 그릇된 가르침

학교는 *제출용 글만 평가*합니다. 그러나 가르쳐야 할 것이 더 있습니다.

```
보이지 않는 것:
- Process 자체
- 수정의 가치
- 실패한 시도의 의미
- 한 글을 5번 수정한 작가의 경험
- 막힌 글을 풀어낸 순간
```

학교가 *결과만 평가*하는 결과 — 학생들이 *완벽한 첫 시도*를 목표로 합니다. 이것이 평생 작가의 가장 큰 함정이 됩니다.

### 좋은 작가의 진실

> 좋은 작가는 *완성품*이 아니라 *일하는 과정*입니다.

좋은 작가는 *매일 일합니다*. 완벽한 영감을 기다리지 않습니다. 매일의 작업이 *습관이 되고 기술이 됩니다*.

## 작가는 평생 학생

```
모든 위대한 작가도 — 다음 글에 대해 두려워합니다.
모든 위대한 작가도 — 매번 새로 배웁니다.
모든 위대한 작가도 — 자기 글을 부끄러워합니다.
```

이 사실이 *위안*입니다. 자기만 어렵다고 생각하지 마세요. *모두가 어렵습니다*.

### 작가의 진짜 차이

```
약한 작가:        강한 작가:
- 완벽함을 기다림  - 시작함
- 한 번에 좋게      - 여러 번 수정
- 결과만 신경       - 과정을 즐김
- 영감을 기다림      - 매일 일함
- 자기 비교         - 자기 발전 비교
```

## 한국 글쓰기 환경에서

한국의 학교 / 직장 / 학술계도 *결과 평가의 폭정*을 가집니다.

```
- 대학 시험 — 답안지만 평가
- 직장 — 발표 슬라이드의 디자인
- 학술 — 출판된 논문만 평가
```

그래서 *수정의 습관*을 배우기 어렵습니다. 자기 글을 *5번, 10번 다시 쓰는 경험*이 드뭅니다.

좋은 작가가 되려면 *이 환경을 의식적으로 거슬러야* 합니다. 자기에게 *시간*을 주고, *나쁜 초안*을 허용하고, *수정을 즐기는* 습관을 만들어야 합니다.

## 자기 점검 — Process

```
□ 첫 초안을 *자유롭게* 쓰는가? 첫 문장에서 막히지 않는가?
□ 첫 초안 후 *시간을 두고* 수정하는가? (24시간 이상)
□ 수정을 *여러 번* 하는가? (2-5번)
□ 자기 글을 *외부 reader*에게 보이는가?
□ 다른 작가의 *수정 과정*을 보는가? (책의 후기, 인터뷰 등)
```

## 정리

- *Final product의 폭정*이 작가를 마비시킴
- *Process*가 중요 — 4 단계 (자유 / 비판 / 다듬기 / 발표)
- 각 단계는 *다른 마인드*
- *Bad first draft*는 좋은 출발점
- 학교의 결과 중심 평가는 작가를 *망친다*
- 모든 작가가 매번 *새로 배움*
- 한국 환경에서 *의식적으로 수정 습관*을 만들기
