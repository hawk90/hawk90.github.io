> **초안** — 정리 진행 중

## 개요

C++는 하나의 통일된 언어가 아니라 **서로 다른 규칙을 가진 네 가지 하위 언어의 연합체**입니다. 어느 영역에서 일하느냐에 따라 효율적이고 자연스러운 코딩 방식이 달라집니다.

## 네 가지 하위 언어

1. **C** — 블록, 문장, 전처리기, 내장 타입, 배열, 포인터. C++는 C에 기반을 두지만 이 부분만 사용하면 OOP, 템플릿, 예외의 장점을 잃음.

2. **객체 지향 C++** — 클래스, 캡슐화, 상속, 다형성, 가상 함수. 전통적인 OOP 규칙이 그대로 적용.

3. **템플릿 C++** — 제네릭 프로그래밍 영역. 템플릿 메타프로그래밍(TMP)은 또 다른 패러다임.

4. **STL** — 컨테이너, 반복자, 알고리즘, 함수 객체. 자체 규약이 있어 따로 익혀야 함.

## 왜 중요한가

각 영역마다 "효율적인 방법"이 다릅니다.

```cpp
// C 영역: pass-by-value가 효율적 (int)
void f(int x);

// OOP 영역: pass-by-reference-to-const가 효율적 (큰 객체)
void g(const std::string& s);

// 템플릿 영역: pass-by-reference-to-const가 보통 효율적
template<typename T>
void h(const T& x);

// STL 영역: 반복자/함수 객체는 pass-by-value가 효율적 (가벼움)
std::for_each(v.begin(), v.end(), Functor());
```

같은 "어떻게 매개변수를 전달할까"라는 질문에도 영역에 따라 답이 다름.

## 핵심 정리

1. C++는 네 가지 하위 언어의 연합 — C, OO C++, 템플릿 C++, STL
2. 효율적 코딩 규칙은 영역마다 다름
3. 한 영역의 관습을 다른 영역에 그대로 적용하면 안 됨
