## 세 요소의 상호작용

### Context Map × Distillation

## 전략적 의사결정 프레임워크

### 1. 조직 관점

```
┌─────────────────────────────────────────────────────────┐
│                   조직 관점 결정                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Q: 팀 구조는 어떻게 되는가?                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │  팀 구조              →  Context 경계           │   │
│  │                                                  │   │
│  │  • 팀당 하나의 Bounded Context 권장              │   │
│  │  • Conway's Law: 조직 구조 = 시스템 구조         │   │
│  │  • 팀 간 관계 = Context 간 관계 (Context Map)    │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  Q: 팀 역량은 어떠한가?                                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │  역량 수준            →  도메인 배치             │   │
│  │                                                  │   │
│  │  • 시니어 팀 → Core Domain                       │   │
│  │  • 주니어 팀 → Supporting / Generic             │   │
│  │  • 외주 팀 → Generic (ACL로 보호)               │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  Q: 팀 간 협력은 어떻게 하는가?                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  협력 방식            →  통합 패턴               │   │
│  │                                                  │   │
│  │  • 긴밀한 협력 → Shared Kernel, Partnership     │   │
│  │  • 협상 가능 → Customer-Supplier                │   │
│  │  • 협력 어려움 → Conformist, ACL               │   │
│  │  • 불가능 → Separate Ways                       │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```
│  │  • Core Domain → 반드시 직접 구축               │   │
│  │  • Supporting → 상황에 따라                     │   │
│  │  • Generic → 구매 또는 오픈소스                 │   │
│  └─────────────────────────────────────────────────┘   │

└─────────────────────────────────────────────────────────┘
```

## 전략적 설계 체크리스트

```python
# 전략적 설계 자가 진단
class StrategicDesignAssessment:
    """
    프로젝트의 전략적 설계 수준 평가

    각 항목에 0-5점 부여
    """

    questions = {
        # Core Domain
        "core_domain": [
            "Core Domain이 명확히 정의되어 있는가?",
            "Core Domain에 최고의 인재가 배치되어 있는가?",
            "Core Domain의 코드 품질이 최고 수준인가?",
            "Domain Vision Statement가 존재하는가?",
        ],

        # Context Map
        "context_map": [
            "Bounded Context가 명확히 식별되어 있는가?",
            "Context 간 관계가 문서화되어 있는가?",
            "적절한 통합 패턴이 선택되어 있는가?",
            "Context Map이 최신 상태로 유지되는가?",
        ],

        # Large-Scale Structure
        "large_scale_structure": [
            "시스템 전체를 설명하는 구조가 있는가?",
            "구조가 팀 전체에 이해되고 따르는가?",
            "구조가 도메인과 함께 진화하는가?",
            "새 기능의 위치를 쉽게 결정할 수 있는가?",
        ],

        # Ubiquitous Language
        "ubiquitous_language": [
            "개발자와 도메인 전문가가 같은 언어를 쓰는가?",
            "코드가 도메인 언어를 반영하는가?",
            "용어집이 존재하고 유지되는가?",
            "새 팀원이 언어를 빠르게 습득할 수 있는가?",
        ],

        # Model Quality
        "model_quality": [
            "모델이 비즈니스 규칙을 명확히 표현하는가?",
            "모델이 쉽게 변경 가능한가 (Supple Design)?",
            "암묵적 개념이 명시적으로 표현되어 있는가?",
            "Aggregate 경계가 적절한가?",
        ],
    }

    def assess(self, scores: dict) -> dict:
        """평가 결과 및 권장 사항 반환"""
        results = {}

        for category, questions in self.questions.items():
            category_scores = scores.get(category, [])
            avg = sum(category_scores) / len(questions) if category_scores else 0

            if avg < 2:
                priority = "HIGH"
                recommendation = f"{category} 영역 개선 필요"
            elif avg < 4:
                priority = "MEDIUM"
                recommendation = f"{category} 영역 강화 권장"
            else:
                priority = "LOW"
                recommendation = f"{category} 영역 양호"

            results[category] = {
                "score": avg,
                "priority": priority,
                "recommendation": recommendation
            }

        return results
```

**시리즈 완결**

이 시리즈는 Eric Evans의 "Domain-Driven Design"을 기반으로, C++와 Python 예제를 통해 화물 해운 시스템 도메인에서 DDD의 핵심 개념을 탐구했다.

DDD는 단순한 기술이 아니라 **사고방식**이다. 비즈니스를 깊이 이해하고, 그 이해를 코드로 표현하며, 팀 전체가 같은 언어로 소통하는 것. 이것이 DDD의 본질이다.
