## 이 팁의 메시지

> **English is Just Another Programming Language** — 자연어 작문도 — **코드 짜듯 정성껏**.

## 핵심 내용

- 문서·이메일·메시지 — 모두 코드처럼 짜라.
- DRY, 명확함, 의도 표현 — 자연어에도 적용.
- 글이 — **자기와 팀의 도구**.
- 글 못 쓰는 사람 = 영향력 작음.

## 코드와 글의 공통점

- **명확함** — 의도가 전달되는가?
- **구조** — 흐름이 있는가?
- **이름** — 단어 선택이 정확한가?
- **DRY** — 같은 내용 반복 X.
- **리뷰** — 보내기 전 다시 본다.

## 영어가 강조되는 이유

이 책은 영어권 독자에게 — 글쓰기의 중요성을 일깨운다. 한국 독자에게도 같은 메시지. **한국어 글도 — 또 하나의 프로그래밍 언어**.

## 작은 실천

- 이메일 — 보내기 전 5초 다시 본다.
- 커밋 메시지 — 정성껏.
- PR 설명 — 의도부터.
- 문서 — 첫 줄로 — "이 문서가 어떤 도움이 되는가" 말한다.

## 정리

- 자연어 = 프로그래밍 언어.
- 코드의 원칙(명확, DRY, 리뷰)이 — 글에도 적용.
- 글 = 영향력의 도구.
