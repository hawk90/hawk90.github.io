## 이 팁의 메시지

> **Prototype to Learn** — 프로토타입은 — **버리는** 것. 학습이 목적.

## 핵심 내용

- 프로토타입 ≠ 살아남는 코드.
- 위험·불확실한 자리만 — 시도.
- 다 마치면 — **폐기**.
- 폐기 못 하면 — 프로토타입 아님.

## 무엇을 프로토타이핑?

- **새 알고리즘** — 성능 검증.
- **새 라이브러리** — API 적합?
- **새 사용자 인터페이스** — 흐름.
- **위험한 자리** — 통합·외부 API.

전체 시스템을 — 프로토타입할 필요 X. 한 점만.

## 폐기의 중요성

> "이 프로토타입 좋네, 그대로 쓰자!"

이 말이 — 가장 흔한 함정. 프로토타입은 — **에러 처리·테스트·문서**가 없다. 그대로 쓰면 — 부채.

## 정리

- 프로토타입 = 학습 도구, 폐기 전제.
- 위험·불확실한 자리만.
- 폐기 못 하면 — 프로토타입 아님.
