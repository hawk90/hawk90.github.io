> **Hackers and Painters** Essay 5 요약

## 핵심 주장

**웹 기반 소프트웨어**가 데스크톱 소프트웨어를 대체할 것이다. 이것은 소프트웨어 산업의 근본적인 변화.

## 데스크톱 vs 웹

### 데스크톱의 문제

전통적 소프트웨어의 한계:
- 설치 필요
- 업데이트 복잡
- 플랫폼 종속
- 배포 비용 높음

### 웹의 장점

웹 기반 소프트웨어:
- 설치 불필요 — 브라우저만
- 자동 업데이트
- 어디서나 접근
- 배포 비용 거의 0

## Viaweb의 경험

Graham은 Viaweb(최초의 웹 기반 이커머스 플랫폼) 창업자.

### 지속적 배포

- 하루에도 여러 번 릴리스
- 버그 즉시 수정
- 기능 점진적 추가
- **사용자 피드백 → 즉각 반영**

### 한 버전만 유지

- 모든 사용자가 같은 버전
- 호환성 문제 없음
- 지원 단순화

## 웹 소프트웨어의 이점

### 개발자에게

1. **빠른 반복**: 배포가 쉬우므로 실험 가능
2. **직접 피드백**: 사용자 행동 실시간 관찰
3. **자유로운 기술 선택**: 서버 측 기술은 자유

### 사용자에게

1. **항상 최신**: 업데이트 신경 안 써도 됨
2. **어디서나**: 인터넷만 있으면
3. **데이터 안전**: 로컬보다 오히려 안전할 수 있음

### 비즈니스에게

1. **낮은 진입 장벽**: 물리적 배포 불필요
2. **구독 모델**: 지속적 수익
3. **글로벌 시장**: 즉시 전 세계 접근

## 기술적 고려

### 서버 아키텍처

웹 앱 운영의 핵심:
- 가용성 (uptime)
- 확장성 (scalability)
- 보안 (security)

### 언어 선택

서버에서는 **좋은 언어** 선택 가능:
- 사용자 PC 환경에 종속 안 됨
- 표현력 있는 언어 선택
- Lisp (Graham의 선택)

## 미래 예측 (2001년 시점)

Graham의 예측:
- 데스크톱 앱 → 웹 앱 이동
- MS Office 같은 것도 웹으로
- 브라우저가 OS가 됨

> **결과**: 대부분 맞았다. Google Docs, Office 365, 모든 SaaS.

## 시사점

1. **패러다임 전환**: 데스크톱 → 웹
2. **지속적 배포**: 릴리스 주기 단축
3. **사용자 중심**: 피드백 기반 개발
4. **기술 자유**: 서버에서는 좋은 도구 선택
