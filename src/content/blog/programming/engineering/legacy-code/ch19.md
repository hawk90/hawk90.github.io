> **Working Effectively with Legacy Code** Chapter 19 요약

## 이 챕터의 메시지

C / Pascal / FORTRAN — 객체 지향이 아닌 코드도 Legacy.

```
객체 seam 사용 못 함
→ Link seam / Preprocessing seam이 주역
```

## Link Seam

같은 이름의 함수 — 다른 .o 파일로 링크.

```c
// production: real_db.o로 링크
int db_query(const char* sql) { /* real DB */ }

// test: mock_db.o로 링크
int db_query(const char* sql) { return 42; }   // mock
```

빌드 시스템에서 — 선택.

```makefile
production: main.o real_db.o
test: main.o mock_db.o
```

**Enabling Point** — Makefile / build script.

## Preprocessing Seam

`#define` / `#ifdef`로 — 컴파일 시 치환.

```c
// 이전:
int process() {
    return external_call();
}

// 이후:
#ifdef TEST
#define external_call() mock_external_call()
#endif

int process() {
    return external_call();   // 테스트 시 mock으로 치환됨
}
```

테스트 빌드에서만 — TEST 매크로 정의.

## Function Pointer

런타임에 — 다른 함수로 교체.

```c
// 이전:
int process() {
    return real_func();
}

// 이후:
int (*func_ptr)() = real_func;   // 기본은 real

int process() {
    return func_ptr();
}

// 테스트:
func_ptr = mock_func;
result = process();
```

객체 seam의 — C 버전.

## Test Harness

C에서도 — CUnit / Check / CMocka 등 xUnit 프레임워크 존재.

```c
void test_process() {
    setup_mocks();
    int result = process();
    CU_ASSERT_EQUAL(result, 42);
}
```

GoogleTest는 C++이지만 — C 코드도 테스트 가능 (extern "C").

## 정리

- 객체 지향 아니어도 — Legacy 작업 가능
- **Link Seam** — 빌드 시 다른 .o
- **Preprocessing Seam** — 매크로 치환
- **Function Pointer** — 런타임 교체
- xUnit 프레임워크 — C/C++에도 있음
