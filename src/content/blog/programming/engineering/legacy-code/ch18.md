> **Working Effectively with Legacy Code** Chapter 18 요약

## 이 챕터의 메시지

테스트가 — 늘어난다. 본 코드보다 — 길어진다.

```
src/Order.java (200 lines)
test/OrderTest.java (1500 lines)   ← 본 코드의 7배
```

좋은 일이지만 — 가독성을 해친다.

## 분리

### 테스트 파일 분리

```
OrderTest.java
OrderTaxTest.java
OrderShippingTest.java
OrderPaymentTest.java
```

기능별로 — 분리. 한 파일에서 모든 시나리오 모음이 아니라 — 책임별로 모음.

### 패키지 분리

```
src/main/java/com/example/Order.java
src/test/java/com/example/OrderTest.java
src/test/java/com/example/OrderFlowTest.java
```

production / test가 — 같은 패키지 (`package private` 접근). 다른 디렉토리.

## 명명

테스트 이름에 — **시나리오**.

```java
// 좋음:
@Test void deposit_to_empty_account_sets_balance() { }
@Test void deposit_negative_amount_throws() { }
@Test void withdraw_below_zero_returns_false() { }

// 나쁨:
@Test void test1() { }
@Test void deposit() { }
```

이름이 — 명세 역할. 테스트 결과 보고도 의미 있게.

## Test Helper

```java
// 모든 테스트에서 반복:
Customer c = new Customer();
c.setName("Alice");
c.setEmail("a@b.com");
c.setLevel(VIP);
```

**Test Helper** / **Object Mother**로 단순화.

```java
class CustomerFixture {
    static Customer createVIP() {
        Customer c = new Customer();
        c.setName("Alice");
        c.setEmail("a@b.com");
        c.setLevel(VIP);
        return c;
    }
}

// 사용:
Customer c = CustomerFixture.createVIP();   // 한 줄
```

## 정리

- 테스트가 늘어남 = 좋은 일
- 분리 / 명명 / Helper로 — 정리
- 테스트 코드도 — **코드** (가독성 중요)
