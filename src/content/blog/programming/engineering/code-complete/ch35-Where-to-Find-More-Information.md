## 이 챕터의 메시지

900페이지의 책이 끝나도 — 학습은 끝나지 않는다. McConnell은 마지막 챕터에서 — **다음 단계의 자원들**을 정리.

> Code Complete는 — **시작점**이다.

## 핵심 추천 도서

### Construction 일반

- **The Pragmatic Programmer** — Hunt & Thomas. 실용주의 원칙.
- **Clean Code** — Martin. 일상 코딩 규칙.
- **Refactoring** — Fowler. 변환 카탈로그.
- **Working Effectively with Legacy Code** — Feathers. 옛 코드 살리기.

### 설계·아키텍처

- **Design Patterns** — Gang of Four. 표준 패턴 카탈로그.
- **Patterns of Enterprise Application Architecture** — Fowler.
- **Domain-Driven Design** — Evans.
- **Clean Architecture** — Martin.

### 알고리즘·자료구조

- **Introduction to Algorithms** — CLRS. 표준 교재.
- **The Algorithm Design Manual** — Skiena. 실용 중심.
- **Algorithms** — Sedgewick. 풍부한 시각화.

### 언어별

### 인간 측면

- **Peopleware** — DeMarco & Lister. 팀과 환경.

### 테스트·품질

## 학습 전략

### 1) 깊이 vs 넓이

- **깊이** — 한 영역을 깊게 파라.
- **넓이** — 다양한 영역을 알라.

둘 다 필요. 시간을 나눠.

### 2) 책 외 자원

- **블로그** — Martin Fowler, Joel Spolsky, Jeff Atwood.
- **콘퍼런스** — GOTO, QCon, Strange Loop.
- **오픈소스** — 좋은 프로젝트의 코드 읽기.
- **공동체** — Stack Overflow, GitHub.

### 3) 실천

> 읽기만 해선 안 된다. **써야 배운다**.

- 사이드 프로젝트.
- 옛 코드 리팩토링.
- 새 언어 실험.
- 동료에게 가르치기 (가르치며 배운다).

## 평생 학습

> 좋은 프로그래머는 — **매일 배운다**.

기술이 빠르게 바뀐다. 5년 전의 베스트 프랙티스가 오늘 — 안티 패턴일 수도. 매일 조금씩 학습하는 사람만이 — 시간과 함께 좋아진다.

## 정리

- Code Complete는 — **시작점**.
- 분야별 깊은 자원들이 있다.
- **깊이 + 넓이**의 균형.
- **읽기 + 실천**의 결합.
- 평생 학습이 — 직업의 본질.

## 책의 마지막 메시지

이로써 35개 챕터가 끝났다. McConnell이 보여 준 것은 — **construction이 그 자체로 깊고 풍부한 분야**라는 것.

> 매일의 코드 한 줄, 함수 이름 하나, 변수 하나의 결정이 — 시간과 함께 시스템의 운명을 만든다.

장인 정신은 — 거대한 깨달음이 아니라 **매일의 작은 선택**의 누적이다.
