## 이 챕터의 메시지

12장의 끝에서 *Expression* 추상의 필요가 드러났다. 이 장은 그 추상을 코드로 옮긴다. `Expression` 인터페이스와 `Sum` 클래스가 도입되고, `Money` 자체가 `Expression`을 구현하게 된다. `Bank.reduce`가 식을 실제 값으로 *축소*하는 책임을 맡는다.

이 변경은 작은 인터페이스 하나의 추가지만, 책 후반부의 모든 다중 통화 처리가 이 토대 위에 서게 된다.

## Expression 인터페이스

처음에는 빈 인터페이스다. 어떤 메서드가 필요한지는 사용 사례가 등장하면서 결정된다.

```java
interface Expression {}

class Money implements Expression {
    Expression plus(Money addend) {
        return new Sum(this, addend);
    }
}

class Sum implements Expression {
    Money augend;
    Money addend;

    Sum(Money augend, Money addend) {
        this.augend = augend;
        this.addend = addend;
    }
}
```

`plus`는 이제 `Money` 대신 `Expression`을 반환한다. 같은 통화 덧셈일지라도 일단 `Sum`을 만든다. 즉시 계산하지 않는다.

`augend`와 `addend`는 수학 용어다. `augend + addend = sum`이다. Beck이 이 어휘를 고른 것은 도메인 명사를 코드에 그대로 옮기는 정신이다.

## Bank.reduce — Sum 처리

`Bank`가 `Sum`을 실제 `Money`로 축소한다.

```java
class Bank {
    Money reduce(Expression source, String to) {
        Sum sum = (Sum) source;
        int amount = sum.augend.amount + sum.addend.amount;
        return new Money(amount, to);
    }
}
```

테스트로 검증한다.

```java
public void testReduceSum() {
    Money five = Money.dollar(5);
    Expression sum = five.plus(five);
    Bank bank = new Bank();
    Money result = bank.reduce(sum, "USD");
    assertEquals(Money.dollar(10), result);
}
```

통과한다. 그러나 이 구현은 두 가지 약점을 가진다.

| 약점 | 의미 |
|------|------|
| `Sum`만 처리 | 입력이 `Money` 단독이면 캐스트 실패 |
| 환율 무시 | augend/addend의 통화가 달라도 amount만 더함 |

두 약점은 다음 장들에서 각각 해결된다. `Money` 단독 처리는 14장의 polymorphism으로, 환율 적용은 15장의 `Bank.addRate`로.

## Expression 패턴의 정신

`Expression`은 *계산을 표현하는 객체*다. 즉시 평가하지 않고, 평가에 필요한 컨텍스트(환율, 목표 통화)가 주어지는 시점에 비로소 결과를 낸다.

이 정신은 여러 분야에서 반복된다.

- LINQ의 `IEnumerable<T>` 쿼리 — 호출 시점에 실제 실행
- React의 가상 DOM — 변경 식을 모은 뒤 한 번에 적용
- 함수형 언어의 `Promise`/`Future` — 결과의 약속
- 컴파일러의 AST — 코드의 트리 표현

지연 평가와 명시적 식 표현이라는 두 정신이 결합한 형태다.

## 정리

- `Expression` 인터페이스 도입 — `Money`와 `Sum`이 구현
- `plus`는 즉시 계산하지 않고 `Sum` 객체를 만든다
- `Bank.reduce`가 식을 실제 값으로 축소
- 두 약점이 남는다 — `Money` 단독 처리 부재, 환율 무시
