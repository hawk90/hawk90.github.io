> **Test-Driven Development by Example** Chapter 23 요약

## 이 챕터의 메시지

여러 테스트를 — 한 번에 실행. `TestSuite`.

```python
class TestSuite:
    def __init__(self):
        self.tests = []
    
    def add(self, test):
        self.tests.append(test)
    
    def run(self, result):
        for test in self.tests:
            test.run(result)
```

이제:

```python
suite = TestSuite()
suite.add(TestCaseTest("testTemplateMethod"))
suite.add(TestCaseTest("testResult"))
suite.add(TestCaseTest("testFailedResult"))
suite.add(TestCaseTest("testSuite"))

result = TestResult()
suite.run(result)
print(result.summary())   # 모든 테스트 결과
```

## Composite 패턴

`TestSuite`도 `run(result)` 메서드 — TestCase와 같은 인터페이스. **Composite 패턴**.

```
TestCase.run(result)    — 단일 테스트
TestSuite.run(result)   — 여러 테스트 묶음, 같은 인터페이스
```

TestSuite를 TestSuite에 — 트리 구조도 가능.
