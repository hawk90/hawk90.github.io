> *Test-Driven Development by Example* Chapter 5 요약

## 이 챕터의 메시지

원래 책의 시작 요구사항은 다중 통화 보고서였다. 지금까지는 USD만 다뤘다. 진짜 다중 통화로 가기 위해 두 번째 통화 `Franc`을 도입한다. 그 과정에서 `Dollar`와 거의 동일한 코드가 등장하며 중복이 명확하게 드러난다.

이 장의 핵심은 중복을 *해결하지 않는다*는 것이다. 한 사이클에 한 가지 일만 한다. 중복은 다음 장의 작업이고, 지금은 일단 두 통화가 동작하는지부터 확인한다.

## Franc 테스트

`Dollar`의 곱셈 테스트와 거의 같은 모양이다.

```java
public void testFrancMultiplication() {
    Franc five = new Franc(5);
    assertTrue(new Franc(10).equals(five.times(2)));
}
```

`Franc` 클래스가 없으므로 컴파일조차 되지 않는다. 빨간 막대다.

## Franc 구현 — Dollar의 복사

가장 빠른 초록은 `Dollar`를 그대로 복사해 이름만 바꾸는 것이다.

```java
class Franc {
    private int amount;

    Franc(int amount) {
        this.amount = amount;
    }

    Franc times(int multiplier) {
        return new Franc(amount * multiplier);
    }

    public boolean equals(Object obj) {
        Franc franc = (Franc) obj;
        return amount == franc.amount;
    }
}
```

`Dollar`와 비교해 본다.

```java
class Dollar {
    private int amount;

    Dollar(int amount) {
        this.amount = amount;
    }

    Dollar times(int multiplier) {
        return new Dollar(amount * multiplier);
    }

    public boolean equals(Object obj) {
        Dollar dollar = (Dollar) obj;
        return amount == dollar.amount;
    }
}
```

거의 같다. 이름과 생성자만 다르다. 그러나 두 테스트는 모두 통과한다. 일단 초록이다.

## 중복을 견딘다 — 한 번에 한 가지

같은 코드가 두 클래스에 그대로 존재한다. 본능적으로 즉시 정리하고 싶지만 Beck은 이 단계에서 일부러 멈춘다.

```text
지금  : Franc 테스트 통과
지금 X: 중복 해결 (다음 사이클)
```

이유는 두 가지다.

첫째, 한 사이클이 끝나기 전에 두 번째 작업을 시작하면 무엇이 깨졌을 때 원인을 알기 어렵다. Franc 테스트가 통과한 직후라면 그 통과가 보장된 상태에서 다음 변경을 시작할 수 있다.

둘째, 중복은 자주 *어떻게 통합할지*가 한 가지 이상이다. 한 번 견뎌 보면 더 좋은 통합 방향이 보인다. 다음 장의 `Money` 추상 부모는 그렇게 떠오른 답이다.

## 중복은 신호다

> Same code in two places → 공통 추상이 그곳에 숨어 있다.

Fowler가 *Refactoring*에서 *Duplicated Code*를 첫 번째 코드 냄새로 꼽는 이유와 같다. 중복은 통합되어야 할 개념이 아직 이름을 가지지 못한 상태다.

`Dollar`와 `Franc`의 차이는 사실 통화 코드 문자열뿐이다. 이 사실을 받아들이면 다음 장의 통합 방향이 자연스럽게 보인다.

## To-Do List 갱신

```text
[ ] $5 + 10 CHF = $10 (환율 2:1)
[X] $5 × 2 = $10
[X] amount를 private으로
[X] Dollar의 사이드 이펙트 제거
[ ] Money rounding
[X] equals()
[ ] hashCode()
[X] Franc × 2 = $10
[ ] Dollar / Franc 중복 제거    ← 새로 추가
[ ] equals(null)
[ ] equals(다른 타입)
```

다음 장은 `Dollar`와 `Franc`의 공통 부모 `Money`를 끌어내면서 `equals`부터 통합하기 시작한다.

## 정리

- 두 번째 통화 `Franc` 도입으로 다중 통화 요구사항에 가까워진다
- 가장 빠른 구현은 `Dollar` 복사다 — 의도된 중복
- TDD는 한 사이클에 한 가지 일만 한다 — 중복 해결은 다음 단계
- **중복은 공통 추상이 숨어 있다는 신호**다
- To-Do의 *중복 제거* 항목이 다음 장을 호출한다
