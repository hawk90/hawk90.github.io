> **Test-Driven Development by Example** Chapter 4 요약

## 이 챕터의 메시지

`equals`가 생기면서 — 외부에서 `amount`에 직접 접근할 필요 X.

## 테스트 수정

```java
// 이전:
public void testMultiplication() {
    Dollar product = new Dollar(5).times(2);
    assertEquals(10, product.amount);   // amount 직접 접근
}

// 수정:
public void testMultiplication() {
    Dollar product = new Dollar(5).times(2);
    assertTrue(new Dollar(10).equals(product));  // equals로
}
```

이제 — `amount`를 private으로.

```java
class Dollar {
    private int amount;
    // ...
}
```

캡슐화 완료.

## 작은 리팩터링

```
이전: 외부가 데이터에 직접 접근
이후: 외부가 동작(equals)으로 비교
```

값을 다루는 게 아니라 — **객체의 의미**를 다룬다.

## 7장의 Privacy 패턴

26장에서 다룰 — Privacy 패턴. 가능한 한 좁은 visibility.

```
private → package → protected → public
   ↑                              ↓
가장 좁음                    가장 넓음
```

기본 — private. 필요할 때만 넓힘.
