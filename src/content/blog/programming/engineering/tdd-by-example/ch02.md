## 이 챕터의 메시지

1장의 사이드 이펙트 문제를 해결.

> **Value Object은 불변이어야.**

## 새 테스트

```java
public void testMultiplication() {
    Dollar five = new Dollar(5);
    Dollar product = five.times(2);
    assertEquals(10, product.amount);
    product = five.times(3);
    assertEquals(15, product.amount);   // five가 안 바뀜
}
```

`times`가 새 Dollar 반환. 원본 변경 X.

## 구현

```java
class Dollar {
    int amount;
    Dollar(int amount) { this.amount = amount; }
    Dollar times(int multiplier) {
        return new Dollar(amount * multiplier);
    }
}
```

`times`가 새 객체를 만든다.

## Value Object 패턴

이게 25장에서 다룰 패턴의 첫 등장.

- 객체의 값이 그 정체성
- 한 번 생성되면 안 바뀜
- 동등성 = 값의 동등성

`String`, `Date` (Java 8 이후), `BigDecimal` 등이 — value object.
