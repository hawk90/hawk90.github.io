> **Test-Driven Development by Example** Chapter 20 요약

## 이 챕터의 메시지

setUp의 짝 — `tearDown`. 매 테스트 후 자원 정리.

```python
class TestCase:
    def setUp(self): pass
    def tearDown(self): pass
    
    def run(self):
        self.setUp()
        method = getattr(self, self.name)
        method()
        self.tearDown()  # 새로
```

## 언제 tearDown인가

- DB 연결 닫기
- 임시 파일 삭제
- 외부 자원 해제

매 테스트가 — **다음 테스트에 영향 X**.

## 정리

- **tearDown** — 매 테스트 후 정리
- 자원 해제 보장
- 테스트 격리의 다른 한 축
