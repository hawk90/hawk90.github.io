> **Test-Driven Development by Example** Chapter 29 요약

## 이 챕터의 메시지

xUnit 프레임워크의 표준 패턴들.

## 패턴 — Assertion

> **Boolean 검증.**

```python
assert_true(value)
assert_equals(expected, actual)
assert_null(value)
```

테스트의 핵심. 모든 테스트가 — assertion으로 끝.

## 패턴 — Fixture

> **테스트 setUp으로 준비된 객체들.**

```python
class TestThing(TestCase):
    def setUp(self):
        self.thing = Thing(initial_state)
    
    def test_x(self):
        # self.thing 사용
```

매 테스트가 — 같은 시작 상태.

## 패턴 — External Fixture

> **테스트 외부 자원 — DB, 파일, 네트워크.**

```python
def setUp(self):
    self.db = openTestDatabase()

def tearDown(self):
    self.db.close()
```

setUp / tearDown으로 — 자원 lifecycle 관리.

## 패턴 — Test Method

> **테스트 메서드 한 개 = 한 시나리오.**

```python
def test_save_user():       # 한 시나리오
    user = User("alice")
    db.save(user)
    assert_equals("alice", db.find("alice").name)

def test_save_duplicate():   # 다른 시나리오
    ...
```

한 메서드 = 한 검증. 여러 시나리오면 — 여러 메서드.

## 패턴 — Exception Test

> **예외가 던져지는 것을 검증.**

```python
def test_invalid_input():
    try:
        process(invalid)
        fail()   # 여기 오면 안 됨
    except ValueError:
        pass   # 의도된 예외
```

또는 더 신식 `assert_raises`:

```python
def test_invalid_input():
    assert_raises(ValueError, lambda: process(invalid))
```

## 패턴 — All Tests

> **모든 테스트를 한 번에 실행.**

```python
def all_tests():
    suite = TestSuite()
    suite.add(TestThing.testA)
    suite.add(TestThing.testB)
    # ...
    return suite
```

또는 자동 검출 — pytest의 `test_*` 패턴.

## 정리

- **Assertion** — 검증의 핵심
- **Fixture** — 테스트 setup
- **External Fixture** — 외부 자원
- **Test Method** — 한 메서드 = 한 시나리오
- **Exception Test** — 예외 검증
- **All Tests** — 통합 실행
