Part II 끝. xUnit 프레임워크가 — 자기 자신으로 테스트.

## "Self-Testing" — Bootstrap의 우아함

```
TestCase 클래스를 만든다
  ↓
그 TestCase를 사용해 — TestCase 자체를 테스트
  ↓
"내가 만든 도구가 자기를 검증한다"
```

이게 — bootstrapping의 우아함. Compiler가 자기 자신을 컴파일하는 것과 같은 정신.

## xUnit의 영향

이 작은 예제가 — 거대한 영향.

- **JUnit** (Java)
- **NUnit** (.NET)
- **pytest** (Python)
- **RSpec** (Ruby)
- **Mocha / Jest** (JavaScript)
- **GoogleTest, Catch2** (C++)

모두 — Beck의 xUnit 패턴에서 진화. 한 사람의 작은 디자인이 — 산업 표준이 됐다.

## 학습된 패턴

- **TestCase** — 한 테스트의 단위
- **setUp / tearDown** — 매 테스트 전후
- **TestResult** — 결과 추적
- **TestSuite** — Composite
- **Exception 처리** — setUp / testMethod / tearDown 각각
