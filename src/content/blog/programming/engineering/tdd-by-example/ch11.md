> **Test-Driven Development by Example** Chapter 11 요약

## 이 챕터의 메시지

이전 챕터에서 Money concrete로 — 이제 Dollar / Franc 완전 제거.

```java
// 제거됨:
// class Dollar extends Money { ... }
// class Franc extends Money { ... }

// 남음:
class Money {
    int amount;
    String currency;
    
    static Money dollar(int amount) {
        return new Money(amount, "USD");
    }
    
    static Money franc(int amount) {
        return new Money(amount, "CHF");
    }
}
```

이제 — 단순한 Value Object 하나.

## "Root of All Evil"

제목의 의미 — **premature inheritance가 모든 악의 근원**.

처음에 Dollar / Franc 두 클래스로 시작 — 자연스러워 보였다. 그러나 같은 패턴이 반복되며 — 결국 통합이 답.

상속을 너무 일찍 도입하면 — 나중에 후회. Refactoring의 메시지와 같다.

## 모든 테스트 통과

리팩터링 끝 — 모든 테스트 통과. 사용자 코드 안 깨짐.
