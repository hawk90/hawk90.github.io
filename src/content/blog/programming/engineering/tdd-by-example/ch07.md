## 이 챕터의 메시지

`getClass()` 비교는 — 언어 구현 디테일. 도메인 개념으로.

> **Currency를 데이터로.**

## Currency 추가

```java
class Money {
    protected int amount;
    protected String currency;
    
    String currency() { return currency; }
    
    public boolean equals(Object obj) {
        Money money = (Money) obj;
        return amount == money.amount
            && currency.equals(money.currency);
    }
}

class Dollar extends Money {
    Dollar(int amount) {
        this.amount = amount;
        this.currency = "USD";
    }
}

class Franc extends Money {
    Franc(int amount) {
        this.amount = amount;
        this.currency = "CHF";
    }
}
```

이제 — `getClass()` 대신 currency 비교.

## 왜 이게 더 좋은가

- **도메인 개념** — 사용자가 이해
- **유연** — 새 currency를 새 클래스 없이 추가 가능 (factory로)
- **확장 가능** — 환율 정보를 currency에 붙일 수 있음
