> *Test-Driven Development by Example* Chapter 12 요약

## 이 챕터의 메시지

12장은 원래 목표였던 *다중 통화 덧셈*으로 본격 진입한다. Beck은 가장 단순한 시나리오부터 시작한다. 같은 통화의 덧셈이다. 그 구현은 단순하지만, 이어서 다른 통화 덧셈을 시도하면 곧바로 한계가 드러난다. 그 한계가 **Expression** 패턴의 필요를 만든다.

이 장의 결정적 디자인 통찰은 다음 한 줄이다. *덧셈의 결과는 즉시 계산된 값이 아니라 아직 계산되지 않은 식이다.*

## 첫 덧셈 테스트

같은 통화의 덧셈으로 시작한다.

```java
public void testSimpleAddition() {
    Money sum = Money.dollar(5).plus(Money.dollar(5));
    assertEquals(Money.dollar(10), sum);
}
```

`plus` 메서드가 없으므로 빨간 막대다.

## 단순 구현

같은 통화라면 `amount`를 더하면 된다.

```java
class Money {
    Money plus(Money addend) {
        return new Money(amount + addend.amount, currency);
    }
}
```

테스트가 통과한다. 그러나 이 구현은 *같은 통화* 가정 위에 서 있다. 다른 통화면 어떻게 되는가?

## 다중 통화 시나리오

원래 목표는 다음과 같다.

```text
$5 + 10 CHF = $10 (환율 2:1)
```

`plus`가 즉시 `Money`를 반환한다면 어떤 통화로 답을 줘야 할까? USD로? CHF로? 환율을 누가 알고 있나? 환율이 없다면 합산 자체를 할 수 없다.

이 막다른 길이 디자인 결정으로 이어진다.

## Expression 아이디어

해법은 *덧셈의 결과를 즉시 계산하지 않는 것*이다. 결과는 "5달러와 10프랑의 합"이라는 *식*으로 표현한다. 환율과 목표 통화가 주어지는 시점에 비로소 계산한다.

```java
// $5 + 10 CHF
Money five = Money.dollar(5);
Money tenFrancs = Money.franc(10);
Expression sum = five.plus(tenFrancs);

// 나중에 USD로 변환
Money result = sum.reduce(bank, "USD");
```

`Expression`이 핵심 추상이다. `Money` 자체도 한 종류의 Expression이고, 합 `Sum`도 한 종류의 Expression이다. `reduce`는 환율 정보가 있는 `Bank`와 목표 통화를 받아 실제 `Money`로 *축소*한다.

## 디자인 결정

두 갈래의 선택이 있다.

| 선택 | 결과 | 평가 |
|------|------|------|
| `plus`가 즉시 `Money` 반환 | 같은 통화만 가능 | 메인 목표 달성 X |
| `plus`가 `Expression` 반환 | 다중 통화도 자연스러움 | 약간의 추상화 비용 |

후자를 선택한다. 추상화 비용은 작고 다중 통화 처리가 자연스러워진다.

## Composite와 Interpreter의 정신

`Expression`은 GoF의 **Composite** 패턴 또는 **Interpreter** 패턴의 응용이다. 식이 객체로 표현되고, 단순 식과 복합 식이 같은 인터페이스를 따른다.

```text
Expression
├── Money (단순 식)
└── Sum   (복합 식 = augend + addend)
```

이 구조 덕에 `5 + (10 CHF + 3 EUR)` 같은 중첩 식도 자연스럽게 표현된다. `Sum`의 두 피연산자가 다시 `Expression`이기 때문이다.

이 패턴은 컴파일러의 AST(추상 구문 트리), 수식 평가기, 쿼리 빌더 등에서 반복적으로 등장한다. *지연 평가* 또는 *값 객체로서의 계산*이라는 정신을 공유한다.

## To-Do List 갱신

```text
[X] $5 + $5 = $10 (같은 통화 덧셈)
[ ] Expression 인터페이스
[ ] Sum 클래스
[ ] Bank.reduce(Expression, currency)
[ ] $5 + 10 CHF = $10 (환율 2:1)
[ ] equals(null)
[ ] equals(다른 타입)
```

## 정리

- 같은 통화 덧셈은 즉시 계산으로 단순하게 풀린다
- 다중 통화 덧셈은 즉시 계산이 불가능 — 환율과 목표 통화가 필요
- 해법은 **Expression** 추상 — 결과를 식으로 표현하고 나중에 reduce
- Composite/Interpreter 패턴의 정신과 일치한다
- 다음 장부터 Expression과 Sum, Bank.reduce가 구체화된다
