> **Test-Driven Development by Example** Chapter 12 요약

## 이 챕터의 메시지

원래 목표 — 다중 통화 덧셈. 시작.

## 첫 덧셈 테스트

```java
public void testSimpleAddition() {
    Money sum = Money.dollar(5).plus(Money.dollar(5));
    assertEquals(Money.dollar(10), sum);
}
```

같은 통화의 덧셈. 단순.

## 단순 구현

```java
Money plus(Money addend) {
    return new Money(amount + addend.amount, currency);
}
```

통과. 그러나 — **다중 통화는?**

```
$5 + 10 CHF = ?
```

직접 더할 수 없음. 환율 필요. Expression 패턴이 등장.

## Expression 아이디어

```java
// $5 + 10 CHF
Money five = Money.dollar(5);
Money tenFrancs = Money.franc(10);
Expression sum = five.plus(tenFrancs);

// 나중에 USD로 변환
Money result = sum.reduce(bank, "USD");
```

`Expression` — 아직 계산 안 된 식. `reduce`로 — 실제 값.

## 디자인 결정

```
Money.plus(Money) 가 Money 반환?  → 다중 통화 어려움
Money.plus(Money) 가 Expression 반환? → 유연
```

후자가 — 다중 통화에 자연스러움.

## 정리

- 같은 통화 덧셈 — 단순
- 다중 통화 — **Expression** 패턴 필요
- 다음 장부터 Expression 구체화
