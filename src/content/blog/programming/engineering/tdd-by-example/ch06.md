> **Test-Driven Development by Example** Chapter 6 요약

## 이 챕터의 메시지

Dollar와 Franc — 공통 부모 `Money`로.

## Money 클래스

```java
class Money {
    protected int amount;
    
    public boolean equals(Object obj) {
        Money money = (Money) obj;
        return amount == money.amount;
    }
}

class Dollar extends Money {
    Dollar(int amount) { this.amount = amount; }
    Dollar times(int multiplier) {
        return new Dollar(amount * multiplier);
    }
}

class Franc extends Money {
    // 동일
}
```

`equals`가 Money로 — Pull Up Method (Refactoring 12장).

## 문제 — Dollar.equals(Franc)?

```java
assertFalse(new Dollar(5).equals(new Franc(5)));
```

이 테스트 — 통과? Money.equals는 amount만 본다. 5와 5는 같다 → 통과 (잘못!).

## 해결 — class 비교 추가

```java
public boolean equals(Object obj) {
    Money money = (Money) obj;
    return amount == money.amount 
        && getClass().equals(money.getClass());
}
```

## Class 비교의 위험

`getClass()` 비교는 — 도메인 개념이 아닌 **언어 구현 디테일**. 다음 장에서 더 좋게 만든다.
