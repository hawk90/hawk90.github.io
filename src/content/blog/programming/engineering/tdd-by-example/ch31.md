> **Test-Driven Development by Example** Chapter 31 요약

## 이 챕터의 메시지

TDD의 세 번째 단계 — **Refactor**. 그 안에서 자주 쓰는 리팩터링들.

## 자주 쓰는 리팩터링

### Reconcile Differences

> **두 코드가 거의 같으면 — 단계적으로 동일하게 만들어 합쳐라.**

```
A: x = foo(1)
B: x = foo(2)
   ↓ 한 변수로 차이 분리
A: param = 1; x = foo(param)
B: param = 2; x = foo(param)
   ↓ 이제 함수 동일, 합칠 수 있음
```

### Isolate Change

> **변경할 부분을 — 먼저 분리.**

```python
# 변경 전
def process(data):
    # 200 lines, 한 부분만 바꿔야 함
    
# 1단계: Extract Function
def process(data):
    do_part_a(data)
    do_part_b(data)  # 이 부분만 바꿀 것
    do_part_c(data)

# 2단계: do_part_b만 변경
```

### Migrate Data

> **데이터 구조 변경 — 점진적으로.**

```
1. 새 구조 추가
2. 새 구조에도 쓰기
3. 새 구조에서 읽기
4. 옛 구조 제거
```

분산 시스템 / 운영 중인 코드에서 — 안전한 변경.

### Extract Method, Inline Method

Fowler의 카탈로그와 같은 패턴. TDD에서 자주.

```python
# Extract
result = process_data(data)
process_data를 추출

# Inline
result = result_with_extra_logic()
extra_logic이 그냥 한 줄이면 인라인
```

## TDD와 Fowler

Beck의 *TDD*와 Fowler의 *Refactoring* — 같은 정신.

```
Fowler: 리팩터링 = 외부 동작 유지 + 작은 단계
Beck:    TDD의 Refactor 단계 = 같은 정신
```

두 책이 — 짝.

## 정리

- TDD의 Refactor 단계 = Fowler의 리팩터링
- **Reconcile Differences** — 두 비슷한 코드 합치기
- **Isolate Change** — 변경할 부분 분리
- **Migrate Data** — 점진적 데이터 변경
- 두 책이 — 짝의 정신
