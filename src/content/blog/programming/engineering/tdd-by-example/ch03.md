> **Test-Driven Development by Example** Chapter 3 요약

## 이 챕터의 메시지

Dollar 객체 비교 — `equals` 구현.

## 첫 테스트

```java
public void testEquality() {
    assertTrue(new Dollar(5).equals(new Dollar(5)));
}
```

가장 단순한 통과:

```java
public boolean equals(Object obj) {
    return true;
}
```

분명히 잘못. 그러나 — 일단 초록.

## Triangulation

테스트를 추가해서 — 일반화를 **강제**.

```java
assertTrue(new Dollar(5).equals(new Dollar(5)));
assertFalse(new Dollar(5).equals(new Dollar(6)));   // 추가
```

이제 `return true`는 — 두 번째 테스트에 실패. 진짜 구현 필요.

```java
public boolean equals(Object obj) {
    Dollar dollar = (Dollar) obj;
    return amount == dollar.amount;
}
```

## Triangulation의 의미

**한 데이터 포인트** = 한 직선이 무한히. 일반화 강제 X.
**두 데이터 포인트** = 한 직선이 유일. 일반화 자연스러움.

따라서 — 비슷한 테스트 둘로 진짜 구현 끌어냄.

## 25장 패턴

이게 책 Part III에서 다룰 패턴 — **Triangulation Pattern**.

## 정리

- equals를 위해 — Triangulation
- 두 데이터 포인트로 일반화 강제
- "Fake it" + 두 번째 테스트 = 진짜 구현
