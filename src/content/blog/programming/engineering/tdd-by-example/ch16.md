> **Test-Driven Development by Example** Chapter 16 요약

## 이 챕터의 메시지

지금까지 — Money만 `plus` / `times`. Sum도 그래야 — 일관성.

```java
interface Expression {
    Money reduce(Bank bank, String to);
    Expression plus(Expression addend);
    Expression times(int multiplier);
}
```

이제 Sum도 plus / times 가능. 일관된 인터페이스.

```java
Expression sum = Money.dollar(5).plus(Money.dollar(5));
Expression doubled = sum.times(2);   // 가능
```

## 정리

- Expression의 모든 연산이 일관되게 polymorphic
- Money / Sum 모두 Expression의 풀 인터페이스
