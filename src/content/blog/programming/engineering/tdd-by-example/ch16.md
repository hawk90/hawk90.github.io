> *Test-Driven Development by Example* Chapter 16 요약

## 이 챕터의 메시지

지금까지 `plus`와 `times`는 `Money`의 메서드였다. `Sum`은 결과로만 등장했고 연산자로는 쓰이지 않았다. 그러나 진짜 `Expression` 추상이라면 `Sum`도 `plus`와 `times`를 가져야 한다. 그래야 `(5 + 10).times(2)` 같은 자연스러운 식이 가능하다.

이 장은 그 인터페이스의 마지막 정합을 맞춘다. Part I의 디자인 작업이 이로써 마무리된다.

## 인터페이스 통합

`Expression`에 세 메서드를 모두 둔다.

```java
interface Expression {
    Money reduce(Bank bank, String to);
    Expression plus(Expression addend);
    Expression times(int multiplier);
}
```

`Money`와 `Sum`이 모두 이 풀 인터페이스를 구현한다.

```java
class Money implements Expression {
    public Expression plus(Expression addend) {
        return new Sum(this, addend);
    }

    public Expression times(int multiplier) {
        return new Money(amount * multiplier, currency);
    }
}

class Sum implements Expression {
    Expression augend;
    Expression addend;

    public Expression plus(Expression addend) {
        return new Sum(this, addend);
    }

    public Expression times(int multiplier) {
        return new Sum(augend.times(multiplier), addend.times(multiplier));
    }
}
```

`augend`와 `addend`의 타입도 `Money`에서 `Expression`으로 일반화했다. 이제 `Sum` 안에 `Sum`이 들어갈 수도 있다.

## 자연스러운 식 표현

새 인터페이스로 다음 같은 식이 자연스럽게 표현된다.

```java
Expression sum = Money.dollar(5).plus(Money.dollar(5));
Expression doubled = sum.times(2);   // (5+5) * 2

Bank bank = new Bank();
Money result = bank.reduce(doubled, "USD");   // 20 USD
```

`(5 + 5) * 2`라는 수식이 객체 트리로 그대로 표현된다. Composite 패턴의 정신이 완성됐다.

## 일관성의 가치

`Sum`이 `plus`/`times`를 가지지 않았다면 사용자는 다음 같은 코드를 짤 수 없었다.

```java
// 만약 Sum이 plus를 안 가졌다면
Expression a = Money.dollar(5).plus(Money.dollar(5));
Expression b = a.plus(Money.franc(10));   // 컴파일 에러
```

인터페이스의 일관성은 사용자가 합성을 자유롭게 하도록 허락한다. *모든 Expression이 모든 연산을 지원한다*라는 단순한 약속이 표현력을 만든다.

## Sum.times의 분배법칙

`Sum.times(n)`이 `augend.times(n) + addend.times(n)`이라는 점은 수학의 분배법칙과 같다.

```text
(a + b) * n = a*n + b*n
```

코드가 수학을 그대로 따른다. 도메인의 규칙이 코드 구조에 반영된 좋은 예다.

## Part I 디자인의 완성

이 시점에서 Money 예제의 디자인은 마무리된다.

| 요소 | 역할 |
|------|------|
| `Money` | 통화 + amount의 값 객체, Expression의 leaf |
| `Sum` | 두 Expression의 합, Expression의 composite |
| `Expression` | 공통 인터페이스 — plus, times, reduce |
| `Bank` | 환율 표 보관, reduce의 진입점 |

이 네 요소가 다중 통화 산술의 모든 요구를 처리한다.
