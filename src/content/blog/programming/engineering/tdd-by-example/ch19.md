> **Test-Driven Development by Example** Chapter 19 요약

## 이 챕터의 메시지

매 테스트 전 — fixture 준비. `setUp`.

```python
class TestCase:
    def setUp(self):
        pass
    
    def run(self):
        self.setUp()       # 매 테스트 전
        method = getattr(self, self.name)
        method()

class WasRun(TestCase):
    def setUp(self):
        self.wasRun = None
        self.wasSetUp = 1
```

매 테스트가 — 깨끗한 초기 상태에서 시작.

## 왜 setUp인가

```
테스트 1: 객체 만들고 사용 → 상태 변경
테스트 2: 그 객체를 다시 사용 → 의도 안 한 상태?
```

이걸 막으려면 — 매번 새로 만든다.

## 정리

- **setUp** — 매 테스트 전 fixture 준비
- 테스트 격리 보장
- xUnit / JUnit / pytest 모두 동일 패턴
