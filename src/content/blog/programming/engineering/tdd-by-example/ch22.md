## 이 챕터의 메시지

setUp이 실패하면? — 적절히 처리.

```python
def run(self, result):
    result.testStarted()
    try:
        self.setUp()
    except:
        result.testFailed()
        return        # setUp 실패하면 testMethod 실행 X
    
    try:
        method = getattr(self, self.name)
        method()
    except:
        result.testFailed()
    
    self.tearDown()
```

setUp 실패 → testMethod 실행 X → 그러나 결과는 기록.

## Exception 처리

xUnit / JUnit 같은 프레임워크에서 — 같은 패턴.

```
setUp 실패  → 테스트 실패로 기록, testMethod 안 실행
testMethod 실패 → 실패로 기록
tearDown 실패 → 실패로 기록 (또는 별도 처리)
```

각각 다른 의미.

## 정리

- setUp 실패 — testMethod 안 실행, 실패 기록
- exception 신중하게 처리
- tearDown은 — 가능하면 항상 실행
