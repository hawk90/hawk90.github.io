## 실제 하드웨어 없이 드라이버 개발하기

리눅스 디바이스 드라이버 개발은 보통 *실제 하드웨어*가 있어야 합니다. 그런데:

- 하드웨어가 아직 없음 (하드웨어 팀이 먼저 끝나야 함)
- 하드웨어가 비쌈 (PCIe 카드 한 장에 수백만 원)
- 하드웨어가 불안정 (프로토타입이라 죽기도 함)
- 에러 주입 어려움 (실제 하드웨어에서 에러 상황 재현 불가)

**QEMU**는 이 모든 문제를 푸는 도구입니다. 가상 머신 안에서 *가상 디바이스*를 만들고, 그 디바이스용 드라이버를 개발합니다.

---

## QEMU란?

QEMU (Quick EMUlator)는 오픈소스 에뮬레이터입니다.

- **전체 시스템 에뮬레이션**: x86, ARM, RISC-V 등 CPU + 페리페럴 전체
- **유저 모드 에뮬레이션**: 다른 아키텍처 바이너리를 호스트에서 실행
- **KVM 연동**: 호스트와 같은 아키텍처일 때 하드웨어 가속

드라이버 개발에서 QEMU의 핵심 가치는:

1. **커스텀 디바이스 추가 가능** — C로 디바이스 모델 작성
2. **재현 가능한 환경** — 같은 QEMU 버전이면 같은 동작
3. **디버깅 용이** — GDB로 커널 + 드라이버 단계별 실행
4. **에러 주입** — 타임아웃, 에러 응답 등 시뮬레이션

## 가상 디바이스 개발 워크플로우

```
1. QEMU 디바이스 모델 작성 (C)
   └── hw/misc/my_device.c

2. QEMU 빌드
   └── ./configure && make

3. 리눅스 커널 + 드라이버 빌드
   └── make M=drivers/my_driver

4. QEMU에서 VM 부팅
   └── qemu-system-x86_64 -device my_device ...

5. VM 안에서 드라이버 로드
   └── insmod my_driver.ko

6. 테스트 / 디버깅
   └── GDB attach, dmesg 확인

7. 반복
```

---

## 이 시리즈에서 다루는 것

1. **QEMU 설치와 빌드** — 소스에서 빌드
2. **QEMU Object Model (QOM)** — 디바이스 모델 프레임워크
3. **PCI 디바이스** — 가장 흔한 디바이스 타입
4. **MMIO 레지스터** — 메모리 맵드 I/O
5. **인터럽트** — MSI/MSI-X 구현
6. **DMA** — 버퍼 전송
7. **드라이버 작성** — 가상 디바이스용 리눅스 드라이버
8. **디버깅** — QEMU + GDB 조합
9. **테스트 자동화** — CI/CD 파이프라인
10. **고급 시나리오** — 에러 주입, 경쟁 조건
11. **사례 연구** — NVMe 가상 디바이스

---

## 정리

- QEMU는 실제 하드웨어 없이 드라이버를 개발/테스트하는 강력한 도구다.
- 가상 디바이스를 C로 작성하고 QEMU에 빌드해 넣는다.
- 리눅스 커널을 VM에서 부팅하고 드라이버를 로드해 테스트한다.
- GDB로 커널 + 드라이버를 단계별 디버깅할 수 있다.
- 에러 주입, 타이밍 테스트 등 실제 하드웨어에서 어려운 시나리오를 시뮬레이션할 수 있다.
